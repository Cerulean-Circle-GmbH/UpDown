#### **5. Generic Component Loading (Phase 8.3.1)** ✅

**The Problem**: No way to dynamically load arbitrary Web4 components

**The Solution**: `componentLoad(name, version)` with delegation pattern

**Architecture**:
```
┌─────────────────────────────────────┐
│         NodeJsOnce                  │
│   componentLoad(name, version)      │
└──────────────┬──────────────────────┘
               │
               ├─ 1. Check cache
               │
               ├─ 2. Delegate to web4tscomponent.componentStart()
               │    └─→ Ensures component is built (npm start)
               │
               ├─ 3. Delegate to web4tscomponent.componentDescriptorRead()
               │    └─→ Reads implementationClassName from descriptor
               │
               ├─ 4. Dynamic import(implementationClassName)
               │    └─→ Loads correct class file
               │
               └─ 5. Cache & return ComponentClass
```

**Web4TSComponent 1.21 - New Methods**:
1. `componentStart(name, version)` - Ensure component built
2. `componentDescriptorRead(name, version)` - Read descriptor

**ONCE Enhancement**:
```typescript
async componentLoad(componentName: string, version: string): Promise<any> {
    // Check cache
    const key = `${componentName}/${version}`;
    if (this.loadedComponents.has(key)) {
        return this.loadedComponents.get(key);
    }

    // Delegate to web4tscomponent (DRY!)
    const web4ts = new DefaultWeb4TSComponent().init();
    await web4ts.componentStart(componentName, version);
    const descriptor = await web4ts.componentDescriptorRead(componentName, version);

    // Get implementation class name
    const implementationClassName = descriptor?.model?.implementationClassName || componentName;

    // Dynamic import
    const componentPath = /* ... */;
    const module = await import(componentPath);
    const ComponentClass = module[implementationClassName];

    // Cache
    this.loadedComponents.set(key, ComponentClass);
    return ComponentClass;
}
```

**Usage Example**:
```bash
# CLI
$ once componentLoad Unit 0.3.19.1

# Code
const UnitClass = await once.componentLoad("Unit", "0.3.19.1");
const unit = new UnitClass().init();
```

**Impact**:
- ✅ Load ANY Web4 component dynamically
- ✅ DRY compliance (delegates to web4tscomponent)
- ✅ Descriptor-driven (implementationClassName)
- ✅ Performance (caching)
- ✅ Testable (web4tscomponent methods tested independently)

**Pattern Benefits**:
1. **Separation of Concerns**:
   - Web4TSComponent: Build + descriptor logic
   - ONCE: Dynamic import + caching
2. **DRY**: No duplicate build/descriptor logic
3. **Extensible**: Add more component management features in web4tscomponent
4. **Proven**: Same pattern as `info()`, `test()`, etc.

## **🆕 NEW WEB4 PRINCIPLES**

### **Principle 17: Component Instance Pattern (startComponent)**
> "Components are versioned, self-contained modules in `components/{Name}/{version}/`. The `static start(config)` method **LITERALLY runs `npm start`** to ensure the component is built, compiled, and all dependencies are installed before creating an instance. Instances are configured scenarios for that component. Use lazy getters to load instances on demand. ALWAYS store instances using `Reference<T>` (Web4 Principle 5), never `Type | null`."

**Example**:
- `HTMLRoute` = Component (class)
- `homeRoute` = Instance (configured scenario)
- `static start()` = Literally `npm start` (builds component)
- `Reference<HTMLRoute>` = Storage pattern

---

### **Principle 18: Black-Box Testing Only**
> "Tests use IOR calls (protocol-agnostic). Tests don't know about HTTP/WS/loaders. White-box tests forbidden. Example: `await kernel.loadViaIOR('ior:...')` not `fetch('http://...')`."

**Benefits**:
- Protocol independence
- Implementation hiding
- Future-proof tests
- True Web4 testing

---

### **Principle 19: Server Binding vs URL Host**
> "Separate `bindInterface` (where to listen) from `urlHost` (how to address). Never confuse binding with addressing."

**Example**:
```typescript
model: {
    bindInterface: "0.0.0.0",  // Listen on all interfaces
    urlHost: "mcdonges-3.fritz.box"  // Address in IORs/scenarios
}
```

---

### **Principle 20: Radical OOP ID Generation**
> "NEVER use functional `uuidv4()` calls. ALWAYS use `idProvider.create()` (OOP, testable, replaceable). IDProvider.interface.ts + UUIDProvider.ts pattern."

**Benefits**:
- Testable (injectable mock)
- Replaceable (UUID → ULID → custom)
- Centralized (one interface)
- Radical OOP compliant
