## 🔴 PROBLEM STATEMENT

### Current State: Two Parallel Persistence Systems

**1. Scenarios (ONCE):**
```typescript
interface Scenario<T extends Model> {
  ior: { uuid, component, version, ... };
  owner: string;  // base64 encoded
  model: T;       // Component-specific model
}
```

**2. Units (Web4Articles):**
```json
{
  "ior": { "uuid": "ccce3656-...", "component": "unit", "version": "0.3.0.5" },
  "owner": "{...}",
  "model": {
    "uuid": "ccce3656-...",
    "name": "DefaultCLI",
    "typeM3": "CLASS",
    "origin": "ior:git:github.com/.../DefaultCLI.ts",
    "definition": "ior:git:github.com/.../DefaultCLI.ts",
    "indexPath": "/workspace/scenarios/index/c/c/c/e/3/ccce3656-...json",
    "symlinkPaths": ["/workspace/components/.../DefaultCLI.ts.unit"],
    "references": [
      {
        "linkLocation": "ior:local:ln:file:/workspace/scenarios/ontology/DefaultCLI.unit",
        "linkTarget": "ior:unit:ccce3656-...",
        "syncStatus": "SYNCED"
      }
    ],
    "createdAt": "2025-09-13T12:56:11.467Z",
    "updatedAt": "2025-09-13T12:56:23.881Z"
  }
}
```

### Key Differences

| Aspect | Scenario | Unit |
|--------|----------|------|
| **indexPath** | Implicit (via UcpStorage) | Explicit in model |
| **symlinkPaths** | Built by caller | In model |
| **references** | Not tracked | Tracked with syncStatus |
| **typeM3** | Not present | M3/M2/M1 classification |
| **origin** | Not present | Source IOR |
| **definition** | Not present | Definition IOR |

### Problems

1. **DRY Violation**: Scenario storage logic duplicated in `UcpStorage`, `ScenarioManager`, `ServerHierarchyManager`
2. **No Reference Tracking**: ONCE scenarios don't track their symlink references
3. **No Version Migration**: No mechanism to upgrade old scenarios to new formats
4. **Data Loss Risk**: Multiple save paths could cause inconsistent state

## 📋 PLAN: Unified Scenario-Unit Architecture

### Target Structure

```typescript
interface UnifiedScenario<T extends Model> {
  // ═══════════════════════════════════════════════════════════════
  // Standard Scenario Fields (unchanged)
  // ═══════════════════════════════════════════════════════════════
  ior: {
    uuid: string;
    component: string;
    version: string;
    protocol?: string;
    host?: string;
    port?: number;
    path?: string;
    profiles?: Array<{ host, port, protocol?, priority? }>;
    iorString?: string;
  };
  owner: string;         // base64 encoded owner
  model: T;              // Component-specific model
  
  // ═══════════════════════════════════════════════════════════════
  // Unit Model Extension (NEW)
  // ═══════════════════════════════════════════════════════════════
  unit?: {
    // Identity (matches ior for self-reference)
    indexPath: string;   // Canonical storage location
    
    // Reference Tracking
    symlinkPaths: string[];   // Created symlinks
    references: UnitReference[];  // All references with sync status
    
    // Classification (MDA M3/M2/M1)
    typeM3?: TypeM3;     // CLASS, ATTRIBUTE, RELATIONSHIP
    
    // Provenance
    origin?: string;     // IOR to original source
    definition?: string; // IOR to definition
    
    // Schema Evolution
    schemaVersion: string;  // e.g., "1.0.0"
    createdBy: string;      // Component that created this
    
    // Timestamps (may duplicate model, but for unit tracking)
    createdAt: string;
    updatedAt: string;
  };
}
```

### Architecture: Single Point of Truth

```
┌─────────────────────────────────────────────────────────────────┐
│                    ScenarioService (Layer 2)                     │
│                    SINGLE POINT OF TRUTH                         │
├─────────────────────────────────────────────────────────────────┤
│  scenarioSave(scenario: UnifiedScenario)                        │
│  scenarioLoad(uuid: string): UnifiedScenario                    │
│  scenarioMigrate(old: any): UnifiedScenario                     │
│  referencesTrack(scenario, linkLocation)                        │
│  referencesSynchronize(scenario)                                 │
├─────────────────────────────────────────────────────────────────┤
│                           ↓                                      │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │            PersistenceManager (Interface)                    ││
│  │  - scenarioSave(uuid, scenario, symlinkPaths)               ││
│  │  - scenarioLoad(uuid)                                        ││

│  └─────────────────────────────────────────────────────────────┘│
│                ↓                         ↓                       │
│    ┌──────────────────┐      ┌──────────────────────┐           │
│    │   UcpStorage     │      │ BrowserScenarioStorage│           │
│    │   (NodeJS)       │      │    (IndexedDB)        │           │
│    └──────────────────┘      └──────────────────────┘           │
└─────────────────────────────────────────────────────────────────┘
```

## 🔄 ACT: Risk Mitigation

### Critical Risks

**Risk 1: Data Loss During Migration**
- **Mitigation**: Write new format to `.scenario.json.new`, verify, rename
- **Rollback**: Keep original until verification succeeds

**Risk 2: Circular Dependencies**
- **Mitigation**: ScenarioService at Layer 2, uses PersistenceManager interface
- **Pattern**: Dependency injection, not direct imports

**Risk 3: Breaking Existing Code**
- **Mitigation**: `unit?` is optional, existing code unchanged
- **Pattern**: Additive changes only in Phase 1
