## 🔴 PROBLEM STATEMENT

### The Interface Impedance Mismatch

**TypeScript**:
```typescript
interface PersistenceManager {
  scenarioSave(uuid: string, scenario: Scenario): Promise<void>;
  scenarioLoad(uuid: string): Promise<Scenario>;
}

class UcpStorage implements PersistenceManager { ... }
```

**JavaScript (after compilation)**:
```javascript
// interface PersistenceManager → ERASED! 🚫
// The word "PersistenceManager" doesn't exist at runtime!

class UcpStorage { ... }  // No implements clause
```

### Current Workaround: Symbols

```typescript
// PersistenceManager.interface.ts
export const PersistenceManager = Symbol('PersistenceManager');

// UcpController.ts
relatedObjectRegister(PersistenceManager, ucpStorageInstance);
relatedObjectLookup(PersistenceManager); // Returns ucpStorageInstance
```

**Problems with Symbols**:
1. Manual registration required
2. No type safety between Symbol and actual interface
3. No introspection of interface methods/properties
4. No automatic discovery of implementations

### The Web4 Vision

> "Web4 is radical OOP in the MDA sense. UcpComponent needs to take the programming 
> language independent role of Web4TSComponent, which is the specialisation for 
> TypeScript. Finally we want a programming language named 'Thinglish' that is not 
> multi-paradigm but radical Web4 OOP MDA CORBA driven only."

**Goal**: Interfaces as first-class citizens with:
- Runtime existence (like classes)
- Type descriptors (introspection)
- Implementation registry (service locator)
- Language-independent representation

## 📋 PLAN: JsInterface Architecture

### Core Concepts

#### 1. JsInterface Base Class
```typescript
/**
 * JsInterface - Base class for runtime-existing interfaces
 * All Web4 interfaces extend this class
 * 
 * Makes TypeScript interfaces "real" at runtime by:
 * - Creating a class that represents the interface
 * - Storing type metadata in static typeDescriptor
 * - Tracking all implementations
 */
export abstract class JsInterface {
  /** Type descriptor - introspection metadata */
  static type: TypeDescriptor;
  
  /** Implementation registry */
  static implementations: Set<InterfaceConstructor> = new Set();
  
  /** Register an implementing class */
  static registerImplementation(implementationClass: InterfaceConstructor): void {
    this.implementations.add(implementationClass);
    // Also register in the class's typeDescriptor
    implementationClass.type?.implementsAdd(this);
  }
  
  /** Get all implementations */
  static getImplementations(): InterfaceConstructor[] {
    return Array.from(this.implementations);
  }
}
```

#### 2. TypeDescriptor Class
```typescript
/**
 * TypeDescriptor - MDA MOF-inspired type metadata
 * 
 * Contains code-first information about a class:
 * - Identity (name, extends, implements)
 * - Structure (attributes, properties, methods)
 * - Relationships (references, collections)
 */
export class TypeDescriptor {
  // ═══════════════════════════════════════════════════════════════
  // Identity
  // ═══════════════════════════════════════════════════════════════
  name: string;                      // Class name
  extends: Reference<TypeDescriptor>;  // Parent class descriptor (Web4 P5)
  implements: Set<InterfaceConstructor>;  // Implemented interfaces
  
  // ═══════════════════════════════════════════════════════════════
  // Structure - MDA M2 Level (Class Definition)
  // ═══════════════════════════════════════════════════════════════
  
  /** Model attributes (properties without getters/setters) */
  attributes: Map<string, AttributeDescriptor>;
  
  /** Properties (TypeScript getters/setters) */
  properties: Map<string, PropertyDescriptor>;
  
  /** References to other objects (Reference<T>) */
  references: Map<string, ReferenceDescriptor>;
  
  /** Collections (Collection<T>) */
  collections: Map<string, CollectionDescriptor>;
  
  /** Methods */
  methods: Map<string, MethodDescriptor>;
  
  // ═══════════════════════════════════════════════════════════════
  // Registry
  // ═══════════════════════════════════════════════════════════════
  
  /** Classes that implement this type (for interfaces) */
  implementingClasses: Set<InterfaceConstructor>;
  
  // ═══════════════════════════════════════════════════════════════
  // Methods
  // ═══════════════════════════════════════════════════════════════
  
  implementsAdd(interfaceType: JsInterface): void;
  implementsCheck(interfaceType: JsInterface): boolean;
  
  attributeGet(name: string): AttributeDescriptor | undefined;
  propertyGet(name: string): PropertyDescriptor | undefined;
  methodGet(name: string): MethodDescriptor | undefined;
  
  /** Build from TypeScript AST node (adaptor) */
  static fromTSAst(astNode: ts.ClassDeclaration): TypeDescriptor;
  
  /** Build from runtime class (reflection) */
  static fromClass(classConstructor: InterfaceConstructor): TypeDescriptor;
}
```

#### 3. Descriptor Types
```typescript
interface AttributeDescriptor {
  name: string;
  type: string;          // TypeScript type as string
  optional: boolean;
  defaultValue?: unknown;
}

interface PropertyDescriptor {
  name: string;
  type: string;
  hasGetter: boolean;
  hasSetter: boolean;
  isReadonly: boolean;
}

interface ReferenceDescriptor {
  name: string;
  targetType: string;    // Referenced type name
  nullable: boolean;     // Reference<T> is always nullable
}

interface CollectionDescriptor {
  name: string;
  elementType: string;   // Collection<T> element type
}

interface MethodDescriptor {
  name: string;
  parameters: ParameterDescriptor[];
  returnType: string;
  isAsync: boolean;
  isAction: boolean;     // Has @action TSDoc tag
  actionMetadata?: ActionMetadata;
}

interface ParameterDescriptor {
  name: string;
  type: string;
  optional: boolean;
  defaultValue?: unknown;
}
```

### Architecture: Static Type Injection

```
┌─────────────────────────────────────────────────────────────────┐
│                    Class Loading Process                         │
├─────────────────────────────────────────────────────────────────┤
│  1. import { UcpStorage } from './UcpStorage.js'                │

│                           ↓                                      │

│  │  - getType(classConstructor): TypeDescriptor                ││

└─────────────────────────────────────────────────────────────────┘
```

### Converting Interfaces to JsInterface

**Before (TypeScript interface - erased at runtime)**:
```typescript
export interface PersistenceManager {
  scenarioSave(uuid: string, scenario: Scenario): Promise<void>;
  scenarioLoad(uuid: string): Promise<Scenario>;
}
```

**After (JsInterface - exists at runtime)**:
```typescript
/**
 * PersistenceManager - Runtime interface definition
 * Extends JsInterface for runtime existence
 */
export abstract class PersistenceManager extends JsInterface {
  /** Type descriptor with method signatures */
  static type: TypeDescriptor = TypeDescriptor.fromDefinition({
    name: 'PersistenceManager',
    methods: {
      scenarioSave: { params: ['uuid: string', 'scenario: Scenario'], returns: 'Promise<void>' },
      scenarioLoad: { params: ['uuid: string'], returns: 'Promise<Scenario>' }
    }
  });
  
  // Abstract methods define the interface contract
  abstract scenarioSave(uuid: string, scenario: Scenario<Model>): Promise<void>;
  abstract scenarioLoad(uuid: string): Promise<Scenario<Model>>;
}

// Implementation registers automatically via UcpComponent.start()
export class UcpStorage extends UcpComponent implements PersistenceManager {
  static start(): void {
    super.start();
    PersistenceManager.registerImplementation(this);
  }
}
```

## 🔄 ACT: Considerations

### TypeScript-JavaScript Impedance

**Challenge**: TypeScript `implements` clause is erased at runtime.

**Solution A: Decorator-based registration**
```typescript
@implements(PersistenceManager)
export class UcpStorage extends UcpComponent { }
```

**Solution B: Static start() registration**
```typescript
export class UcpStorage extends UcpComponent {
  static start(): void {
    super.start();
    PersistenceManager.registerImplementation(this);
  }
}
```

**Solution C: Reflection-based discovery**
```typescript
// At build time or runtime, analyze prototype chain
// and match method signatures to interface definitions
```

### Recommended: Solution B (static start())
- Explicit registration (no magic)
- Works with standard TypeScript
- No decorators required
- Aligns with Web4 "explicit over implicit"
