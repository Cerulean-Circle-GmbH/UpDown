## 🔴 PROBLEM STATEMENT

### The Interface Impedance Mismatch

**TypeScript**:
```typescript
interface PersistenceManager {
  scenarioSave(uuid: string, scenario: Scenario): Promise<void>;
  scenarioLoad(uuid: string): Promise<Scenario>;
}

class UcpStorage implements PersistenceManager { ... }
```

**JavaScript (after compilation)**:
```javascript
// interface PersistenceManager → ERASED! 🚫
// The word "PersistenceManager" doesn't exist at runtime!

class UcpStorage { ... }  // No implements clause
```

### Current Workaround: Symbols

```typescript
// PersistenceManager.interface.ts
export const PersistenceManager = Symbol('PersistenceManager');

// UcpController.ts
relatedObjectRegister(PersistenceManager, ucpStorageInstance);
relatedObjectLookup(PersistenceManager); // Returns ucpStorageInstance
```

**Problems with Symbols**:
1. Manual registration required
2. No type safety between Symbol and actual interface
3. No introspection of interface methods/properties
4. No automatic discovery of implementations

### The Web4 Vision

> "Web4 is radical OOP in the MDA sense. UcpComponent needs to take the programming

> TypeScript. Finally we want a programming language named 'Thinglish' that is not

**Goal**: Interfaces as first-class citizens with:

## 📋 PLAN: JsInterface Architecture

### Core Concepts

#### 1. JsInterface Base Class
```typescript
/**
 * JsInterface - Base class for runtime-existing interfaces
 * All Web4 interfaces extend this class
 * 
 * Makes TypeScript interfaces "real" at runtime by:
 * - Creating a class that represents the interface
 * - Storing type metadata in static typeDescriptor
 * - Tracking all implementations
 */
export abstract class JsInterface {
  /** Type descriptor - introspection metadata */
  static type: TypeDescriptor;
  
  /** Implementation registry */
  static implementations: Set<InterfaceConstructor> = new Set();
  
  /** Register an implementing class */
  static registerImplementation(implementationClass: InterfaceConstructor): void {
    this.implementations.add(implementationClass);
    // Also register in the class's typeDescriptor
    implementationClass.type?.implementsAdd(this);
  }
  
  /** Get all implementations */
  static getImplementations(): InterfaceConstructor[] {
    return Array.from(this.implementations);
  }
}
```

#### 2. TypeDescriptor Class
```typescript
/**
 * TypeDescriptor - MDA MOF-inspired type metadata
 * 
 * SINGLE POINT OF TRUTH: Built from TypeScript AST, never runtime reflection.
 * 
 * Contains code-first information about a class:
 * - Identity (name, extends, implements)
 * - Structure (attributes, properties, methods)
 * - Relationships (references, collections)
 */
export class TypeDescriptor implements FeatureDescriptor {
  // ═══════════════════════════════════════════════════════════════
  // Identity (FeatureDescriptor: name required)
  // ═══════════════════════════════════════════════════════════════
  name: string;                      // Class name (FeatureDescriptor)
  extends: Reference<TypeDescriptor>;  // Parent class descriptor (Web4 P5)
  implements: Set<InterfaceConstructor>;  // Implemented interfaces
  
  // ═══════════════════════════════════════════════════════════════
  // Structure - MDA M2 Level (Class Definition)
  // ═══════════════════════════════════════════════════════════════
  
  /** Model attributes (properties without getters/setters) */
  attributes: Map<string, AttributeDescriptor>;
  
  /** Properties (TypeScript getters/setters) */
  properties: Map<string, PropertyDescriptor>;
  
  /** References to other objects (Reference<T> - nullable, serialize as IOR) */
  references: Map<string, ReferenceDescriptor>;
  
  /** Collections (Collection<T> - array of references, serialize as IOR[]) */
  collections: Map<string, CollectionDescriptor>;
  
  /** Methods */
  methods: Map<string, MethodDescriptor>;
  
  // ═══════════════════════════════════════════════════════════════
  // Registry (populated during static start() from AST data)
  // ═══════════════════════════════════════════════════════════════
  
  /** Classes that implement this type (for interfaces) */
  implementingClasses: Set<InterfaceConstructor>;
  
  // ═══════════════════════════════════════════════════════════════
  // Methods
  // ═══════════════════════════════════════════════════════════════
  
  implementsAdd(interfaceType: JsInterface): void;
  implementsCheck(interfaceType: JsInterface): boolean;
  
  attributeGet(name: string): Reference<AttributeDescriptor>;
  propertyGet(name: string): Reference<PropertyDescriptor>;
  methodGet(name: string): Reference<MethodDescriptor>;
  referenceGet(name: string): Reference<ReferenceDescriptor>;
  collectionGet(name: string): Reference<CollectionDescriptor>;
  
  // ═══════════════════════════════════════════════════════════════
  // Factory Methods (AST-First, Code-First Principle)
  // ═══════════════════════════════════════════════════════════════
  
  /** Build from TypeScript AST data (.type.json) - PREFERRED */
  static fromASTData(astData: TypeDescriptorJSON): TypeDescriptor;
  
  /** Build from TypeScript AST node during build - USED BY TsAstExtractor */
  static fromTSAstNode(astNode: ts.ClassDeclaration | ts.InterfaceDeclaration): TypeDescriptor;
  
  // ❌ NO fromClass() - runtime reflection violates DRY principle
}

/** JSON representation stored in .type.json */
interface TypeDescriptorJSON {
  name: string;
  extends: Reference<string>;  // IOR of parent type
  implements: string[];        // IORs of implemented interfaces
  attributes: AttributeDescriptor[];
  properties: PropertyDescriptor[];
  references: ReferenceDescriptor[];
  collections: CollectionDescriptor[];
  methods: MethodDescriptor[];
}
```

#### 3. Descriptor Types (All extend FeatureDescriptor)

**Web4 Principle: DRY + Code-First**
> All descriptors' single point of truth is the code and the component's TypeScript AST.
> Descriptors are built from AST after build, never from runtime reflection.

```typescript
/**
 * FeatureDescriptor - Base for all descriptor types
 * Minimal attribute: name
 */
interface FeatureDescriptor {
  name: string;  // Required - every feature has a name
}

/**
 * AttributeDescriptor - Model properties (data fields)
 */
interface AttributeDescriptor extends FeatureDescriptor {
  name: string;
  type: string;          // TypeScript type as string
  optional: boolean;
  defaultValue?: unknown;
}

/**
 * PropertyDescriptor - TypeScript getters/setters
 */
interface PropertyDescriptor extends FeatureDescriptor {
  name: string;
  type: string;
  hasGetter: boolean;
  hasSetter: boolean;
  isReadonly: boolean;
}

/**
 * ReferenceDescriptor - Nullable references to instances
 * Reference<T> = T | null
 * Can be serialized into scenarios as IORs
 */
interface ReferenceDescriptor extends FeatureDescriptor {
  name: string;
  targetType: string;    // Referenced type name
  // References are ALWAYS nullable (Reference<T> pattern)
  // When serialized: becomes IOR string in scenario
}

/**
 * CollectionDescriptor - Arrays of references
 * Collection<T> = Reference<T>[]
 * Each element can be serialized as IOR
 */
interface CollectionDescriptor extends FeatureDescriptor {
  name: string;
  elementType: string;   // Collection<T> element type
  // Elements are references, serialized as IOR array
}

/**
 * MethodDescriptor - Class methods
 */
interface MethodDescriptor extends FeatureDescriptor {
  name: string;
  parameters: ParameterDescriptor[];
  returnType: string;
  isAsync: boolean;
  isAction: boolean;     // Has @action TSDoc tag
  actionMetadata?: ActionMetadata;
}

/**
 * ParameterDescriptor - Method parameters
 */
interface ParameterDescriptor extends FeatureDescriptor {
  name: string;
  type: string;
  optional: boolean;
  defaultValue?: unknown;
}
```

#### 4. Reference & Collection Serialization

```typescript
// Reference<T> serialized as IOR in scenarios
interface ReferenceInScenario {
  ior: string;  // e.g., "ior:esm:/ONCE/0.3.21.8/UcpStorage"
}

// Collection<T> serialized as IOR array
interface CollectionInScenario {
  iors: string[];  // Array of IOR strings
}
```

### Architecture: TypeRegistry as ONCE Kernel Component

**TypeRegistry is integral to ONCE** - the kernel responsible for:
- Loading/unloading classes and units
- Starting/stopping classes
- Initializing types from AST
- Propagating scenario updates
- Hibernating and restoring object state

```
┌─────────────────────────────────────────────────────────────────┐
│                    ONCE Kernel Lifecycle                         │
├─────────────────────────────────────────────────────────────────┤
│  1. BUILD PHASE (tsc compiles .ts → .js)                        │
│     - TypeScript AST available in source files                   │
│     - Generate .type.json from AST (single point of truth)       │
│                                                                  │
│  2. LOAD PHASE (import module)                                   │
│     - import { UcpStorage } from './UcpStorage.js'              │
│     - Load corresponding UcpStorage.type.json                    │
│                                                                  │
│  3. START PHASE (static start())                                 │
│     - TypeRegistry.register(UcpStorage, typeJsonData)           │
│     - Build TypeDescriptor from AST data (not reflection!)       │
│     - UcpStorage.type = typeDescriptor                           │
│     - Auto-discover implements from AST data                     │
│     - Register implementations with JsInterface classes          │
│                                                                  │
│  4. INIT PHASE (instance.init(scenario))                        │
│     - Restore object state from scenario                         │
│     - Resolve Reference<T> from IORs                             │
│     - Resolve Collection<T> from IOR arrays                      │
│                                                                  │
│  5. HIBERNATE PHASE (instance.toScenario())                     │
│     - Serialize object state                                     │
│     - Convert Reference<T> to IORs                               │
│     - Convert Collection<T> to IOR arrays                        │
│                                                                  │
│  6. STOP PHASE (static stop())                                   │
│     - Unregister from TypeRegistry                               │
│     - Clean up implementations                                   │
├─────────────────────────────────────────────────────────────────┤
│                           ↓                                      │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │           TypeRegistry (Layer 2 - ONCE Kernel)               ││
│  │  - register(class, astData): void                           ││
│  │  - typeGet(class): TypeDescriptor                           ││
│  │  - implementationsGet(interface): Constructor[]             ││
│  │  - descriptorBuildFromAST(astData): TypeDescriptor          ││
│  │  - classLoad(ior): Promise<Constructor>                     ││
│  │  - classUnload(class): void                                 ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

**Web4 Principle: DRY + Code-First**
> The TypeScript AST is the SINGLE POINT OF TRUTH for all type information.
> TypeDescriptors are built from AST data, never from runtime reflection.
> This ensures consistency between compile-time types and runtime metadata.

### Converting Interfaces to JsInterface

**Before (TypeScript interface - erased at runtime)**:
```typescript
export interface PersistenceManager {
  scenarioSave(uuid: string, scenario: Scenario): Promise<void>;
  scenarioLoad(uuid: string): Promise<Scenario>;
}
```

**After (JsInterface - exists at runtime)**:
```typescript
/**
 * PersistenceManager - Runtime interface definition
 * Extends JsInterface for runtime existence
 */
export abstract class PersistenceManager extends JsInterface {
  /** Type descriptor with method signatures */
  static type: TypeDescriptor = TypeDescriptor.fromDefinition({
    name: 'PersistenceManager',
    methods: {
      scenarioSave: { params: ['uuid: string', 'scenario: Scenario'], returns: 'Promise<void>' },
      scenarioLoad: { params: ['uuid: string'], returns: 'Promise<Scenario>' }
    }
  });
  
  // Abstract methods define the interface contract
  abstract scenarioSave(uuid: string, scenario: Scenario<Model>): Promise<void>;
  abstract scenarioLoad(uuid: string): Promise<Scenario<Model>>;
}

// Implementation registers automatically via UcpComponent.start()
export class UcpStorage extends UcpComponent implements PersistenceManager {
  static start(): void {
    super.start();
    PersistenceManager.registerImplementation(this);
  }
}
```

## 🔄 ACT: Considerations

### Web4 Principles Applied

#### P28: DRY + Code-First (NEW)
> All descriptors' single point of truth is the code and the component's TypeScript AST.
> Never use runtime reflection when AST data is available.

#### P5: Reference<T>
> References are nullable by definition (`T | null`).
> Serialized as IOR strings in scenarios.

#### P22: Collection<T>
> Collections are arrays of references.
> Serialized as IOR arrays in scenarios.

### TypeScript-JavaScript Impedance

**Challenge**: TypeScript `implements` clause is erased at runtime.

**❌ REJECTED: Decorator-based registration**
```typescript
@implements(PersistenceManager)
export class UcpStorage extends UcpComponent { }
// Violates Web4 "no magic" principle
```

**❌ REJECTED: Runtime reflection**
```typescript
// Analyze prototype chain at runtime
// Violates DRY - duplicates info already in AST
```

**✅ CHOSEN: AST + Static start()**
```typescript
// At BUILD TIME: Generate .type.json from TypeScript AST
// At RUNTIME: Load .type.json in static start()

export class UcpStorage extends UcpComponent implements PersistenceManager {
  static start(): void {
    super.start();
    // TypeRegistry loads UcpStorage.type.json (from AST)
    // Auto-discovers: implements [PersistenceManager]
    // Auto-registers: PersistenceManager.implementations.add(UcpStorage)
  }
}
```

### Why AST-First?

1. **Single Point of Truth**: TypeScript source is authoritative
2. **DRY**: No duplicate metadata definitions
3. **Type Safety**: AST preserves full type information
4. **Build-Time Validation**: Catch errors before runtime
5. **Language Independence**: .type.json is portable
