### 3. Identify Root Cause

**Issue**: `DefaultCLI.getValidCompletionValues()` calls:
```typescript
if (this.component !== null) {
  this.component.listMethods().forEach((name: string) => allMethodNames.add(name));
}
```

**Problem**:
1. ONCECLI stores delegated methods in `this.methodSignatures` Map ✅
2. But `this.component.listMethods()` returns ONLY NodeJsOnce methods (by design)
3. Parent never checks `this.methodSignatures` Map
4. Result: Delegated methods invisible to completion

### 4. Implement Fix

**Solution**: Override `listMethods()` in ONCECLI to aggregate ALL methods:

```typescript:148:177:components/ONCE/0.3.21.8/src/ts/layer5/ONCECLI.ts
/**
 * Override listMethods to include delegated methods for completion
 * @pdca 2025-12-02-UTC-1830.pdca.md - ROOT CAUSE FIX
 * Parent's getValidCompletionValues() calls this.component.listMethods() which only returns
 * NodeJsOnce methods, NOT delegated Web4TSComponent methods like 'tootsie', 'test', etc.
 * Solution: Override listMethods() at CLI level to aggregate ALL methods
 */
listMethods(): string[] {
  const methods = new Set<string>();
  
  // Add component's own methods (if component exists)
  if (this.component && typeof this.component.listMethods === 'function') {
    this.component.listMethods().forEach((m: string) => methods.add(m));
  }
  
  // Add ALL discovered methods from our methodSignatures Map
  // This includes both own methods AND delegated methods
  for (const methodName of this.methodSignatures.keys()) {
    methods.add(methodName);
  }
  
  return Array.from(methods).sort();
}
```

## **🎯 ACT**

**Success Achieved:** Shell tab completion restored for delegated methods

**Root Cause Understanding:**
- **DelegationProxy pattern**: `listMethods()` returns ONLY target methods (by design)
- **CLI responsibility**: Must aggregate component + delegated methods
- **Fix location**: CLI level (ONCECLI.listMethods() override)

**Architecture Benefits:**
- Clean separation: Proxy doesn't aggregate, CLI does
- Explicit delegation discovery via `hasDelegation()` + `getDelegationTarget()`
- Maintainable: All completion logic in one place (CLI)

**Future Enhancements:**
1. **Parameter completion**: Propagate @cliValues through delegation chain
