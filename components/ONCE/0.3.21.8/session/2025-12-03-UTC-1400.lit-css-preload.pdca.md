## 🔴 Problem Statement

### Problem 1: CSS Loading

Lit components require CSS at **class definition time** via `static styles`. However:

1. Web4 Principle 19 mandates external CSS files (separation of concerns)
2. Fetching CSS is **asynchronous** - happens AFTER class definition
3. HTML Import Maps only resolve top-level imports, NOT dynamically imported modules
4. Compiled JS files have bare `import { css } from 'lit'` that browsers can't resolve

**Current Blocker:** The `/demo-lit` page shows a black screen because Lit components can't load.

### Problem 2: HTML Template Loading

Web4 Principle 19 also mandates external HTML templates:

```
layer5/views/webBeans/
├── ItemView.html
├── DefaultView.html
└── OnceOverView.html
```

Lit's `render()` method expects a `TemplateResult` returned synchronously. But:

1. External HTML templates require async fetch
2. Templates need to be parsed into Lit's `html` tagged template format
3. Template interpolation (`${this.model.name}`) needs to work

**Goal:** Preload both CSS AND HTML templates before component registration.

## 📋 PLAN: Analysis of Solutions

### Option A: Vite/Bundler (Traditional Solution)

```
Build Step → Bundle Lit + CSS → Single file with inlined CSS
```

**Pros:**
- Industry standard
- Resolves all imports at build time
- Tree shaking, minification

**Cons:**
- ❌ Violates Web4 Principle 21 (wrap functional tools in OOP)
- ❌ Introduces external dependency
- ❌ Requires additional build step
- ❌ Not "Web4 native"

### Option B: HTML Preload + adoptedStyleSheets

```html
<link rel="preload" href="/css/ItemView.css" as="style">
```

Then use `adoptedStyleSheets` API:

```typescript
// UcpView.connectedCallback()
const sheet = new CSSStyleSheet();
sheet.replaceSync(await fetch(this.cssPath).then(r => r.text()));
this.shadowRoot.adoptedStyleSheets = [sheet];
```

**Pros:**
- ✅ No bundler needed
- ✅ Browser-native API
- ✅ Works with Lit

**Cons:**
- ⚠️ Requires async setup before render
- ⚠️ CSS flicker on first render (FOUC)
- ⚠️ Not "static styles" pattern

### Option C: Unit Component CSS Tracking (Web4 Native)

Each CSS file becomes a **Unit** with UUID:

```
layer5/views/css/
├── ItemView.css          → Unit UUID: abc123
├── OnceOverView.css      → Unit UUID: def456
└── themeBase.css         → Unit UUID: ghi789
```

**Unit Scenario:**
```json
{
  "ior": { "uuid": "abc123", "component": "Unit", "version": "0.3.19.1" },
  "model": {
    "origin": "ior:local:file://layer5/views/css/ItemView.css",
    "mimeType": "text/css",
    "contentHash": "sha256:...",
    "preload": true
  }
}
```

**Pros:**
- ✅ Pure Web4 OOP
- ✅ CSS files are tracked objects with UUIDs
- ✅ Can sync, version, and reference CSS

**Cons:**
- ⚠️ Requires Unit component integration
- ⚠️ More complex setup

### Option D: ONCE CSSLoader (Inspired by Legacy ONCE)

Create a `CSSLoader` class that preloads CSS before component registration:

```typescript
// layer2/CSSLoader.ts
export class CSSLoader {
  private static cache: Map<string, CSSStyleSheet> = new Map();
  
  static async preload(cssPath: string): Promise<CSSStyleSheet> {
    if (this.cache.has(cssPath)) {
      return this.cache.get(cssPath)!;
    }
    
    const response = await fetch(cssPath);
    const cssText = await response.text();
    const sheet = new CSSStyleSheet();
    sheet.replaceSync(cssText);
    
    this.cache.set(cssPath, sheet);
    return sheet;
  }
  
  static async preloadAll(cssPaths: string[]): Promise<void> {
    await Promise.all(cssPaths.map(this.preload.bind(this)));
  }
}
```

**Usage in demo-lit.html:**
```javascript
// Preload CSS BEFORE importing components
await CSSLoader.preloadAll([
  '/dist/ts/layer5/views/css/themeBase.css',
  '/dist/ts/layer5/views/css/ItemView.css',
  '/dist/ts/layer5/views/css/OnceOverView.css'
]);

// NOW import components (CSS is ready)
import './dist/ts/layer5/views/OnceOverView.js';
```

**Pros:**
- ✅ Simple, no external dependencies
- ✅ CSS loaded before components
- ✅ Uses browser-native `CSSStyleSheet`
- ✅ Can be OOP class

**Cons:**
- ⚠️ Must manually list CSS files
- ⚠️ Components need to use adoptedStyleSheets

### Option E: UcpView + CSSLoader + Unit (Combined Solution) ⭐

**The Web4 Way:**

1. **Unit** tracks CSS files with UUIDs and metadata
2. **CSSLoader** (or `UcpCSSManager`) preloads CSS as `CSSStyleSheet`
3. **AbstractWebBean** uses `adoptedStyleSheets` from cache
4. **StaticFileRoute** serves CSS with correct MIME types

```
┌─────────────────────────────────────────────────────────────┐
│  Browser Boot Sequence                                       │
├─────────────────────────────────────────────────────────────┤
│  1. HTML loads                                               │
│  2. CSSLoader.preloadAll() fetches all CSS → CSSStyleSheet  │
│  3. Import Lit components (CSS already in cache)            │
│  4. Components use adoptedStyleSheets from cache            │
│  5. Render without FOUC                                      │
└─────────────────────────────────────────────────────────────┘
```

**UcpView Enhancement:**

```typescript
export abstract class UcpView<TModel = any> extends LitElement {
  
  // Each view declares its CSS path
  static cssPath: string = '';
  
  connectedCallback(): void {
    super.connectedCallback();
    
    // Apply preloaded CSS from cache
    const cssPath = (this.constructor as typeof UcpView).cssPath;
    if (cssPath) {
      const sheet = CSSLoader.get(cssPath);
      if (sheet) {
        this.shadowRoot!.adoptedStyleSheets = [sheet];
      }
    }
  }
}
```

### Option F: HTMLTemplateLoader (Parallel to CSSLoader)

Create an `HTMLTemplateLoader` class that preloads HTML templates:

```typescript
// layer2/HTMLTemplateLoader.ts
export class HTMLTemplateLoader {
  private static cache: Map<string, string> = new Map();
  
  static async preload(templatePath: string): Promise<string> {
    if (this.cache.has(templatePath)) {
      return this.cache.get(templatePath)!;
    }
    
    const response = await fetch(templatePath);
    const htmlText = await response.text();
    
    this.cache.set(templatePath, htmlText);
    return htmlText;
  }
  
  static get(templatePath: string): string | null {
    return this.cache.get(templatePath) || null;
  }
  
  static async preloadAll(templatePaths: string[]): Promise<void> {
    await Promise.all(templatePaths.map(this.preload.bind(this)));
  }
}
```

**Usage in AbstractWebBean:**

```typescript
export abstract class AbstractWebBean<TModel> extends UcpView<TModel> {
  
  abstract get templatePath(): string;
  
  render(): TemplateResult {
    const template = HTMLTemplateLoader.get(this.templatePath);
    if (!template) {
      return html`<div class="loading">Loading template...</div>`;
    }
    
    // Parse template with model interpolation
    return this.parseTemplate(template);
  }
  
  private parseTemplate(htmlString: string): TemplateResult {
    // Replace ${model.xxx} with actual values
    const processed = this.interpolate(htmlString);
    return html([processed] as any);  // unsafe but works
  }
  
  private interpolate(template: string): string {
    // Simple interpolation: ${this.model.xxx} → actual value
    return template.replace(/\$\{this\.model\.(\w+)\}/g, 
      (match, prop) => String(this.model?.[prop] ?? ''));
  }
}
```

**Template Example (`webBeans/ItemView.html`):**

```html
<div class="status-dot ${this.model.state}"></div>
<div class="item-info">
  <div class="item-title">Peer ${this.model.uuid}</div>
  <div class="item-subtitle">Port ${this.model.port} · ${this.model.state}</div>
</div>
```

**Pros:**
- ✅ HTML templates are external files (P19)
- ✅ Simple string interpolation
- ✅ Preloaded before render

**Cons:**
- ⚠️ Not full Lit template syntax (no @click, .property)
- ⚠️ Need custom parser for bindings
- ⚠️ Limited reactivity

### Option G: Hybrid Approach (Templates + Lit Render) ⭐

Use external HTML for **structure only**, Lit for **bindings**:

```html
<!-- webBeans/ItemView.html - Structure only, no bindings -->
<div class="status-dot"></div>
<div class="item-info">
  <div class="item-title"></div>
  <div class="item-subtitle"></div>
</div>
<div class="action-zone"></div>
```

```typescript
// ItemView.ts - Lit handles bindings
render(): TemplateResult {
  const template = HTMLTemplateLoader.get(this.templatePath);
  
  // Inject Lit bindings into structure
  return html`
    <div class="status-dot ${this.model.state?.toLowerCase()}"></div>
    <div class="item-info">
      <div class="item-title">Peer ${this.model.uuid?.substring(0, 8)}</div>
      <div class="item-subtitle">Port ${this.model.port} · ${this.model.state}</div>
    </div>
    <div class="action-zone">
      ${this.renderActions()}
    </div>
  `;
}
```

**Pros:**
- ✅ Full Lit reactivity and bindings
- ✅ External templates define structure
- ✅ Best of both worlds

**Cons:**
- ⚠️ Templates and TS must stay in sync
- ⚠️ Templates become documentation more than runtime

### Option H: Unit-Tracked Templates with WebBean Registry

Each template is a **Unit** with metadata:

```json
{
  "ior": { "uuid": "xyz789", "component": "Unit", "version": "0.3.19.1" },
  "model": {
    "origin": "ior:local:file://layer5/views/webBeans/ItemView.html",
    "mimeType": "text/html",
    "viewComponent": "ItemView",
    "cssUnit": "ior:unit:uuid:abc123",
    "bindings": ["model.uuid", "model.port", "model.state"]
  }
}
```

**WebBeanRegistry:**

```typescript
export class WebBeanRegistry {
  private static beans: Map<string, WebBeanDescriptor> = new Map();
  
  static async discover(): Promise<void> {
    // Fetch all Unit scenarios for webBeans
    const response = await fetch('/units?type=webBean');
    const units = await response.json();
    
    for (const unit of units) {
      await CSSLoader.preload(unit.cssPath);
      await HTMLTemplateLoader.preload(unit.templatePath);
      this.beans.set(unit.viewComponent, unit);
    }
  }
  
  static get(viewName: string): WebBeanDescriptor | null {
    return this.beans.get(viewName) || null;
  }
}
```

## ✅ Recommended Solution: Option E (CSS) + Option G (HTML)

**Rationale:**
- Pure Web4 OOP (no bundler)
- CSS and HTML files are external (P19 compliance)
- Browser-native APIs (CSSStyleSheet, adoptedStyleSheets)
- Preloading prevents FOUC
- Compatible with Lit's shadow DOM
- HTML templates define structure, Lit handles bindings

**Architecture:**

```
┌─────────────────────────────────────────────────────────────────┐
│  Browser Boot Sequence                                           │
├─────────────────────────────────────────────────────────────────┤
│  1. HTML loads demo-lit.html                                     │
│  2. Fetch /asset-manifest → { css: [...], templates: [...] }    │
│  3. CSSLoader.preloadAll() → CSSStyleSheet cache                 │
│  4. HTMLTemplateLoader.preloadAll() → template string cache      │
│  5. Import Lit components (assets already cached)                │
│  6. connectedCallback() applies adoptedStyleSheets               │
│  7. render() uses cached template structure + Lit bindings       │
│  8. Render without FOUC                                          │
└─────────────────────────────────────────────────────────────────┘
```

**Directory Structure:**

```
layer5/views/
├── css/
│   ├── themeBase.css       # CSS variables
│   ├── ItemView.css        # Item styles
│   └── OnceOverView.css    # Overview styles
```

### ✅ RESOLVED - Full Implementation Complete

**Root Cause:** Lit's `update()` method is a protected lifecycle method. Our override was breaking the render cycle.

**Fix:**
1. Renamed `update()` to `refresh()` in View interface and UcpController
2. Removed `update()` override from UcpView (let Lit handle its lifecycle)
3. Use `requestUpdate()` directly instead of `this.update()`
4. `connectedCallback()` now correctly applies `adoptedStyleSheets`

**Result:** Lit component renders with external CSS loaded via `CSSLoader.preloadAll()`!
