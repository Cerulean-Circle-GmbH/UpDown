## 🔴 Problem Statement

Lit components require CSS at **class definition time** via `static styles`. However:

1. Web4 Principle 19 mandates external CSS files (separation of concerns)
2. Fetching CSS is **asynchronous** - happens AFTER class definition
3. HTML Import Maps only resolve top-level imports, NOT dynamically imported modules
4. Compiled JS files have bare `import { css } from 'lit'` that browsers can't resolve

**Current Blocker:** The `/demo-lit` page shows a black screen because Lit components can't load.

---

## 📋 PLAN: Analysis of Solutions

### Option A: Vite/Bundler (Traditional Solution)

```
Build Step → Bundle Lit + CSS → Single file with inlined CSS
```

**Pros:**
- Industry standard
- Resolves all imports at build time
- Tree shaking, minification

**Cons:**
- ❌ Violates Web4 Principle 21 (wrap functional tools in OOP)
- ❌ Introduces external dependency
- ❌ Requires additional build step
- ❌ Not "Web4 native"

### Option B: HTML Preload + adoptedStyleSheets

```html
<link rel="preload" href="/css/ItemView.css" as="style">
```

Then use `adoptedStyleSheets` API:

```typescript
// UcpView.connectedCallback()
const sheet = new CSSStyleSheet();
sheet.replaceSync(await fetch(this.cssPath).then(r => r.text()));
this.shadowRoot.adoptedStyleSheets = [sheet];
```

**Pros:**
- ✅ No bundler needed
- ✅ Browser-native API
- ✅ Works with Lit

**Cons:**
- ⚠️ Requires async setup before render
- ⚠️ CSS flicker on first render (FOUC)
- ⚠️ Not "static styles" pattern

### Option C: Unit Component CSS Tracking (Web4 Native)

Each CSS file becomes a **Unit** with UUID:

```
layer5/views/css/
├── ItemView.css          → Unit UUID: abc123
├── OnceOverView.css      → Unit UUID: def456
└── themeBase.css         → Unit UUID: ghi789
```

**Unit Scenario:**
```json
{
  "ior": { "uuid": "abc123", "component": "Unit", "version": "0.3.19.1" },
  "model": {
    "origin": "ior:local:file://layer5/views/css/ItemView.css",
    "mimeType": "text/css",
    "contentHash": "sha256:...",
    "preload": true
  }
}
```

**Pros:**
- ✅ Pure Web4 OOP
- ✅ CSS files are tracked objects with UUIDs
- ✅ Can sync, version, and reference CSS

**Cons:**
- ⚠️ Requires Unit component integration
- ⚠️ More complex setup

### Option D: ONCE CSSLoader (Inspired by Legacy ONCE)

Create a `CSSLoader` class that preloads CSS before component registration:

```typescript
// layer2/CSSLoader.ts
export class CSSLoader {
  private static cache: Map<string, CSSStyleSheet> = new Map();
  
  static async preload(cssPath: string): Promise<CSSStyleSheet> {
    if (this.cache.has(cssPath)) {
      return this.cache.get(cssPath)!;
    }
    
    const response = await fetch(cssPath);
    const cssText = await response.text();
    const sheet = new CSSStyleSheet();
    sheet.replaceSync(cssText);
    
    this.cache.set(cssPath, sheet);
    return sheet;
  }
  
  static async preloadAll(cssPaths: string[]): Promise<void> {
    await Promise.all(cssPaths.map(this.preload.bind(this)));
  }
}
```

**Usage in demo-lit.html:**
```javascript
// Preload CSS BEFORE importing components
await CSSLoader.preloadAll([
  '/dist/ts/layer5/views/css/themeBase.css',
  '/dist/ts/layer5/views/css/ItemView.css',
  '/dist/ts/layer5/views/css/OnceOverView.css'
]);

// NOW import components (CSS is ready)
import './dist/ts/layer5/views/OnceOverView.js';
```

**Pros:**
- ✅ Simple, no external dependencies
- ✅ CSS loaded before components
- ✅ Uses browser-native `CSSStyleSheet`
- ✅ Can be OOP class

**Cons:**
- ⚠️ Must manually list CSS files
- ⚠️ Components need to use adoptedStyleSheets

### Option E: UcpView + CSSLoader + Unit (Combined Solution) ⭐

**The Web4 Way:**

1. **Unit** tracks CSS files with UUIDs and metadata
2. **CSSLoader** (or `UcpCSSManager`) preloads CSS as `CSSStyleSheet`
3. **AbstractWebBean** uses `adoptedStyleSheets` from cache
4. **StaticFileRoute** serves CSS with correct MIME types

```
┌─────────────────────────────────────────────────────────────┐
│  Browser Boot Sequence                                       │
├─────────────────────────────────────────────────────────────┤
│  1. HTML loads                                               │
│  2. CSSLoader.preloadAll() fetches all CSS → CSSStyleSheet  │
│  3. Import Lit components (CSS already in cache)            │
│  4. Components use adoptedStyleSheets from cache            │
│  5. Render without FOUC                                      │
└─────────────────────────────────────────────────────────────┘
```

**UcpView Enhancement:**

```typescript
export abstract class UcpView<TModel = any> extends LitElement {
  
  // Each view declares its CSS path
  static cssPath: string = '';
  
  connectedCallback(): void {
    super.connectedCallback();
    
    // Apply preloaded CSS from cache
    const cssPath = (this.constructor as typeof UcpView).cssPath;
    if (cssPath) {
      const sheet = CSSLoader.get(cssPath);
      if (sheet) {
        this.shadowRoot!.adoptedStyleSheets = [sheet];
      }
    }
  }
}
```

## ✅ Recommended Solution: Option E

**Rationale:**
- Pure Web4 OOP (no bundler)
- CSS files are objects (Unit integration possible)
- Browser-native APIs (CSSStyleSheet, adoptedStyleSheets)
- Preloading prevents FOUC
- Compatible with Lit's shadow DOM
