## 📋 PLAN: Problem Statement

### Challenge
Lit views (DefaultItemView, OncePeerItemView) use decorators (`@property`, `@customElement`) that fail in the tsx test environment:
```
Error: Unsupported decorator location: field
```

### Current Coverage
- **Test05**: RelatedObjects with dummy classes ✅ (works in tsx)
- **Test21**: RelatedObjects with JsInterface (PersistenceManager, UcpStorage) ✅ (works in tsx)
- **Test02**: Browser-based Playwright test (validates demo page) ✅
- **Test07**: ViewComponentIntegration (partial Lit testing) ⚠️

### Goal
Verify RelatedObjects registry works with real Lit view classes:
- ItemView (interface/JsInterface)
- DefaultItemView (concrete Lit component)
- OncePeerItemView (extends DefaultItemView)
- Prototype chain registration for Lit inheritance

## 📝 Notes

### Why Lit Decorators Fail in tsx
The tsx loader uses esbuild which has limited decorator support. Lit's `@property` decorator expects the "standard" decorator proposal but tsx/esbuild uses "legacy" decorators.

### Future Work
- Consider migrating to Vitest with browser mode
- Or use Playwright component testing
- Or configure esbuild for standard decorators
