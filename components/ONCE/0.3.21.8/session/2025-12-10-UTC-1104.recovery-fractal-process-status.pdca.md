## **🏗️ CURRENT ONCE ARCHITECTURE SUMMARY**

**Based on detailed PDCAs analysis:**

### **Core Architecture Principles**

**1. Protocol-Less Communication**
- Web4 communicates by replicating object states (scenarios), not sending messages
- IOR-based method invocation: `/component/version/uuid/method`
- Scenario-driven: All state hibernated/replicated as scenarios

**2. Hierarchical Server Architecture**
- Primary server (port 42777) manages registry of client servers
- Client servers (ports 8080+) register with primary, relay browser messages
- Browser clients connect to any server, send/receive via WebSocket

**3. Radical OOP Layer Structure**
```
Layer 1: Infrastructure (OS-level abstractions)
Layer 2: Implementation (NodeJsOnce, HTTPServer, UcpStorage)
Layer 3: Interfaces (ONCE.interface.ts, models, JsInterface)
Layer 4: Utilities (TestFileParser, DefaultColors)
Layer 5: CLI & Views (Lit components, CLI commands)
```

### **Key Architectural Components**

**1. JsInterface Pattern (Phase G)**
- **Problem Solved**: TypeScript interfaces erased at runtime
- **Solution**: Abstract classes extending `JsInterface` base class
- **Current State**: JsInterface.ts + TypeDescriptor.ts implemented
- **Active Migration**: PersistenceManager → abstract class (G.4)
- **Reference**: [GitHub](https://github.com/Cerulean-Circle-GmbH/UpDown/blob/dev/web4v0100/components/ONCE/0.3.21.8/session/2025-12-08-UTC-1100.jsinterface-type-descriptors.pdca.md) | [§/components/ONCE/0.3.21.8/session/2025-12-08-UTC-1100.jsinterface-type-descriptors.pdca.md](./2025-12-08-UTC-1100.jsinterface-type-descriptors.pdca.md)

**2. Scenario-Unit Unification (Phase F.0)**
- **Problem Solved**: Two parallel persistence systems (Scenarios vs Units)
- **Solution**: Unified `Scenario<T>` with optional `unit?` field
- **Architecture**: ScenarioService as single point of truth
- **Storage**: Index-based (`scenarios/index/{uuid-folders}/`) with symlinks
- **Current State**: Core implementation complete, reference tracking deferred
- **Reference**: [GitHub](https://github.com/Cerulean-Circle-GmbH/UpDown/blob/dev/web4v0100/components/ONCE/0.3.21.8/session/2025-12-08-UTC-1000.scenario-unit-unification.pdca.md) | [§/components/ONCE/0.3.21.8/session/2025-12-08-UTC-1000.scenario-unit-unification.pdca.md](./2025-12-08-UTC-1000.scenario-unit-unification.pdca.md)

**3. PersistenceManager Architecture**
- **Interface**: `PersistenceManager` (abstract class extending JsInterface)
- **Implementations**:

### **Data Flow Architecture**

**Scenario Persistence Flow:**
```
Component → ScenarioService.scenarioSave()
  → PersistenceManager.scenarioSave()
    → UcpStorage (NodeJS) / BrowserScenarioStorage (Browser)
      → scenarios/index/{uuid-folders}/{uuid}.scenario.json
      → Symlinks: type/domain/capability/{uuid}.scenario.json
```

**View Rendering Flow:**
```
scenario.model (readonly)
  → ModelProxy (detects changes)
    → viewsUpdateAll() triggers rerender
      → Lit adapter reads scenario.model
        → LitElement.render() updates DOM
```

**Message Broadcasting Flow:**
```
Browser → Client Server (WebSocket)
  → Primary Server (WebSocket)
    → All Client Servers (broadcast)
      → All Browser Clients (WebSocket)
```

### **Key Design Patterns**

1. **JsInterface Pattern**: Runtime-existing interfaces via abstract classes
2. **Delegation Pattern**: Web4TSComponent methods delegated transparently
3. **Scenario-Driven**: All state as hibernatable scenarios
4. **ModelProxy Pattern**: Automatic view updates and peer sync
5. **RelatedObjects Registry**: Service locator for interface implementations
6. **Index-Based Storage**: UUID-indexed primary storage with symlink views
