## **🎯 PROBLEM** (The What)

### **Problem Statement**
ONCE v0.3.21.5 had comprehensive lifecycle tests (`lifecycle-management.test.ts`, `message-exchange.test.ts`) that validated server startup, shutdown, crash recovery, broadcasting, relay, and P2P communication. However, these tests used **functional anti-patterns**:

1. **HTTP Health Checks**: `curl` and `fetch()` calls for server status
2. **REST Endpoints**: `/shutdown-all`, `/discover-servers` (functional, not IOR-based)
3. **Protocol Knowledge**: Tests knew about HTTP/WebSocket implementation details
4. **Procedural Execution**: Linear test scripts, not hibernatable objects
5. **No Evidence Preservation**: Pass/fail only, no quality archaeology

After Radical OOP refactoring (Iteration 5, Phases 5-8), these tests are **incompatible** with the new architecture. We need to **recreate them completely** using **Tootsie** (Total Object-Oriented Testing Suite) - where **tests are living quality objects**.

### **Web4 Principle Violations in Old Tests**
- ❌ **Principle 18 (Black-Box Testing Only)**: Tests use HTTP/WebSocket protocol details
- ❌ **Principle 12 (IOR-based Method Invocation)**: Tests use REST endpoints instead of IOR
- ❌ **Principle 4 (TRUE Radical OOP)**: Tests are procedural scripts, not objects
- ❌ **Hibernation**: Tests can't be saved/restored with full context
- ❌ **Evidence-Based**: No quality archaeology, just pass/fail

### **Current State (0.3.21.5 Tests)**
```typescript
// FUNCTIONAL ANTI-PATTERN (0.3.21.5)
async function checkServerHealth(port: number): Promise<boolean> {
  try {
    const response = await fetch(`http://localhost:${port}/health`);
    return response.ok;  // WHITE-BOX: Knows about HTTP
  } catch {
    return false;
  }
}

test('should shutdown all servers', async () => {
  await execAsync('curl -X POST http://localhost:42777/shutdown-all');  // FUNCTIONAL!
});
```

### **Target State (Tootsie Radical OOP)**
```typescript

export class ServerLifecycleQualityOracle extends DefaultWeb4TestCase {
```

```typescript
// Inside Quality Oracle
protected async observeScenarioFile(serverName: string): Promise<Scenario<ONCEModel>> {
  const scenariosDir = path.join(this.model.testDataDir, 'scenarios/ONCE');
  
  // Find scenario by server name pattern
  const scenarioFiles = await fs.readdir(scenariosDir);
  const matchingFile = scenarioFiles.find(f => 
    f.includes(serverName) && f.endsWith('.scenario.json')
  );
  
  // Read and parse scenario (no HTTP!)
  const scenarioPath = path.join(scenariosDir, matchingFile);
  const scenarioContent = await fs.readFile(scenarioPath, 'utf-8');
  return JSON.parse(scenarioContent);
}
```

```typescript
class ServerStartupQualityOracle extends DefaultWeb4TestCase {
  protected async executeTestLogic(): Promise<any> {
    // 1. Record input
    this.recordEvidence('input', 'Server configuration', {
      port: this.scenario.testDataScenario.port,
      hostname: this.scenario.testDataScenario.hostname
    });
    
    // 2. Execute & record execution
    const startTime = Date.now();
    await this.invokeServerMethod('startServer');
    const startupDuration = Date.now() - startTime;
    
    this.recordEvidence('execution', 'Startup process', {
      duration: startupDuration,
      timestamp: new Date()
    });
    
    // 3. Observe outcome
    const serverScenario = await this.observeScenarioFile('primary');
    this.recordEvidence('output', 'Server scenario', serverScenario);
    
    // 4. Make judgment
    const isHealthy = serverScenario.model.primaryServer !== null;
    this.recordEvidence('assertion', 'Health assessment', {
      verdict: isHealthy ? 'GOOD' : 'POOR',
      confidence: 0.95
    });
    
    return { success: isHealthy, evidence: this.evidence };
```
