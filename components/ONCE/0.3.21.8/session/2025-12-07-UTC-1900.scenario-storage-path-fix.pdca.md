## Problem Analysis

The `UcpStorage` class is creating symlink directories (`type/`, `capability/`, `domain/`) at the wrong level.

### Current (WRONG) Structure:
```
test/data/
├── type/
│   └── TestComponent/      ❌ WRONG - should be under scenarios/
├── capability/
│   └── testCap/            ❌ WRONG - should be under scenarios/
├── domain/
│   └── test.local/         ❌ WRONG - should be under scenarios/
└── scenarios/
    └── index/
        └── {uuid}/         ✅ CORRECT
```

### Expected (CORRECT) Structure:
```
test/data/
└── scenarios/
    ├── index/
    │   └── {a}/{b}/{c}/{d}/{e}/
    │       └── {uuid}.scenario.json    ✅ Primary storage
    ├── type/
    │   └── TestComponent/
    │       └── {uuid}.scenario.json → symlink
    ├── capability/
    │   └── testCap/
    │       └── value/
    │           └── {uuid}.scenario.json → symlink
    └── domain/
        └── test.local/
            └── {uuid}.scenario.json → symlink
```

---

## Root Cause

In `UcpStorage.symlinkCreate()`, the symlink base path uses `projectRoot` directly instead of `projectRoot/scenarios/`:

```typescript
// CURRENT (WRONG):
const fullLinkPath = path.join(this.model.projectRoot, symlinkPath, `${uuid}.scenario.json`);

// SHOULD BE:
const fullLinkPath = path.join(this.model.projectRoot, 'scenarios', symlinkPath, `${uuid}.scenario.json`);
```
