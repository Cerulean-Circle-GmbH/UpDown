## **🏛️ ROUTE ARCHITECTURE: HTTP vs SPA (Two Distinct Systems)**

### **1. HTTP Routes (Server-Side) - layer2**

Server-side request handling for HTTP endpoints:

| File | Purpose |
|------|---------|
| `layer2/Route.ts` | Abstract base class for HTTP routes |
| `layer2/IORRoute.ts` | IOR method invocation (`/{component}/{version}/{uuid}/{method}`) |
| `layer2/HTMLRoute.ts` | HTML page serving with provider functions |
| `layer2/StaticFileRoute.ts` | Static files (JS, CSS, images) with MIME types |
| `layer3/RouteModel.interface.ts` | HTTP RouteModel: `{ pattern, method, priority, statistics }` |

**Used by:** `HTTPRouter` on server - routes incoming HTTP requests.

### **2. SPA Routes (Client-Side) - layer3**

Client-side navigation for Single Page Application:

| File | Purpose |
|------|---------|
| `layer3/Route.ts` | Concrete Route class for SPA navigation |
| `layer3/RouteScenario.interface.ts` | SPA RouteModel: `{ pattern, viewTag, title, isActive, params, ... }` |
| `layer5/views/UcpRouter.ts` | SPA Router view component |

**Used by:** `UcpRouter` in browser - manages client-side navigation.

### **Key Difference: Two RouteModel Interfaces!**

```
HTTP RouteModel (RouteModel.interface.ts):
  { uuid, name, pattern, method, priority, statistics }
  
SPA RouteModel (RouteScenario.interface.ts):
  { uuid, name, pattern, viewTag, title, serverRoute, currentPath, params, query, isActive, viewProps }
```

### **RouteItemView Uses SPA RouteModel**

`RouteItemView` is a **client-side view** for displaying navigation links.
- **Model:** SPA RouteModel from `RouteScenario.interface.ts`
- **NOT:** HTTP RouteModel from `RouteModel.interface.ts`

This means:
- ✅ Has: `pattern`, `viewTag`, `title`, `isActive`
- ❌ Does NOT have: `method` (GET/POST), `priority`, `statistics`

/**
 * Router.interface.ts - Router Component Interface
 * 
 * Web4 Principles:

import type { Route } from './Route.js';
import type { Reference } from './Reference.interface.js';
import type { RouterModel } from './RouterModel.interface.js';

/**
 * Router interface - SPA routing abstraction
 */
export interface Router {
  /** Router model */
  readonly model: RouterModel;
  
  /** Register a route */
  routeRegister(route: Route): void;
  
  /** Get route by pattern */
  routeLookup(pattern: string): Reference<Route>;
  
  /** Resolve path to matching route */
  routeResolve(path: string): Reference<Route>;
  
  /** Navigate to path */
  navigateTo(path: string, replace?: boolean): void;
  
  /** Get all registered routes */
  get routes(): Route[];
  
  /** Get current active route */
  get activeRoute(): Reference<Route>;
}

## **📝 PHASE R.2: UcpRouter IS the Implementation (TRON Decision)**

**TRON Decision (2025-12-11):** `UcpRouter` IS the Router implementation - no separate `DefaultRouter` needed.

**Current Architecture:**
- `UcpRouter` (layer5/views/) extends LitElement - it's a **view** that handles SPA routing
- It already implements all Router functionality: route registration, navigation, view switching
- No need for a separate layer2 "DefaultRouter" component

**HTTPRouter Relationship:**
- `HTTPRouter` (layer2) extends/implements Router for **server-side** HTTP request routing
- It uses HTTP Route classes (IORRoute, HTMLRoute, StaticFileRoute)
- Separate concern from UcpRouter (browser) vs HTTPRouter (server)

**Simplification:**
- ~~DefaultRouter.ts~~ → NOT NEEDED
- UcpRouter IS the SPA Router implementation
- HTTPRouter IS the HTTP Router implementation

**Future Migration (Naming Debt):**
- When extracting to `components/Router/`, UcpRouter → DefaultRouterView
- But for now, keep current naming to avoid breaking changes

```typescript
/**
 * DefaultRouter.ts - Default Router Implementation
 * 
 * Extends UcpComponent for MVC integration.
 * UcpController handles ModelProxy → view updates.
 * 
 * Web4 Principles:
 * - P4: Radical OOP (no arrow functions)
 * - P6: Empty Constructor + init()
 * - P16: nameVerb pattern
 * 
 * @component Router
 * @layer 2
 */

import { UcpComponent } from './UcpComponent.js';
import type { Router } from '../layer3/Router.interface.js';
import type { RouterModel } from '../layer3/RouterModel.interface.js';
import type { Reference } from '../layer3/Reference.interface.js';
import type { Scenario } from '../layer3/Scenario.interface.js';
import { Route } from '../layer3/Route.js';
import { randomUUID } from 'crypto';

export class DefaultRouter 
  extends UcpComponent<RouterModel> 
  implements Router {
  
  /** Route instances (not just patterns) */
  private routesMap: Map<string, Route> = new Map();
  
  /** Active route reference */
  private activeRouteRef: Reference<Route> = null;
  
  /**
   * Empty constructor - Web4 P6
   * UcpComponent creates UcpController
   */
  constructor() {
    super();  // Creates this.controller
  }
  
  /**
   * REQUIRED by UcpComponent: Provide default model
   * Matches RouterModel.interface.ts: routes, defaultRoute?, statistics
   */
  protected modelDefault(): RouterModel {
    return {
      uuid: randomUUID(),
      name: 'DefaultRouter',
      routes: [],
      statistics: {
        totalRequests: 0,
        successfulRequests: 0,
        failedRequests: 0,
        lastRequestTime: null
      }
    };
  }
  
  /**
   * Initialize from scenario
   * ✅ Web4 Pattern: super.init() handles model + controller init
   */
  async init(scenario?: Scenario<RouterModel>): Promise<this> {
    await super.init(scenario);  // Sets model, inits controller with ModelProxy
    return this;
  }
  
  // ═══════════════════════════════════════════════════════════════
  // Router Interface Implementation
  // ═══════════════════════════════════════════════════════════════
  
  routeRegister(route: Route): void {
    this.routesMap.set(route.pattern, route);
    // Model change triggers view update via ModelProxy
    // RouterModel.routes is RouteModel[], not string[]
    this.model!.routes = [...this.model!.routes, route.model];
  }
  
  routeLookup(pattern: string): Reference<Route> {
    return this.routesMap.get(pattern) || null;
  }
  
  routeResolve(path: string): Reference<Route> {
    // Try exact match first
    const exact = this.routesMap.get(path);
    if (exact) return exact;
    
    // Try pattern matching
    for (const route of this.routesMap.values()) {
      const params = route.pathMatch(path);
      if (params !== null) {
        return route;
      }
    }
    
    return null;
  }
  
  navigateTo(path: string, replace: boolean = false): void {
    if (replace) {
      window.history.replaceState({}, '', path);
    } else {
      window.history.pushState({}, '', path);
    }
    
    // Model change via proxy triggers view updates automatically!
    this.model!.currentPath = path;
    
    // Deactivate previous
    if (this.activeRouteRef) {
      this.activeRouteRef.deactivate();
    }
    
    // Activate new
    const route = this.routeResolve(path);
    if (route) {
      const params = route.pathMatch(path);
      route.activate(path, params || {}, {});
      this.activeRouteRef = route;
    }
  }
  
  get routes(): Route[] {
    return Array.from(this.routesMap.values());
  }
  
  get activeRoute(): Reference<Route> {
    return this.activeRouteRef;
  }
}
```

### **UcpComponent Integration Benefits**

| Feature | How It Works |
|---------|--------------|
| ModelProxy | Controller wraps model; property changes auto-notify views |
| View Updates | `this.model!.currentPath = x` → Proxy set trap → `viewsUpdateAll()` |
| View Registration | `router.viewRegister(routerView)` → view gets proxied model |
| Scenario Receive | `router.scenarioReceive({model})` → merges into proxy → updates views |
| Hibernate | `router.hibernate()` → saves to Storage via `toScenario()` |
| RelatedObjects | `controller.relatedObjectRegister(Router, this)` for lookup |

## **📝 PHASE R.3: RouteItemView Component**

**Key Pattern:** Views extend `UcpView<TModel>` (NOT UcpComponent!)
- Views BORROW the model from UcpController (don't own it)
- Model is set via `view.model = controller.modelProxy`
- `refresh()` is called when model changes

```typescript
/**
 * RouteItemView.ts - Route Item View Component (SPA Routes)
 * 
 * ⚠️ IMPORTANT: Uses SPA RouteModel from RouteScenario.interface.ts
 *    NOT HTTP RouteModel from RouteModel.interface.ts!
 * 
 * SPA RouteModel has: pattern, viewTag, title, isActive, params, query
 * HTTP RouteModel has: pattern, method, priority, statistics (DIFFERENT!)
 * 
 * Hierarchy:
 *   LitElement ← UcpView<T> ← DefaultItemView<T> ← RouteItemView
 * 
 * Web4 Principles:
 * - P4: Radical OOP (use method references, not arrow functions)
 * - P16: nameVerb pattern + TypeScript accessors
 * - P19: CSS in separate file
 * - P27: Web Components ARE Radical OOP
 * 
 * @component Router
 * @layer 5
 */

import { html, TemplateResult } from 'lit';
import { customElement, property } from 'lit/decorators.js';
import { DefaultItemView } from './DefaultItemView.js';
// ⚠️ Use SPA RouteModel, NOT HTTP RouteModel!
import type { RouteModel } from '../../layer3/RouteScenario.interface.js';
import type { Reference } from '../../layer3/Reference.interface.js';
import type { Router } from '../../layer3/Router.interface.js';

/**
 * RouteItemView - Displays a single SPA route as a clickable link
 * 
 * Uses SPA RouteModel directly - NO separate RouteItemModel needed!
 * 
 * SPA RouteModel fields:
 *   - pattern: URL pattern (e.g., '/', '/demo', '/peer/:uuid')
 *   - viewTag: Custom element tag (e.g., 'once-over-view')
 *   - title: Page title for display
 *   - isActive: Currently active?
 *   - params, query: URL parameters
 *   - viewProps: Additional view properties
 * 
 * Usage:
 *   <route-item-view .model=${spaRouteModel}></route-item-view>
 */
@customElement('route-item-view')
export class RouteItemView extends DefaultItemView<RouteModel> {
  
  static cssPath = 'RouteItemView.css';
  
  /** Router reference for navigation */
  @property({ attribute: false }) router: Reference<Router> = null;
  
  // ═══════════════════════════════════════════════════════════════
  // COMPUTED ACCESSORS (Web4 P16) - derive view values from SPA RouteModel
  // ═══════════════════════════════════════════════════════════════
  
  /** Route pattern as display path */
  get path(): string {
    return this.model?.pattern ?? '';
  }
  
  /** Route title from model (or derived from pattern) */
  get routeTitle(): string {
    // Use model.title if available, otherwise derive from pattern
    if (this.model?.title) return this.model.title;
    // "/" → "Main", "/demo" → "Demo"
    if (this.path === '/') return 'Main';
    return this.path.slice(1).charAt(0).toUpperCase() + this.path.slice(2);
  }
  
  /** View tag for display (e.g., 'once-over-view') */
  get viewTag(): string {
    return this.model?.viewTag ?? '';
  }
  
  /** Is route currently active? */
  get isActive(): boolean {
    return this.model?.isActive ?? false;
  }
  
  // ═══════════════════════════════════════════════════════════════
  // NAVIGATION (Web4 P4: Object method references, not arrow functions)
  // ═══════════════════════════════════════════════════════════════
  
  /**
   * Handle route link click - SPA navigation
   * ✅ Web4 P4: Regular method, not arrow function
   */
  private linkClickHandle(event: Event): void {
    event.preventDefault();
    if (this.router) {
      this.router.navigateTo(this.path, false);
    } else {
      // Fallback: dispatch popstate for UcpRouter to handle
      window.history.pushState({}, '', this.path);
      window.dispatchEvent(new PopStateEvent('popstate'));
    }
  }
  
  /**
   * Get bound click handler
   * ✅ Web4 P4: Returns bound method reference
   */
  private linkClickHandlerGet(): (event: Event) => void {
    return this.linkClickHandle.bind(this);
  }
  
  // ═══════════════════════════════════════════════════════════════
  // RENDER
  // ═══════════════════════════════════════════════════════════════
  
  render(): TemplateResult {
    const activeClass = this.isActive ? 'active' : '';
    
    return html`
      <div class="route-item ${activeClass}">
        <a href="${this.path}" 
           @click=${this.linkClickHandlerGet()}>
          <strong>${this.routeTitle}</strong>
          <code>${this.path}</code>
        </a>
      </div>
    `;
  }
}
```

### **View vs Component: Key Distinction**

| Aspect | UcpComponent | UcpView |

| Has Controller | ✅ Yes | ❌ No |

| ModelProxy | Creates via controller | Receives from controller |
| Purpose | Business logic + state | UI rendering |
