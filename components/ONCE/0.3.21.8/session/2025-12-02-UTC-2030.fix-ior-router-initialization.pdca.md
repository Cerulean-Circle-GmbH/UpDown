## **📊 SUMMARY**

### **Problem Statement**
When starting ONCE 0.3.21.8 server, the following error occurs:
```
[HTTPRouter] Routing error: Error: IORRoute: IORMethodRouter not initialized with ONCE kernel
```

This blocks all tests because the server cannot serve the `/demo` route or any IOR-based requests.

### **Root Cause Hypothesis**
The `IORMethodRouter` requires the ONCE kernel to be passed during initialization. In 0.3.21.8, this initialization is either:
1. Not happening
2. Happening in wrong order
3. Happening with wrong parameters

### **Comparison Point**
- **0.3.21.5:** Server works, `/demo` serves correctly
- **0.3.21.8:** Server broken, IORMethodRouter not initialized

## **🔧 DO**

### **Step 1: Find the Error Source** ✅

```bash
grep -r "IORMethodRouter not initialized" components/ONCE/0.3.21.8/
```

Found in `IORRoute.ts` line 68-70:
```typescript
if (!this.iorRouter.model.uuid) {
    throw new Error('IORRoute: IORMethodRouter not initialized with ONCE kernel');
}
```

### **Step 2: Root Cause Analysis** ✅

In `ServerHierarchyManager.ts` line 187:
```typescript
this.iorRouter.init(undefined, this.component);
```

The `init()` method expects a `scenario.model` to set the UUID, but `undefined` was passed.
Since `scenario` is `undefined`, the UUID stays empty `''`, triggering the error.

### **Step 3: Apply Fix** ✅

**Web4 Principle 6:** `init()` is responsible for complete initialization - not patching afterwards!

Fixed in `IORMethodRouter.ts` - added idProvider parameter and UUID generation inside init():
```typescript
public init(
    scenario: Scenario<any> | undefined, 
    kernel: any,
    idProvider: IDProvider = new UUIDProvider()
): this {
    if (scenario?.model) {
        Object.assign(this.model, scenario.model);
    }
    // ✅ Web4 Principle 6: init() generates UUID if not provided
    if (!this.model.uuid) {
        this.model.uuid = idProvider.create();
    }
    this.onceKernel = kernel;
    return this;
}
```

Updated `ServerHierarchyManager.ts` to pass idProvider:
```typescript
this.iorRouter.init(undefined, this.component, this.idProvider);
```
