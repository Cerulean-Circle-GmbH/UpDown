## **🎯 PROBLEM** (The What)

### **Problem Statement**
During implementation of ONCE lifecycle tests with Modern Tootsie (Phase 8.4), multiple **Web4 principle violations** were introduced in `ONCETestCase.ts` and potentially in the 15 test specializations.

### **Critical Violations Identified**

#### **Violation 1: CommonJS `__filename` Pattern** 🚨
**Location:** `ONCETestCase.ts` lines 276, 350  
**Code:**
```typescript
const currentFile = __filename;  // ❌ CommonJS!
const filename = path.basename(__filename);  // ❌ CommonJS!
```

**Web4 Principle Violated:** Pure ESM architecture (from Iteration 1.16)  
**Reference:** [Iteration 1.16 Test Suite Restoration PDCA](../../0.3.21.2/session/2025-11-21-UTC-1030.fix-tests-with-isolation.pdca.md)  
**Quote from PDCA:**
```
- **Critical Issue 1**: 2x `__filename` CommonJS violations in `DefaultONCE.ts` (Web4 principle violation)
- ❌ Removed: `const __filename = url.pathname;` (CommonJS naming)
- Zero `__filename` usages in codebase ✅ required
```

**Impact:** 
- Tests won't work in ESM environment
- Breaks Web4's pure ESM architecture
- Violates established codebase standard (670+ uses all properly use `fileURLToPath(import.meta.url)`)

#### **Violation 2: CommonJS `require()` Dynamic Imports** 🚨
**Location:** `ONCETestCase.ts` lines 272-273, 346-347  
**Code:**
```typescript
const fs = require('fs');       // ❌ CommonJS!
const path = require('path');   // ❌ CommonJS!
```

**Web4 Principle Violated:** Pure ESM architecture  
**Correct Pattern (from 670+ codebase examples):**
```typescript
import { fileURLToPath } from 'url';
import { dirname, basename } from 'path';
import { readFileSync, writeFileSync, unlinkSync } from 'fs';
```

**Impact:**
- Dynamic require() breaks ESM module graph
- Cannot be tree-shaken or optimized
- Inconsistent with entire codebase pattern

#### **Violation 3: Hardcoded Absolute Path in Constructor** 🚨
**Location:** `ONCETestCase.ts` line 31  
**Code:**
```typescript
constructor() {
  super();
  this.projectRoot = '/Users/Shared/Workspaces/2cuGitHub/UpDown';  // ❌ Hardcoded!
  this.testDataRoot = join(this.projectRoot, 'test/data');
}
```

**Web4 Principle Violated:** Path Authority pattern  
**Reference:** Web4TSComponent README, DefaultCLI implementation  
**Correct Pattern:**
- Paths come from scenario via `init(scenario)`
- Or auto-detect from `import.meta.url`
- Never hardcode absolute paths

**Impact:**
- Tests only work on one machine
- Cannot be used in CI/CD
- Violates test portability principle

#### **Violation 4: Missing ESM Import for `import.meta.url`** 🚨
**Location:** `ONCETestCase.ts` top of file  
**Missing:**
```typescript
import { fileURLToPath } from 'url';
import { dirname, basename } from 'path';
```

**Impact:**
- Cannot use correct ESM pattern for file path detection
- Forces use of CommonJS anti-patterns

#### **Violation 5: Functional Getter Pattern in `compareScenarios()`** ⚠️
**Location:** `ONCETestCase.ts` lines 136-138  
**Code:**
```typescript
const getField = (obj: any, path: string): any => {  // ❌ Functional!
  return path.split('.').reduce((acc, part) => acc?.[part], obj);
};
```

**Web4 Principle Violated:** No functional helpers (Radical OOP)  
**Correct Pattern:** Method on a class, or use direct property access

#### **Violation 6: Missing Constructor in Test Specializations** ⚠️
**Location:** All Test01-Test15 files  
**Missing:** Each test should have an empty constructor per Web4 Principle 4

**Affected Files:**
- Test01_PathAuthorityAndProjectRootDetection.ts
- Test02_ComponentDescriptorValidation.ts
- ... (all 15 tests)
