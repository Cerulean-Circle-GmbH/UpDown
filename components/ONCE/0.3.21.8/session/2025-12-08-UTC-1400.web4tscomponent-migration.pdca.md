## 🔴 PROBLEM STATEMENT

### Current State
1. `Web4TSComponent/0.3.20.6` is a separate component
2. Has its own copies of interfaces (`Scenario`, `Model`, `Reference`, etc.)
3. Implements `Component` interface, not `UcpComponent`
4. Contains 6500+ lines of functionality needed in ONCE
5. Unit creation after build is in Web4TSComponent, not accessible to ONCE

### Target State
1. Web4TSComponent functionality integrated into ONCE
2. `DefaultWeb4TSComponent` extends `UcpComponent`
3. Uses ONCE's existing layer3 interfaces (no duplication)
4. Can create units for files after build
5. Structured for easy extraction as standalone component in 0.4.0.0
