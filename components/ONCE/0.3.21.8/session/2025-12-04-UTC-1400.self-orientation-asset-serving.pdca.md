## 📋 Complete Analysis

### The Real Problem: URL Resolution

**demo-lit.html is served at URL:** `/demo-lit`

**Inside demo-lit.html:**
```javascript
import { CSSLoader } from '../../dist/ts/layer2/CSSLoader.js';
```

**Browser resolves this relative to URL `/demo-lit`:**
- `/demo-lit` + `../../dist/...` → `/dist/ts/layer2/CSSLoader.js`

**But StaticFileRoute only matches:**
- `/{Component}/{version}/**` pattern

**So `/dist/ts/layer2/CSSLoader.js` doesn't match StaticFileRoute!**

It falls through to IORRoute which:
1. Returns `application/json` (wrong MIME type)
2. Browser fails to load JS module

### Data Flow Verification

```
✅ getAssetManifest() returns:
   css: ['/ONCE/0.3.21.8/src/ts/layer5/views/css/ItemView.css', ...]

✅ CSSLoader.preloadAll() fetches these paths
   (These DO match StaticFileRoute: /{Component}/{version}/**)

✅ CSSLoader stores by filename:
   'ItemView.css' → CSSStyleSheet

✅ OnceOverView.ts has:
   static cssPath = 'OnceOverView.css'

✅ UcpView.applyCachedStyles() calls:
   CSSLoader.get('OnceOverView.css')  → Works!

❌ BUT demo-lit.html imports FAIL FIRST:
   /dist/ts/layer2/CSSLoader.js → IORRoute → application/json → CRASH
```

### Root Cause

The demo-lit.html page cannot load because:

1. `import { CSSLoader } from '../../dist/ts/layer2/CSSLoader.js'`
2. Browser resolves to: `/dist/ts/layer2/CSSLoader.js`
3. StaticFileRoute doesn't match (no `/{Component}/{version}/` prefix)
4. IORRoute matches and returns JSON
5. Browser throws: "Failed to load module script: expected JavaScript MIME type"

### Three Possible Fixes

#### Option A: Change demo-lit.html to use absolute paths

```javascript
// Before (broken):
import { CSSLoader } from '../../dist/ts/layer2/CSSLoader.js';

// After (working):
import { CSSLoader } from '/ONCE/0.3.21.8/dist/ts/layer2/CSSLoader.js';
```

**Problem**: Version hardcoded, breaks on version bump.

#### Option B: Add `<base href>` to demo-lit.html

HTMLRoute injects base href dynamically:

```html
<base href="/ONCE/0.3.21.8/">
```

Then relative paths resolve correctly.

**Problem**: Needs dynamic injection from server.

#### Option C: Extend StaticFileRoute to also match `/dist/**` and `/src/**`

Make StaticFileRoute match both patterns:
- `/{Component}/{version}/**` → existing
- `/(dist|src)/**` → NEW (maps to current component)

**Best option**: Component context is known, serves from current component.

## 🔵 Recommended Fix: Option C

Extend StaticFileRoute to match:

1. `/{Component}/{version}/(src|dist)/**` → existing behavior
2. `/(src|dist)/**` → NEW: serve from **current** component

This way:
- `/demo-lit` page imports `../../dist/ts/...` → `/dist/ts/...`
- StaticFileRoute matches `/dist/ts/...`
- Serves from `{componentRoot}/dist/ts/...`

### Implementation Plan

```typescript
// StaticFileRoute.matches()

// Pattern 1: /{Component}/{version}/**
if (this.matchesComponentPrefixed(urlPath)) return true;

// Pattern 2 (NEW): /(src|dist)/** without component prefix
if (this.matchesCurrentComponent(urlPath)) return true;
```
