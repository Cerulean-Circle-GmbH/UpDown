## 🔴 PROBLEM STATEMENT

### Current Architecture (Problematic)

```
┌─────────────────────────────────────────────────────────┐
│                    CURRENT (COUPLED)                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  LitElement  ←── UcpView ←── DefaultItemView            │
│      ↑              ↑              ↑                    │
│   (Lit lib)    (Web4 class)   (Lit component)          │
│                                                         │
│  ❌ UcpView EXTENDS LitElement                          │
│  ❌ View/ItemView are interfaces (not JsInterface)      │
│  ❌ Framework lock-in (Lit required for all views)      │
│  ❌ Cannot implement View as JsInterface (single inherit)│
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Problems

1. **Framework Lock-in**: UcpView extends LitElement → all views tied to Lit
2. **JsInterface Impossible**: TypeScript single inheritance blocks View as JsInterface
3. **Testing Issues**: Lit decorators fail in tsx environment
4. **Portability Lost**: Can't use React, Vue, or vanilla JS for views
5. **Web4 Principle Violation**: Views should be framework-independent

## 🟢 TARGET ARCHITECTURE

### Delegation Pattern (Decoupled)

```
┌─────────────────────────────────────────────────────────┐
│                    TARGET (DECOUPLED)                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  LAYER 3 (Interfaces - JsInterface)                     │
│  ┌─────────────┐                                        │
│  │ JsInterface │                                        │
│  └──────┬──────┘                                        │
│         │                                               │
│  ┌──────▼──────┐    ┌───────────────┐                  │
│  │    View     │───▶│   ItemView    │                  │
│  │ (abstract)  │    │  (abstract)   │                  │
│  └──────┬──────┘    └───────┬───────┘                  │
│         │                   │                          │
│  LAYER 5 (Web4 Implementations)                        │
│  ┌──────▼──────┐    ┌───────▼───────┐                  │
│  │  UcpView    │───▶│DefaultItemView│                  │
│  │(no LitElement)│  │ (Web4 class)  │                  │
│  └──────┬──────┘    └───────┬───────┘                  │
│         │                   │                          │
│  LAYER 5 (Framework Adapters)                          │
│  ┌──────▼──────────────────▼───────┐                   │
│  │        LitViewAdapter           │                   │
│  │   (extends LitElement)          │                   │
│  │   - delegates to UcpView        │                   │
│  │   - renders via Lit templates   │                   │
│  └─────────────────────────────────┘                   │
│                                                         │
│  Future: ReactViewAdapter, VueViewAdapter, etc.         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Class Relationships

```typescript
// ═══════════════════════════════════════════════════════════════
// LAYER 3: JsInterface hierarchy (Abstract classes)
// ═══════════════════════════════════════════════════════════════

abstract class AbstractView<TModel> extends JsInterface {
  protected modelField: TModel | null = null;
  protected childViews: AbstractView<unknown>[] = [];
  
  abstract get model(): TModel;
  abstract set model(m: TModel);
  abstract refresh(): void;
  abstract render(): ViewRenderData;  // Returns data, not template!
}

abstract class AbstractItemView<T extends ItemViewModel> extends AbstractView<T> {
  // Enforced attributes (mapped to model)
  abstract get name(): string;
  abstract set name(v: string);
  abstract get description(): string;
  abstract set description(v: string);
  abstract get icon(): string;
  abstract set icon(v: string);
  abstract get badge(): Reference<string | number>;
  abstract set badge(v: Reference<string | number>);
}

// ═══════════════════════════════════════════════════════════════
// LAYER 5: Lit Adapter (delegates to Abstract)
// ═══════════════════════════════════════════════════════════════

@customElement('web4-item-view')
class LitItemView extends LitElement {
  // The Web4 view (injected or created)
  private delegate: AbstractItemView<ItemViewModel>;
  
  @property({ type: Object }) model: ItemViewModel;
  
  constructor() {
    super();
    // Default: create vanilla implementation
    this.delegate = new DefaultItemView();
  }
  
  // Inject custom delegate (for OncePeerItemView, etc.)
  delegateSet(delegate: AbstractItemView<ItemViewModel>): void {
    this.delegate = delegate;
  }
  
  willUpdate(changed: PropertyValues) {
    if (changed.has('model')) {
      this.delegate.model = this.model;
    }
  }
  
  render(): TemplateResult {
    // Get data from Web4 view
    const data = this.delegate.render();
    
    // Convert to Lit template
    return html`
      <div class="item-view">
        <i class="fa ${data.attributes.icon}"></i>
        <span class="name">${data.attributes.name}</span>
        <span class="description">${data.attributes.description}</span>
        ${data.attributes.badge ? html`<span class="badge">${data.attributes.badge}</span>` : nothing}
      </div>
    `;
  }
}

// ═══════════════════════════════════════════════════════════════
// LAYER 5: Concrete Web4 Implementation (business logic)
// ═══════════════════════════════════════════════════════════════

class OncePeerItemView extends AbstractItemView<OncePeerItemViewModel> {
  
  get name(): string { return this.modelField?.name ?? 'ONCE Peer'; }
  set name(v: string) { if (this.modelField) this.modelField.name = v; }
  
  get description(): string { return this.modelField?.connectionStatus ?? ''; }
  set description(v: string) { /* read-only for peer */ }
  
  get icon(): string { 
    return this.modelField?.isConnected ? 'fa-server' : 'fa-server-slash'; 
  }
  set icon(v: string) { /* computed from state */ }
  
  get badge(): Reference<string | number> { 
    return this.modelField?.clientCount ?? null; 
  }
  set badge(v: Reference<string | number>) { /* computed */ }
  
  refresh(): void {
    for (const child of this.childViews) {
      child.refresh();
    }
  }
  
  /**
   * Returns ViewRenderData - framework-agnostic!
   * Both LitItemView and DefaultItemView consume this.
   */
  render(): ViewRenderData {
    return {
      type: 'once-peer-item-view',
      attributes: {
        name: this.name,
        description: this.description,
        icon: this.icon,
        badge: this.badge,
        uuid: this.modelField?.uuid,
        port: this.modelField?.port
      },
      children: [],
      events: {
        'click': 'handlePeerClick',
        'contextmenu': 'handlePeerMenu'
      }
    };
  }
  
  handlePeerClick(): void {
    console.log(`Peer clicked: ${this.name}`);
    // Navigate to peer details
  }
  
  handlePeerMenu(): void {
    console.log(`Peer context menu: ${this.name}`);
    // Show action menu
  }
}

// ═══════════════════════════════════════════════════════════════
// LAYER 5: Vanilla Adapter (ONCE model-based template)
// PARALLEL to LitItemView - not inheritance!
// ═══════════════════════════════════════════════════════════════

class DefaultItemView extends HTMLElement {
  // The Web4 view delegate (injected)
  private delegate: AbstractItemView<ItemViewModel>;
  private container: HTMLElement;
  
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
    this.container = document.createElement('div');
    this.shadowRoot!.appendChild(this.container);
  }
  
  // Inject the Web4 delegate
  delegateSet(delegate: AbstractItemView<ItemViewModel>): void {
    this.delegate = delegate;
    this.renderToDOM();
  }
  
  connectedCallback(): void {
    if (this.delegate) {
      this.renderToDOM();
    }
  }
  
  /**
   * Render ViewRenderData to vanilla DOM
   * Uses ONCE's model-based template engine
   */
  private renderToDOM(): void {
    const data = this.delegate.render();
    
    // Convert ViewRenderData to DOM (ONCE template engine)
    this.container.innerHTML = `
      <div class="item-view">
        <i class="fa ${data.attributes.icon}"></i>
        <span class="name">${data.attributes.name}</span>
        <span class="description">${data.attributes.description}</span>
        ${data.attributes.badge ? `<span class="badge">${data.attributes.badge}</span>` : ''}
      </div>
    `;
    
    // Bind events to delegate methods
    this.container.querySelector('.item-view')?.addEventListener('click', () => {
      if (data.events?.click && typeof (this.delegate as any)[data.events.click] === 'function') {
        (this.delegate as any)[data.events.click]();
      }
    });
  }
}

// Register vanilla custom element
customElements.define('default-item-view', DefaultItemView);

// ═══════════════════════════════════════════════════════════════
// USAGE: Same Web4 view, different renderers
// ═══════════════════════════════════════════════════════════════

// Create the Web4 business logic view
const peerView = new OncePeerItemView();
peerView.model = { name: 'Primary', uuid: '...', port: 42777, isConnected: true };

// Option A: Render with Lit
@customElement('web4-once-peer-item-view')
class LitOncePeerItemView extends LitItemView {
  constructor() {
    super();
    this.delegateSet(peerView);  // Same peerView!
  }
}

// Option B: Render with Vanilla (DefaultItemView)
const vanillaView = document.createElement('default-item-view') as DefaultItemView;
vanillaView.delegateSet(peerView);  // Same peerView!
document.body.appendChild(vanillaView);

// Both render the same data, just different DOM strategies!
```

### Naming Pattern

| Layer | Pattern | Example | Description |
|-------|---------|---------|-------------|
| **Abstract** | `Abstract{X}View` | `AbstractItemView` | Web4 JsInterface class (business logic) |
| **Concrete** | `Once{X}View` | `OncePeerItemView` | ONCE-specific implementation |
| **Lit Adapter** | `Lit{X}View` | `LitItemView` | LitElement wrapper (parallel) |
| **Vanilla Adapter** | `Default{X}View` | `DefaultItemView` | DOM wrapper using ONCE template (parallel) |

### Key Insight: Parallel Adapters

```
AbstractItemView (Web4 business logic)
       │
       ├──── LitItemView (Lit rendering) ──────┐
       │                                       │ PARALLEL
       └──── DefaultItemView (Vanilla DOM) ────┘ ALTERNATIVES
```

Both adapters:
- Receive the same `AbstractItemView` delegate
- Render to DOM using different frameworks
- Handle events differently (Lit vs vanilla)
- Are interchangeable in the browser

## 📝 DESIGN DECISIONS

### D1: ViewRenderData vs Template Strings

**Question**: What should UcpView.render() return?

**Options**:
1. **Data object** (`ViewRenderData`) - framework-agnostic
2. **Template string** - portable but limited
3. **Virtual DOM** - heavy, reinvents React

**Decision**: **Data object (ViewRenderData)**

```typescript
interface ViewRenderData {
  type: string;
  attributes: Record<string, unknown>;
  children: ViewRenderData[];
  events?: Record<string, string>; // event name → method name
}
```

Rationale:
- Framework-agnostic
- Adapters convert to Lit html``, React JSX, etc.
- Testable without browser
- Serializable (can send over WebSocket for SSR)

### D2: CSS Loading Location

**Question**: Where should CSS loading happen?

**Options**:
1. **UcpView** - current location
2. **LitViewAdapter** - framework-specific
3. **CSSLoader singleton** - separate concern

**Decision**: **LitViewAdapter** (framework-specific)

Rationale:
- UcpView becomes framework-independent
- Lit uses `adoptedStyleSheets`, React uses CSS modules, etc.
- Each adapter handles its framework's CSS mechanism

### D3: Event Handling

**Question**: How do Web4 views handle events?

**Decision**: **ViewRenderData.events** + Adapter binding

```typescript
// Web4 view defines events
class DefaultItemView extends ItemView {
  templateRender(): ViewRenderData {
    return {
      type: 'item',
      events: {
        'click': 'handleClick',     // method name on this view
        'contextmenu': 'handleMenu'
      },
      // ...
    };
  }
  
  handleClick(): void {
    // Action invocation
  }
}

// Lit adapter binds them
class LitItemViewAdapter extends LitElement {
  render() {
    const data = this.delegate.render();
    return html`
      <div @click=${() => this.delegate.handleClick()}>
        ...
      </div>
    `;
  }
}
```
