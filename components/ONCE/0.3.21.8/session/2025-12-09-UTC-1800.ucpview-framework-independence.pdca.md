### Problems

1. **Framework Lock-in**: UcpView extends LitElement вҶ’ all views tied to Lit
2. **JsInterface Impossible**: TypeScript single inheritance blocks View as JsInterface
3. **Testing Issues**: Lit decorators fail in tsx environment
4. **Portability Lost**: Can't use React, Vue, or vanilla JS for views
5. **Web4 Principle Violation**: Views should be framework-independent

### Class Relationships

```typescript
// LAYER 3: JsInterface hierarchy
abstract class View extends JsInterface {
  abstract get model(): TModel;
  abstract set model(m: TModel);
  abstract refresh(): void;
  abstract render(): unknown;
}

abstract class ItemView<T extends ItemViewModel> extends View<T> {
  abstract get name(): string;
  abstract set name(v: string);
  // ... enforced attributes
}

// LAYER 5: Framework-independent implementations
class UcpView<T> extends View<T> {
  protected modelField: T | null = null;
  protected childViews: UcpView[] = [];
  
  // Web4 behavior (no Lit dependency!)
  refresh(): void { /* notify children */ }
  render(): unknown { return this.templateRender(); }
  protected abstract templateRender(): unknown;
}

class DefaultItemView<T extends ItemViewModel> extends ItemView<T> {
  // Implements ItemView attributes
  get name(): string { return this.modelField?.name ?? ''; }
  set name(v: string) { if (this.modelField) this.modelField.name = v; }
  
  protected templateRender(): unknown {
    // Returns data structure (not Lit template!)
    return {
      type: 'item-view',
      name: this.name,
      description: this.description,
      icon: this.icon,
      badge: this.badge
    };
  }
}

// LAYER 5: Lit Adapter (framework-specific)
@customElement('lit-item-view')
class LitItemViewAdapter extends LitElement {
  private viewDelegate: DefaultItemView;
  
  constructor() {
    super();
    this.viewDelegate = new DefaultItemView();
  }
  
  @property() model: ItemViewModel;
  
  willUpdate(changed: PropertyValues) {
    if (changed.has('model')) {
      this.viewDelegate.model = this.model;
    }
  }
  
  render(): TemplateResult {
    const data = this.viewDelegate.render();
    return html`
      <div class="item-view">
        <i class="fa ${data.icon}"></i>
        <span class="name">${data.name}</span>
        <span class="description">${data.description}</span>
        ${data.badge ? html`<span class="badge">${data.badge}</span>` : nothing}
      </div>
    `;
  }
}
```

## рҹ“қ DESIGN DECISIONS

### D1: ViewRenderData vs Template Strings

**Question**: What should UcpView.render() return?

**Options**:
1. **Data object** (`ViewRenderData`) - framework-agnostic
2. **Template string** - portable but limited
3. **Virtual DOM** - heavy, reinvents React

**Decision**: **Data object (ViewRenderData)**

```typescript
interface ViewRenderData {
  type: string;
  attributes: Record<string, unknown>;
  children: ViewRenderData[];
  events?: Record<string, string>; // event name вҶ’ method name
}
```

Rationale:
- Framework-agnostic
- Adapters convert to Lit html``, React JSX, etc.
- Testable without browser
- Serializable (can send over WebSocket for SSR)

### D2: CSS Loading Location

**Question**: Where should CSS loading happen?

**Options**:
1. **UcpView** - current location
2. **LitViewAdapter** - framework-specific
3. **CSSLoader singleton** - separate concern

**Decision**: **LitViewAdapter** (framework-specific)

Rationale:
- UcpView becomes framework-independent
- Lit uses `adoptedStyleSheets`, React uses CSS modules, etc.
- Each adapter handles its framework's CSS mechanism

### D3: Event Handling

**Question**: How do Web4 views handle events?

**Decision**: **ViewRenderData.events** + Adapter binding

```typescript
// Web4 view defines events
class DefaultItemView extends ItemView {
  templateRender(): ViewRenderData {
    return {
      type: 'item',
      events: {
        'click': 'handleClick',     // method name on this view
        'contextmenu': 'handleMenu'
      },
      // ...
    };
  }
  
  handleClick(): void {
    // Action invocation
  }
}

// Lit adapter binds them
class LitItemViewAdapter extends LitElement {
  render() {
    const data = this.delegate.render();
    return html`
      <div @click=${() => this.delegate.handleClick()}>
        ...
      </div>
    `;
  }
}
```
