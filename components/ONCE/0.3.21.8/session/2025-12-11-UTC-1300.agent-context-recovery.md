## **🏛️ ROUTE ARCHITECTURE: HTTP vs SPA (Critical Understanding!)**

### **⚠️ Two Distinct Route Systems with Different RouteModel Interfaces!**

| Aspect | HTTP Routes (Server) | SPA Routes (Client) |
|--------|---------------------|---------------------|
| **Base Class** | `layer2/Route.ts` (abstract) | `layer3/Route.ts` (concrete) |
| **Model File** | `RouteModel.interface.ts` | `RouteScenario.interface.ts` |
| **Model Fields** | `pattern, method, priority, statistics` | `pattern, viewTag, title, isActive, params...` |
| **Used By** | HTTPRouter (server) | UcpRouter (browser) |
| **Purpose** | Handle HTTP requests | SPA navigation |

### **HTTP Routes (layer2) - Server-Side Request Handling**

| File | Purpose |
|------|---------|
| `layer2/Route.ts` | Abstract base with `matches()`, `handleRequest()` |
| `layer2/IORRoute.ts` | IOR pattern: `/{component}/{version}/{uuid}/{method}` |
| `layer2/HTMLRoute.ts` | HTML pages via provider function |
| `layer2/StaticFileRoute.ts` | Static files (JS, CSS) with MIME types |

### **SPA Routes (layer3) - Client-Side Navigation**

| File | Purpose |
|------|---------|
| `layer3/Route.ts` | Concrete class with `pathMatch()`, `activate()` |
| `layer3/RouteScenario.interface.ts` | SPA RouteModel definition |
| `layer5/views/UcpRouter.ts` | SPA Router view component |

### **RouteItemView Uses SPA RouteModel!**

`RouteItemView` displays navigation links → uses **SPA** RouteModel:
- ✅ Import: `import type { RouteModel } from '../../layer3/RouteScenario.interface.js'`
- ✅ Has: `pattern`, `viewTag`, `title`, `isActive`
- ❌ NOT: `method`, `priority`, `statistics` (those are HTTP-only)

## **🔗 VIEW COMPOSITION PATTERN: add() + viewTag**

### **UcpView.add() - Child View Composition**

```typescript
// In UcpView.ts - Base pattern for adding child views
add(childView: View<any>): void {
  if (childView instanceof UcpView) {
    this.childViews.push(childView);
  }
  // Subclass decides where to append in DOM
}
```

### **Tag-to-Component Mapping (viewTag)**

Routes store a `viewTag` string that maps to a custom element tag:

```typescript
// Registration (stores tag name):
router.routeRegister('/', 'once-peer-default-view', { title: 'Server Status' });
router.routeRegister('/demo', 'once-over-view', { title: 'Demo' });

// Creation (uses tag name):
const view = document.createElement(route.viewTag);  // 'once-peer-default-view'
```

### **Key Pattern: Adding to Overview = Registering with Router**

```
router.routeRegister(pattern, viewTag)  →  routes.set(pattern, route)
                     ↓
router.routesGet()  →  Map<pattern, Route>
                     ↓
<route-over-view>   →  iterates routes, creates <route-item-view> for each
                     ↓
<route-item-view>   →  displays route as clickable link
```

**The overview gets its content from registered routes - no separate "add to overview" needed!**

## **🔴 WEB4 PRINCIPLES (CRITICAL - CHECK BEFORE EVERY COMMIT)**

**Read deeply:** [§/components/ONCE/0.3.21.8/session/web4-principles-checklist.md](./web4-principles-checklist.md)

**Most Violated (watch for these):**

| # | Principle | Common Violation | Correct Fix |
|---|-----------|------------------|-------------|
| P4a | Radical OOP - Method Refs | Arrow `() => {}` or `function(){}` | `this.method.bind(this)` |
| P4b | Radical OOP - Polymorphism | `if/switch` on type | `object.methodGet()` - ask the object! |
| P7 | Async Only Layer 4 | `async` in Layer 5 views | Use `.then()` or delegate to Layer 4 |
| P16 | nameVerb + Accessors | `getXyz()` | Use `get xyz()` accessor |
| P26 | No Factories | `createXyz()` | Use `new X().init()` |
| P27 | Web Components = OOP | `${this.sectionRender()}` | `<my-component>` declarative tag |
| JsI | No Type Enums | `enum RouteType {...}` | `obj.constructor.name` - object IS type |

**⚠️ P4a: Object Method References:**
```typescript
// ❌ WRONG: Arrow function or anonymous function
items.forEach((item) => { this.itemProcess(item); });

// ✅ CORRECT: Method reference (Radical OOP)
items.forEach(this.itemProcess.bind(this));
```

**⚠️ P4b: Polymorphism (ASK THE OBJECT, DON'T SWITCH!):**
```typescript
// ❌ WRONG: External switch/if on type
get icon(): string {
  if (className.includes('Static')) return '📁';  // BAD!
}

// ❌ WRONG: Lookup table (same anti-pattern!)
const icons = { 'StaticFileRoute': '📁' };
return icons[className];

// ✅ CORRECT: Each class defines its own behavior
class StaticFileRoute extends Route {
  public iconGet(): string { return '📁'; }
}
// Usage: route.iconGet() - polymorphism!
```

## **🔷 WEB4 PRINCIPLE: JsInterface Pattern**

**PDCA Reference:** [§/session/2025-12-12-UTC-1103.http-routes-display.pdca.md](./2025-12-12-UTC-1103.http-routes-display.pdca.md)

**Core Concept:** In JavaScript/TypeScript, **the object IS its type**. No redundant enums or type fields needed - use the class hierarchy itself.

### **Pattern 1: No Type Enums**

```typescript
// ❌ WRONG: Redundant enum duplicates class type information
enum RouteType { SPA, STATIC, IOR, HTML, SCENARIO }

interface RouteModel {
  type: RouteType;  // Redundant! The class already tells us the type!
}

if (route.type === RouteType.STATIC) { ... }  // Double maintenance!

// ✅ CORRECT: Class hierarchy IS the type system
route.constructor.name           // "StaticFileRoute"
route instanceof StaticFileRoute // true

// Type check without enum:
if (route instanceof StaticFileRoute) { ... }
```

### **Pattern 2: Serialize Type as className**

When serializing objects (e.g., to JSON for HTTP response), include `className`:

```typescript
// Server serializes with className
private routeModelExtract(route: Route): object {
  return {
    pattern: route.model.pattern,
    className: route.constructor.name,  // JsInterface: serialize type
    icon: route.iconGet(),              // Polymorphism: ask object
    label: route.labelGet()             // Polymorphism: ask object
  };
}
```

### **Pattern 3: Runtime Type Access**

```typescript
// Get type name
const typeName = object.constructor.name;

// Type check
if (object instanceof SpecificClass) { ... }

// Get all implemented interfaces (future: via TypeRegistry)
const interfaces = TypeRegistry.interfacesOf(object);
```

### **Why JsInterface?**

| Approach | Problem |
|----------|---------|
| Enums | Duplicate information, manual sync, no polymorphism |
| Type fields | Same issues, plus storage overhead |
| **JsInterface** | Single source of truth (the class), enables polymorphism |

## **🔑 UcpComponent / UcpController / ModelProxy Pattern**

**This is the CORE Web4 MVC architecture. Every component must understand this.**

### **Architecture Diagram**

```
┌─────────────────────────────────────────────────────────────────────┐
│                        UcpComponent<TModel>                          │
│                        (Layer 2 - abstract base)                     │
├─────────────────────────────────────────────────────────────────────┤
│  model: Reference<TModel>                                            │
│  controller: UcpController<TModel>                                   │
│  storage: Reference<Storage>                                         │
├─────────────────────────────────────────────────────────────────────┤
│  init(scenario?: { model?: TModel }): Promise<this>                  │
│    → this.model = scenario?.model || this.modelDefault()            │
│    → this.controller.init(this.model)  ← creates ModelProxy         │
│                                                                      │
│  abstract modelDefault(): TModel  ← MUST implement                   │
│                                                                      │
│  viewRegister(view) → controller.viewRegister(view)                 │
│  scenarioReceive(s) → controller.scenarioReceive(s)                 │
│  viewsUpdateAll()   → controller.viewsUpdateAll()                   │
└─────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      UcpController<TModel>                           │
│                      (Layer 2 - MVC controller)                      │
├─────────────────────────────────────────────────────────────────────┤
│  modelProxy: TModel (JavaScript Proxy)                               │
│  registeredViews: Set<View<TModel>>                                 │
│  relatedObjectsRegistry: Map<InterfaceKey, Set<instance>>           │
├─────────────────────────────────────────────────────────────────────┤
│  init(model: TModel): this                                           │
│    → this.modelProxy = new Proxy(model, { set: ... })               │
│    → set trap: target[prop] = value; this.viewsUpdateAll();        │
│                                                                      │
│  viewRegister(view): void                                            │
│    → view.model = this.modelProxy  ← view gets proxied model        │
│                                                                      │
│  scenarioReceive(scenario): void                                     │
│    → Object.assign(this.modelProxy, scenario.model)                 │
│    → triggers Proxy set trap → viewsUpdateAll() SYNC!               │
│                                                                      │
│  viewsUpdateAll(): void                                              │
│    → this.registeredViews.forEach(this.viewRefresh.bind(this))      │
└─────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        UcpView<TModel>                               │
│                        (Layer 5 - extends LitElement)                │
├─────────────────────────────────────────────────────────────────────┤
│  modelRef: Reference<TModel>                                         │
│  set model(m) { this.modelRef = m; this.requestUpdate(); }          │
│  get model() { return this.modelRef; }                               │
│  refresh(): void { this.requestUpdate(); }                           │
│                                                                      │
│  NOTE: UcpView does NOT extend UcpComponent!                         │
│        It borrows the model from UcpController via viewRegister()   │
└─────────────────────────────────────────────────────────────────────┘
```

### **Key Flow: WebSocket → View Update**

```
Layer 4 (async):  WebSocket.onmessage(data)
       ↓
Layer 2 (sync):   controller.scenarioReceive({ model: data })
       ↓
                  Object.assign(modelProxy, scenario.model)
       ↓
                  Proxy set trap triggers viewsUpdateAll()
       ↓
Layer 5 (sync):   each view.refresh() → requestUpdate() → Lit re-render
```

### **Pattern Usage**

**Creating a new component:**
```typescript
// Layer 2 - Implementation
export class DefaultRouter extends UcpComponent<RouterModel> {
  
  constructor() {
    super();  // Creates UcpController
  }
  
  // REQUIRED: Provide default model
  protected modelDefault(): RouterModel {
    return { uuid: crypto.randomUUID(), name: 'Router', ... };
  }
  
  // Optional: Override init for custom initialization
  async init(scenario?: Scenario<RouterModel>): Promise<this> {
    await super.init(scenario);  // Sets model, inits controller
    // Custom init logic here
    return this;
  }
}
```

**Creating a view for the component:**
```typescript
// Layer 5 - View (does NOT extend UcpComponent!)
export class DefaultRouterView extends UcpView<RouterModel> {
  // Model is set by controller.viewRegister()
  // refresh() called automatically on model changes
  
  render(): TemplateResult {
    return html`<div>${this.model?.name}</div>`;
  }
}
```

## **🎯 RADICAL OOP: JSINTERFACE + POLYMORPHISM PATTERNS**

**PDCA Reference:** [§/session/2025-12-12-UTC-1103.http-routes-display.pdca.md](./2025-12-12-UTC-1103.http-routes-display.pdca.md)

### **Pattern 1: No Enums for Type Classification (JsInterface)**

```typescript
// ❌ WRONG: Redundant enum (duplicates class type!)
enum RouteType { SPA, STATIC, IOR... }
if (route.type === RouteType.STATIC) { ... }

// ✅ RIGHT: Object IS its type (JsInterface)
route.constructor.name           // "StaticFileRoute"
route instanceof StaticFileRoute // true
```

**The class hierarchy IS the type system. No enum needed!**

### **Pattern 2: Polymorphism Over Conditionals**

```typescript
// ❌ WRONG: External switch/if on type
get routeIcon(): string {
  if (className.includes('Static')) return '📁';  // BAD!
  if (className.includes('IOR')) return '🔗';     // BAD!
}

// ✅ RIGHT: Each class defines its own behavior
class StaticFileRoute extends Route {
  public iconGet(): string { return '📁'; }
}
class IORRoute extends Route {
  public iconGet(): string { return '🔗'; }
}

// Usage: ASK THE OBJECT
const icon = route.iconGet();  // Polymorphism!
```

### **Pattern 3: Declarative Web Components (P27)**

```typescript
// ❌ WRONG: Programmatic render method calls
render() {
  return html`
    ${this.routesSectionRender()}      // Method call - BAD!
    ${this.httpRoutesSectionRender()}  // Method call - BAD!
  `;
}

// ✅ RIGHT: Declarative tags (Web Components ARE OOP)
render() {
  return html`
    <route-over-view .router=${this.router}></route-over-view>
  `;
}
```

### **Summary: Three Core Radical OOP Patterns**

| Pattern | Wrong | Right |
|---------|-------|-------|
| **Type Classification** | `enum RouteType { ... }` | `route.constructor.name` |
| **Type-Based Behavior** | `if (type === X) icon = ...` | `route.iconGet()` |
| **View Composition** | `${this.sectionRender()}` | `<my-component>` |
