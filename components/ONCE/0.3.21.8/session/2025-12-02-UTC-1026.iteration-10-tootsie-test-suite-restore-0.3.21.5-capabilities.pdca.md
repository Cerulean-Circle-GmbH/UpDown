### **Problem Statement**
ONCE v0.3.21.8 has successfully completed architectural refactoring (Iteration 5: HTTPServer/Router, Generic Component Loading) but lacks the comprehensive operational lifecycle test coverage that ONCE v0.3.21.5 possessed. Without a robust test suite:
- Cannot verify server lifecycle (start/stop/restart)
- Cannot validate component loading and initialization
- Cannot test multi-peer communication and message exchange
- Cannot verify scenario hibernation and persistence
- **CRITICAL**: Cannot confidently deploy or upgrade ONCE

**Historical Context**: ONCE 0.3.21.5 had lifecycle tests, but they were:
- Protocol-dependent (HTTP REST endpoints, WebSocket protocol knowledge)
- Functional architecture (not Radical OOP)
- NOT black-box compliant (violated Web4 Principle 18)

**Current State**: 
- ONCE 0.3.21.8 has Modern Tootsie available (from Web4Articles)
- `ONCETestCase.ts` exists but has Web4 Principle 21 violations (functional imports)
- No operational lifecycle tests implemented yet
- Quality Oracle not integrated

### **Why This Matters**
**"Testing is not just verification - it's the consciousness of quality itself."** - Tootsie Philosophy

Without comprehensive tests:
- ❌ Regressions go undetected
- ❌ Refactoring becomes risky
- ❌ Confidence in deployment is low
- ❌ CMM level drops (cannot achieve CMM4 without systematic testing)

### **Web4 Principles at Stake**
- **Principle 18: Black-Box Testing Only** - Tests must use IOR/CLI, not HTTP/WS protocols
- **Principle 21: Wrap Functional Imports** - Tests must use Radical OOP patterns, not functional Node built-ins
- **Principle 4: Radical OOP Discipline** - Tests are objects, not scripts

### **Sub-Iteration 10.1: Fix ONCETestCase Web4 Compliance**
**Duration**: 2-3 hours  
**Status**: ✅ **COMPLETE** (2025-12-02)
**Focus**: Replace functional imports with `import * as` pattern (Web4 Principle 21 compliance)

**Problem**: Current `ONCETestCase.ts` violates Web4 Principle 21:
```typescript
// ❌ CURRENT (Functional pollution)
import { readFileSync, writeFileSync, unlinkSync } from 'fs';
import { join, dirname, basename } from 'path';
import { fileURLToPath } from 'url';

// Creates context-less functions: readFileSync(), join(), dirname()
```

**Solution**: Use `import * as` pattern for object namespaces:
```typescript
// ✅ WEB4 COMPLIANT (Radical OOP)
import * as fs from 'fs';
import * as path from 'path';
import { fileURLToPath } from 'url'; // OK - url is special for ESM

// Creates objects with methods: fs.readFileSync(), path.join(), path.dirname()
```

```typescript
/**
 * Test 01: Path Authority and Project Root Detection
 * Verifies ONCE can correctly detect project root and test isolation environment
 * 
 * ✅ VERSION-AGNOSTIC: Uses import.meta.url to detect current test version
 */

import { ONCETestCase } from './ONCETestCase.js';
import { TestScenario } from '../../../../Web4Test/0.3.20.6/src/ts/layer3/TestScenario.js';
import * as path from 'path';
import { fileURLToPath } from 'url';

export class Test01_PathAuthorityAndProjectRootDetection extends ONCETestCase {
  constructor() { super(); }
  
  /**
   * Get ONCE version dynamically from test file location
   * ✅ VERSION-AGNOSTIC: Works in ANY version (0.3.21.8, 0.3.21.9, etc.)
   */
  private getONCEVersion(): string {
    // From: /components/ONCE/X.Y.Z.W/test/lifecycle-management/Test01.ts
    const testFile = fileURLToPath(import.meta.url);
    const testDir = path.dirname(testFile);           // test/lifecycle-management
    const testRoot = path.dirname(testDir);           // test
    const versionDir = path.dirname(testRoot);        // X.Y.Z.W
    const version = path.basename(versionDir);        // "0.3.21.8" or "0.3.21.9"
    return version;
  }
  
  async executeTestLogic(): Promise<void> {
    // ✅ Dynamic version detection
    const onceVersion = this.getONCEVersion();
    
    // 1. Arrange - Create test scenario
    const scenario = this.createTest01Scenario();
    this.init(scenario);
    
    // 2. Act - Invoke ONCE info command (CLI - black-box)
    const result = await this.invokeONCEMethod('info');
    
    // 3. Assert - Validate path detection (VERSION-AGNOSTIC)
    expect(result.projectRoot).toContain('/UpDown');
    expect(result.componentRoot).toContain('/components/ONCE'); // ✅ No hardcoded version!
    expect(result.componentRoot).toContain(onceVersion);        // ✅ Dynamic version!
    expect(result.isTestIsolation).toBe(true);
    
    // 4. Evidence - Record all 4 types
    this.logEvidence('input', 'Test scenario initialized', { scenario, onceVersion });
    this.logEvidence('step', 'Invoked ONCE info command', { method: 'info' });
    this.logEvidence('output', 'Path detection result', { result });
    this.logEvidence('assertion', 'Path validation', {
      projectRootValid: result.projectRoot.includes('/UpDown'),
      componentRootValid: result.componentRoot.includes('/ONCE'),
      versionMatches: result.componentRoot.includes(onceVersion),
      testIsolationDetected: result.isTestIsolation === true
    });
  }
  
  private createTest01Scenario(): TestScenario {
    return {
      uuid: 'test:uuid:once-lifecycle-01-path-authority',
      name: 'Test 01: Path Authority and Project Root Detection',
      testDataScenario: {
        description: 'Verify ONCE correctly detects paths and test isolation',
        expectedResults: {
          projectRootPattern: '/UpDown',
          componentRootPattern: '/ONCE',  // ✅ No version!
          testIsolation: true
        }
      }
    };
  }
}
```
