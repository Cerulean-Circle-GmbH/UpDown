### **Test Isolation & Path Authority (Learning Update)**

#### **Path Authority Pattern**
The component auto-detects execution context by checking if current path contains `/test/data`:

```typescript
// DefaultCLI.ts - calculateProjectRootInternal()
if (cwd.includes('/test/data')) {
  // Return path UP TO AND INCLUDING test/data
  return parts.slice(0, testIndex + 2).join('/');
}
```

**Production vs Test Isolation:**

| Context | Command | projectRoot | isTestIsolation |
|---------|---------|-------------|-----------------|
| Production | `once info` | `/Users/.../UpDown` | `false` |
| Test Shell | `once test shell` → `once info` | `...ONCE/0.3.21.8/test/data` | `true` |

#### **Test Shell Mechanics**

`once test shell` (delegated to `web4tscomponent.testShell()`):

1. **Setup**: Runs baseline test to populate `test/data/` with components
2. **Environment**: Sources component's `source.env` with test isolation PATH override
3. **PS1 Prompt**: Sets special prompt `[TEST ISOLATION ONCE 0.3.21.8]`
4. **Interactive Shell**: Launches bash with custom init file
5. **PATH Isolation**: `test/data/scripts` ONLY (production paths removed)

**Key Implementation** (`DefaultWeb4TSComponent.testShell()`):
```typescript
// Replace PATH (not append) to prevent production pollution
SYSTEM_PATH="/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin"
export PATH="${testDataDir}/scripts:${testDataDir}/node_modules/.bin:$SYSTEM_PATH"

// Launch bash with custom init
execSync(`bash --init-file "${wrapperPath}"`, {
  cwd: testDataDir,
  env: { PROJECT_ROOT: testDataDir, IS_TEST_ISOLATION: 'true' }
});
```

**Why This Matters for Phase 8.4:**
- **Black-box tests run in test shell** → automatic isolation
- **CLI commands work identically** → production vs test determined by PATH Authority
- **No mocking needed** → real components, real scenarios, isolated filesystem
- **Interactive debugging** → `once test shell` lets us manually verify test scenarios
