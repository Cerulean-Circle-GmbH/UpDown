## P - PLAN

### Problem Statement

The demo-lit page currently uses `setInterval` polling to update peer display. This violates:
- **P11: Protocol-Less Communication** - Should use raw scenario replication
- **Performance** - Polling is inefficient vs push notifications
- **Reactivity** - Views should auto-update when model changes

### Target

Replace polling with WebSocket-based scenario push:
1. Server broadcasts peer scenarios when state changes
2. Browser receives scenarios via WebSocket
3. ModelProxy detects changes and notifies views
4. Views auto-render via Lit's reactive system

### Web4 Principles Applied

| Principle | Application |
|-----------|-------------|
| **P4** | No arrow functions in WebSocket handlers |
| **P7** | Layer 2 async (WebSocket), Layer 5 sync (view update) |
| **P11** | Raw scenario JSON over WebSocket (no protocol messages) |
| **P16** | TypeScript accessors for model properties |
