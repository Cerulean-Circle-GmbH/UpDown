## 🎯 PLAN

### Problem Statement

The ONCE component has its own scenario storage implementation that evolved independently from the Unit component's storage architecture. For v0.3.21.x we need to:

1. **Integrate Unit into ONCE** - temporarily unify Unit functionality into ONCE
2. **Unify scenario storage** - use Unit's index-based architecture
3. **Enable scenario sync** - tracked synchronization across locations (NodeJS ↔ BrowserOnce IndexedDB)
4. **Prepare for v0.4.0.0** - structure allows later extraction back to standalone Unit component

### Current State Analysis

#### Current Scenario Storage (ONCE 0.3.21.8)
```
scenarios/
├── {Component}/{version}/{uuid}.scenario.json           # Flat by component
├── box/{domain}/{Component}/{version}/{uuid}.scenario.json  # By domain
└── box/{domain}/{Component}/{version}/capability/       # Capability links
    └── httpPort/{port}/{uuid}.scenario.json
```

#### Unit's Index-Based Architecture (Unit 0.3.0.5)
```
scenarios/
├── index/{a}/{b}/{c}/{d}/{e}/{uuid}.scenario.json      # PRIMARY: UUID-indexed
├── type/{Component}/{version}/*.scenario.json          # SYMLINK: By type
├── domain/{domain}/{Component}/{version}/*.json        # SYMLINK: By domain  
└── ontology/{name}.scenario.json                       # SYMLINK: Named concepts
```

### Target Architecture

```
scenarios/
├── index/                          # PRIMARY STORAGE (units live here)
│   └── {a}/{b}/{c}/{d}/{e}/       # 5-level UUID folder structure
│       ├── {uuid}.scenario.json   # Actual scenario file
│       └── {uuid}.unit            # Optional: Unit metadata file
│
├── type/                           # SYMLINKS by component type
│   └── {Component}/{version}/
│       └── {uuid}.scenario.json → ../../index/{a}/{b}/{c}/{d}/{e}/{uuid}.scenario.json
│
├── domain/                         # SYMLINKS by domain/machine
│   └── {domain}/{Component}/{version}/
│       └── {uuid}.scenario.json → ../../../index/{a}/{b}/{c}/{d}/{e}/{uuid}.scenario.json
│
├── capability/                     # SYMLINKS by capability
│   └── httpPort/{port}/
│       └── {uuid}.scenario.json → ../../index/{a}/{b}/{c}/{d}/{e}/{uuid}.scenario.json
│
└── ontology/                       # SYMLINKS by named concept
    └── {name}.scenario.json → ../index/{a}/{b}/{c}/{d}/{e}/{uuid}.scenario.json
```

### BrowserOnce IndexedDB Structure

```typescript
// Same logical structure in IndexedDB
indexedDB: {
  'scenarios-index': {       // Object store - primary storage
    key: uuid,
    value: ScenarioWithUnit
  },
  'scenarios-type': {        // Object store - type index
    key: `${component}/${version}/${uuid}`,
    value: uuid  // Reference to scenarios-index
  },
  'scenarios-domain': {      // Object store - domain index  
    key: `${domain}/${component}/${version}/${uuid}`,
    value: uuid  // Reference to scenarios-index
  },
  'scenarios-capability': {  // Object store - capability index
    key: `${capability}/${value}/${uuid}`,
    value: uuid
  }
}
```

### Key Principles

1. **Unit = Atomic Element**: Every scenario IS a unit
2. **Index = Source of Truth**: `scenarios/index/` contains actual data
3. **Symlinks = Views**: type/domain/capability are just indexed views
4. **Sync Tracking**: ONCE tracks which units are synced where
5. **Web4 Compliance**: All principles apply (P1-P27)

## 🏗️ Architecture Diagrams

### PersistenceManager Interface Hierarchy

```
┌─────────────────────────────────────────────────────────────┐
│                    PersistenceManager                       │
│  scenarioSave | scenarioLoad | scenarioFind                │
│  scenarioDelete | scenarioExists                           │
└─────────────────────────────────────────────────────────────┘
                            ▲
                            │ extends
┌─────────────────────────────────────────────────────────────┐
│                        Storage                              │
│  + init(scenario) | + toScenario()                         │
└─────────────────────────────────────────────────────────────┘
                            ▲
           ┌────────────────┴────────────────┐
           │                                  │
┌──────────┴──────────┐          ┌───────────┴───────────┐
│    UcpStorage       │          │  BrowserScenarioStorage│
│  (NodeJS/filesystem)│          │      (IndexedDB)       │
└─────────────────────┘          └────────────────────────┘
```

### RelatedObjects Registration Flow

```typescript
// In NodeJsOnce.init()
import { PersistenceManager } from '../layer3/PersistenceManager.interface.js';

const storage = new UcpStorage().init({...});
this.controller.relatedObjectRegister(PersistenceManager, storage);

// Lookup from anywhere in the component
const pm = controller.relatedObjectLookup(PersistenceManager);
await pm.scenarioSave(uuid, scenario, symlinkPaths);
```

## 🔄 Integration Points

### UcpComponent Changes

```typescript
// Before
export abstract class UcpComponent<TModel extends Model> {
  protected model: TModel;
  async toScenario(): Promise<Scenario<TModel>>;
}

// After  
export abstract class UcpComponent<TModel extends UnitModel> {
  protected model: TModel;  // Now includes indexPath, symlinkPaths
  protected storage: UcpStorage;
  
  async toScenario(): Promise<Scenario<TModel>>;
  async toUnit(): Promise<Unit>;
  async hibernate(): Promise<void>;      // Save to index
  async restore(uuid: string): Promise<this>;  // Load from index
}
```

### Scenario Interface Changes

```typescript
// Before
interface Scenario<T extends Model> {
  ior: IOR;
  owner: string;
  model: T;
}

// After (same but model is UnitModel)
interface Scenario<T extends UnitModel> {
  ior: IOR;
  owner: string;
  model: T;  // Includes: indexPath, symlinkPaths, syncState
}
```
