**1.2 Protocol-Less Scenario Live Updates (Web4 Principle 11)**

> ⚠️ **CRITICAL**: WebSocket messages like `server-registered` and `server-stopped` are **PROTOCOL MESSAGES** - this violates Web4 Principle 11!

**The Wrong Way (0.3.21.5 - Protocol Messages):**

```typescript
// ❌ WRONG: Protocol-based message types
ws.onmessage = (event) => {
    const message = JSON.parse(event.data);
    switch (message.type) {
        case 'server-registered':  // ❌ Protocol type!
            handleServerRegistered(message.data);
            break;
    }
};
```

**The Web4 Way (Scenario Replication):**

```typescript
// ✅ RIGHT: Receive scenarios, replicate objects
ws.onmessage = (event) => {
    const scenario = JSON.parse(event.data) as Scenario;
    
    // Scenario IS the peer - just replicate it!
    const existingPeer = kernel.model.peers.find(p => p.ior?.uuid === scenario.ior?.uuid);
    
    if (existingPeer) {
        // Update existing clone's state
        Object.assign(existingPeer, scenario);
    } else {
        // New peer - add clone to registry
        kernel.model.peers.push(scenario);
    }
    
    updatePeerDisplay();
};
```

**How It Works:**
1. Client server calls `toScenario()` to hibernate itself
2. Sends raw scenario over WebSocket (no protocol wrapper!)
3. Primary receives scenario, creates replica (clone) with same UUID
4. Browser receives scenario, updates local replica
5. **Objects communicate by state replication, not messages!**

**Reference:** [§/components/ONCE/0.3.21.2/session/2025-11-21-UTC-1900.iteration-01.6-once-architecture-consolidation.pdca.md#L1728](../../../0.3.21.2/session/2025-11-21-UTC-1900.iteration-01.6-once-architecture-consolidation.pdca.md#L1728) (Section 12: Protocol-Less Communication)

**2.2.1 AbstractWebBean Base Class (layer5/views/AbstractWebBean.ts)**

Leverages Unit component to track and load external CSS/HTML files:

```typescript
import { LitElement, CSSResult, TemplateResult, html, unsafeCSS } from 'lit';
import { Unit } from '../../layer3/Unit.interface.js';

/**
 * AbstractWebBean - Base class for all Web4 Lit web components
 * 
 * Manages loading of external CSS and HTML template files.
 * Uses Unit component to track file references with UUIDs.
 * 
 * Web4 Principles:
 * - P19: One File One Type (CSS/HTML/TS separate)
 * - P6: Empty constructor, init via scenario
 * - P8: DRY - shared base class
 */
export abstract class AbstractWebBean extends LitElement {
  protected cssUnit: Unit | null = null;
  protected templateUnit: Unit | null = null;
  
  /** Load CSS from external file via Unit */
  protected async cssLoad(cssPath: string): Promise<CSSResult> {
    const cssContent = await this.unitLoad(cssPath);
    return unsafeCSS(cssContent);
  }
  
  /** Load HTML template from external file via Unit */
  protected async templateLoad(templatePath: string): Promise<string> {
    return this.unitLoad(templatePath);
  }
  
  /** Generic file loader via Unit component */
  private async unitLoad(filePath: string): Promise<string> {
    const response = await fetch(filePath);
    return response.text();
  }
  
  /** Abstract: Subclass must provide view name for file discovery */
  abstract get viewName(): string;
  
  /** Construct paths based on view name */
  protected get cssPath(): string {
    return `../../../view/css/${this.viewName}.css`;
  }
  
  protected get templatePath(): string {
    return `../../../view/webBeans/${this.viewName}.html`;
  }
}
```

import { customElement, property } from 'lit/decorators.js';
import { html, TemplateResult } from 'lit';

import { View } from '../../layer3/View.interface.js';

import { Scenario } from '../../layer3/Scenario.interface.js';

/**
 * ItemView - Compact list/tree item view
 * 
 * @ior ior:esm:/ONCE/{version}/ItemView
 */
@customElement('once-item-view')
export class ItemView extends AbstractWebBean implements View {
  @property({ type: Object }) scenario: Scenario<any> | null = null;
  @property({ type: Boolean }) touchEnabled = true;
  
  get viewName(): string { return 'item-view'; }
  
  // CSS loaded from view/css/item-view.css (via AbstractWebBean)
  // Template structure defined in view/webBeans/item-view.html
  
  render(): TemplateResult {
    // Template delegates to external HTML file pattern
    // Implementation uses lit-html for reactive binding
    return this.renderFromModel();
  }
  
  private renderFromModel(): TemplateResult {
    const model = this.scenario?.model || {};
    const state = model.state?.state || 'unknown';
    const uuid = this.scenario?.ior?.uuid?.substring(0, 8) || '???';
    const port = model.state?.capabilities?.find(
      (c: any) => c.capability === 'httpPort'
    )?.port || 'N/A';
    
    return html`
      <div class="status-dot ${state.toLowerCase()}"></div>
      <div class="item-info">
        <div class="item-title">Peer ${uuid}</div>
        <div class="item-subtitle">Port ${port} · ${state}</div>
      </div>
      <div class="action-zone">
        ${this.actionsRender()}
      </div>
    `;
  }
  
  private actionsRender(): TemplateResult {
    const actions = this.actionsDiscover();
    return html`${actions.map(action => html`
      <button @click=${() => this.actionInvoke(action)}>
        ${action.icon} ${action.label}
      </button>
    `)}`;
  }
  
  actionsDiscover(): Action[] {
    return [{ 
      method: 'peerStop', 
      label: 'Stop', 
      icon: '🛑', 
      style: ActionStyle.DANGER,  // ✅ ENUM not string!
      shortcut: null,
      confirmRequired: null,
      confirmMessage: null
    }];
  }
  
  // ✅ TypeScript setter - NOT modelConnect()!
  set model(model: any) {
    this.scenario = { model } as Scenario<any>;
    this.requestUpdate();
  }
  
  get model(): any {
    return this.scenario?.model;
  }
  
  private async actionInvoke(action: Action): Promise<void> {
    const ior = this.scenario?.ior;
    if (!ior) return;
    // IOR-based method invocation
  }
}

**ModelProxy - Default Controller (layer2/ModelProxy.ts):**

> ⚠️ **CRITICAL Web4 Principles for MVC:**
> 
> **Principle 7: Async Only in Layer 4**
> - ModelProxy lives in Layer 2 (synchronous domain logic)
> - `scenarioReceive()` is called from Layer 4 (async WebSocket handler)
> - `viewsNotify()` and `view.update()` are **SYNCHRONOUS** (Layer 5)
> 
> **Flow:**
> ```
> Layer 4 (async)     Layer 2 (sync)        Layer 5 (sync)
> ─────────────────   ─────────────────     ─────────────────
> ws.onmessage()  →   scenarioReceive()  →  view.update()
>     (async)              (sync)              (sync)
> ```

```typescript
import { Reference } from './Reference.interface.js';

/**
 * ModelProxy - Wraps model to detect changes and notify views
 * 
 * This is the DEFAULT CONTROLLER in Web4 MVC:
 * - Wraps model with JavaScript Proxy
 * - Detects property changes
 * - Notifies registered views to update() **SYNCHRONOUSLY**
 * 
 * Web4 Principle 7: Layer 5 interactions are SYNCHRONOUS
 * - scenarioReceive() is called from Layer 4 (async WebSocket)
 * - viewsNotify() triggers SYNCHRONOUS view.update() calls
 * 
 * Alternative: Lit's reactive properties handle this automatically
 */
export class ModelProxy<TModel extends object> {
  private views: Set<View<TModel>> = new Set();
  private proxy: TModel;
  
  constructor(model: TModel) {
    this.proxy = this.proxyCreate(model);
  }
  
  private proxyCreate(model: TModel): TModel {
    return new Proxy(model, {
      set: (target, property, value) => {
        (target as any)[property] = value;
        this.viewsNotify();  // ✅ SYNCHRONOUS - Layer 5!
        return true;
      }
    });
  }
  
  /**
   * Receive scenario from WebSocket (called from Layer 4)
   * Updates model which triggers SYNCHRONOUS view updates
   */
  scenarioReceive(scenario: Scenario<TModel>): void {
    // Merge scenario.model into proxy (triggers viewsNotify via Proxy set trap)
    Object.assign(this.proxy, scenario.model);
  }
  
  /** Get proxied model */
  get model(): TModel {
    return this.proxy;
  }
  
  /** Register view for updates */
  viewRegister(view: View<TModel>): void {
    this.views.add(view);
    view.model = this.proxy;
  }
  
  /** Unregister view */
  viewUnregister(view: View<TModel>): void {
    this.views.delete(view);
  }
  
  /** 
   * Notify all views to update - SYNCHRONOUS!
   * Web4 Principle 7: Layer 5 is synchronous
   */
  private viewsNotify(): void {
    // ✅ Use method reference, NOT arrow function!
    this.views.forEach(this.viewNotify.bind(this));
  }
  
  private viewNotify(view: View<TModel>): void {
    view.update();  // SYNCHRONOUS call
  }
}
```

// Controller.interface.ts (layer3)
// ═══════════════════════════════════════════════════════════════
export interface Controller<TModel = any> {
  /** Register a view for model updates */
  viewRegister(view: View<TModel>): void;
  
  /** Unregister a view */
  viewUnregister(view: View<TModel>): void;
  
  /** Update all registered views */
  viewsUpdateAll(): void;

  get model(): TModel;

// ═══════════════════════════════════════════════════════════════
// UcpController.ts (layer2) - Base class for all controllers
// ═══════════════════════════════════════════════════════════════
export class UcpController<TModel extends object> implements Controller<TModel> {
  private registeredViews: Set<View<TModel>> = new Set();
  private modelProxy: TModel;
  
  constructor() {
    // Empty constructor - Web4 Principle 6
  }
  
  /** Initialize with model - creates proxy for auto-updates */
  init(model: TModel): this {
    this.modelProxy = this.proxyCreate(model);
    return this;
  }
  
  private proxyCreate(model: TModel): TModel {
    return new Proxy(model, {
      set: (target, property, value) => {
        (target as any)[property] = value;
        this.viewsUpdateAll();  // ✅ Auto-update all views!
        return true;
      }
    });
  }
  
  get model(): TModel {
    return this.modelProxy;
  }
  
  viewRegister(view: View<TModel>): void {
    this.registeredViews.add(view);
    view.model = this.modelProxy;
  }
  
  viewUnregister(view: View<TModel>): void {
    this.registeredViews.delete(view);
  }
  
  viewsUpdateAll(): void {
    this.registeredViews.forEach(view => view.update());
  }
}

// ═══════════════════════════════════════════════════════════════
// UcpComponent.ts (layer2) - Abstract base for all components
// ═══════════════════════════════════════════════════════════════
export abstract class UcpComponent<TModel extends object> implements Component<TModel> {
  model: TModel;
  protected controller: UcpController<TModel>;
  
  constructor() {
    // Empty constructor - Web4 Principle 6
    this.controller = new UcpController<TModel>();
  }
  
  async init(scenario?: Scenario<TModel>): Promise<this> {
    this.model = scenario?.model || this.modelDefault();
    this.controller.init(this.model);
    return this;
  }
  
  /** Override to provide default model */
  protected abstract modelDefault(): TModel;
  
  // ✅ Delegate view management to controller
  viewRegister(view: View<TModel>): void {
    this.controller.viewRegister(view);
  }
  
  viewUnregister(view: View<TModel>): void {
    this.controller.viewUnregister(view);
  }
  
  viewsUpdateAll(): void {
    this.controller.viewsUpdateAll();
  }
  
  async toScenario(): Promise<Scenario<TModel>> {
    // ... hibernate to scenario
  }
}

// UcpView.ts (layer5/views) - Base class for all views
// ═══════════════════════════════════════════════════════════════

// ⚠️ IMPORTANT: If LitElement isn't compatible with Web4 empty constructors,
// use delegation pattern instead of inheritance:
//
// Option A: Direct inheritance (if Lit decorators work with empty constructor)
//   export abstract class UcpView extends LitElement { ... }
//
// Option B: Delegation pattern (if Lit requires constructor params)
//   export abstract class UcpView { 
//     private litDelegate: LitElement;  // Lit as private delegate
//   }
//   export abstract class LitUcpView extends UcpView {
//     // Lit-specific adapter
//   }

export abstract class UcpView<TModel = any> extends LitElement implements View<TModel> {
  // ✅ Use Reference<T> for nullable - NOT `| null`!
  private modelRef: Reference<TModel> = null;
  protected childViews: UcpView<any>[] = [];
  
  // TypeScript setter - NOT modelConnect()!
  set model(model: TModel) {
    this.modelRef = model;
    this.update();
  }
  
  get model(): TModel {
    if (this.modelRef === null) {
      throw new Error('Model not initialized - call set model first');
    }
    return this.modelRef;
  }
  
  /** 
   * Called when model changes - SYNCHRONOUS! (P7: Layer 5 is sync)
   * Override for custom behavior.
   * 
   * Called by: ModelProxy.viewsNotify() → view.update()
   * This is the entry point from Layer 2 into Layer 5.
   */
  update(): void {
    this.requestUpdate();  // Lit schedules re-render (still sync call)
  }
  
  /** Add child view */
  add(childView: UcpView<any>): void {
    this.childViews.push(childView);
    // Subclass decides where to append
  }
  
  /** Remove child view */
  remove(childView: UcpView<any>): void {
    const index = this.childViews.indexOf(childView);
    if (index > -1) {
      this.childViews.splice(index, 1);
      childView.remove();
    }
  }
  
  /** Abstract: Subclass must implement render */
  abstract render(): TemplateResult;
}

/**
 * OnceOverView - Overview of all ONCE peers

 * This is the Web4 OOP replacement for demo-hub.html

export class OnceOverView extends AbstractWebBean implements View<ONCEModel> {

    this.modelRef = model;

    if (this.modelRef === null) {
      throw new Error('Model not initialized');
    }
    return this.modelRef;

    this.peersRender();
    this.requestUpdate();  // Lit re-render

    this.childViewList.push(childView);
    this.shadowRoot?.querySelector('.peer-grid')?.appendChild(childView);
  }

    if (index > -1) {
      this.childViewList.splice(index, 1);
      childView.remove();
    }

    // ✅ Collection<T> - NOT raw array!

    peers.forEach(peerScenario => {
      const itemView = new ItemView();
      itemView.model = peerScenario.model;
      this.add(itemView);
    });

## 🔄 ACT

### Lessons Learned
- 0.3.21.8 broke real-time updates by removing WebSocket listeners (but NOT to be replaced with protocol messages!)
- IOR-based architecture is correct but needs scenario replication push layer
- Button state management must be in `finally` block for reliability
- **NO STRING LITERALS** - always use enums (ActionStyle, ViewTab)
- **NO STANDALONE FUNCTIONS** - always methods on View classes
- **NO ARROW FUNCTIONS** - use method references: `forEach(this.method.bind(this))`
- **TypeScript setters** (`set model()`) instead of `modelConnect()`
- **View.add()** instead of `appendChild()`
- **View.update()** called by controller (ModelProxy or Lit reactivity)
- CSS and HTML in `layer5/views/css/` and `layer5/views/webBeans/` (not `view/`)
- PascalCase for file names (`ItemView.css` not `item-view.css`)

**Layer Async Rules (P7):**
- **Layer 4 (async)**: WebSocket `onmessage`, `scenarioReceive()` entry point
- **Layer 2 (sync)**: ModelProxy, `viewsNotify()`, domain logic
- **Layer 5 (sync)**: `view.update()`, all View methods are **SYNCHRONOUS**
- **Flow**: `ws.onmessage (async) → scenarioReceive (sync) → view.update (sync)`

**Lazy Deprecation & Migration (P16):**
- Mark old methods `@deprecated` with migration note
- Migrate ONE deprecation per commit (if token budget allows)
- Examples: `startServer() → peerStart()`, `modelConnect() → set model()`

**UCP Base Classes:**
- `UcpComponent` - abstract base for ALL generated Web4 components
- `UcpController` - base class with viewRegister/viewUnregister/viewsUpdateAll
- `UcpView` - base class for ALL views (extends LitElement)
- **Delegation**: UcpComponent delegates view management to UcpController
- **Hierarchy**: Component → manages → Controller → notifies → Views
