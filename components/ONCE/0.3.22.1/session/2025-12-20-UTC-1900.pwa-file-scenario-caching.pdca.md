## P - PLAN

### **Problem Statement**

Current architecture has a **fundamental gap** for PWA offline file storage:

1. **FileModel** (layer3) contains only **metadata** (path, filename, mimetype, size, hash)
2. **FileContent** is explicitly "NOT stored in model" because "Blobs can't be serialized to JSON"
3. **I.7 caching** stores arbitrary JSON in BrowserScenarioStorage — NOT proper Scenarios
4. **Offline use** requires FULL FILE CONTENT to be cached, not just metadata

### **Current Architecture Review**

```
FileModel (layer3/FileModel.interface.ts):
├── path: string           ✅ Metadata
├── filename: string       ✅ Metadata  
├── mimetype: string       ✅ Metadata
├── size: number           ✅ Metadata
├── contentHash: string    ✅ For deduplication
├── createdAt: number      ✅ Metadata
├── modifiedAt: number     ✅ Metadata
├── isLink: boolean        ✅ Metadata
└── linkTarget: string     ✅ Metadata

❌ MISSING: Actual file content!

FileContent (runtime only):
├── blob: Blob             ❌ "Can't serialize to JSON"
├── objectUrl: string      ❌ Runtime-only
└── arrayBuffer: ArrayBuffer ❌ "Can't serialize"
```

### **Web4 Principle Violations**

| Principle | Violation |
|-----------|-----------|
| **P1: Everything is a Scenario** | File content is NOT in the Scenario |
| **P1a: All models extend Model** | FileContent is separate, not in FileModel |
| **PWA Offline** | Can't work offline without full content |

### **Research: Browser Storage Options**

| Question | Answer |
|----------|--------|
| Can IndexedDB store Blobs directly? | **YES** — Native Blob storage, no encoding needed |
| Are Blob URLs (`blob:...`) persistent? | **NO** — Session-only, invalidated on page refresh |
| Base64 overhead? | **33%** larger than binary |
| Best for PWA files? | **IndexedDB for Blobs** OR **Cache API for HTTP** |

**Conclusion**: Use **native Blob in IndexedDB**, NOT Base64 or Blob URLs.

---

### **Proper Architecture for PWA File Caching**

```
Scenario<FileModel>
├── ior: {uuid, component: 'File', version}
├── owner: string
├── model: FileModel
│   ├── path: string
│   ├── filename: string
│   ├── mimetype: string
│   ├── size: number
│   ├── contentHash: string         → For deduplication/Artefact link
│   ├── createdAt: number
│   ├── modifiedAt: number
│   └── content?: Blob              ← NATIVE BLOB (IndexedDB supports!)
└── unit: ScenarioUnit              ← FULL ScenarioUnit (NOT optional for files)
    ├── indexPath: string           → scenarios/index/path
    ├── symlinkPaths: string[]      → ["type/File/...", "domain/..."]
    ├── references: UnitReference[] → Related scenarios
    ├── typeM3?: TypeM3             → M3/M2/M1 classification
    ├── origin?: string             → IOR to source
    ├── definition?: string         → IOR to definition
    ├── schemaVersion: string       → "1.1.0"
    ├── createdBy: string           → "ONCE/0.3.22.1"
    ├── createdAt: string           → ISO timestamp
    └── updatedAt: string           → ISO timestamp
```

### **Storage Strategy: Option C (Approved)**

**Hybrid: IndexedDB for Scenarios, Cache API for HTTP responses**

```
┌───────────────────────────────────────────────────────────────┐
│ BrowserScenarioStorage (IndexedDB)                           │
├───────────────────────────────────────────────────────────────┤
│ scenarios-index:   Scenario<FileModel> with Blob content     │
│ scenarios-type:    Index by component/version                │
│ scenarios-domain:  Index by domain                           │
│ scenarios-artefact: Artefact<ArtefactModel> by contentHash   │
└───────────────────────────────────────────────────────────────┘
           ↓ For HTTP fetch deduplication
┌───────────────────────────────────────────────────────────────┐
│ UnitCacheManager (Cache API)                                 │
├───────────────────────────────────────────────────────────────┤
│ once-units-v1:     Response objects by URL                   │
│                    (for Service Worker fetch interception)   │
└───────────────────────────────────────────────────────────────┘
```

**Why Hybrid?**
1. **Scenarios in IndexedDB** — Structured queries, proper typing
2. **Blob content in IndexedDB** — Native support, no encoding
3. **Cache API for SW** — HTTP response interception pattern
4. **Artefacts for dedup** — Same contentHash = same Artefact

### **Data Flow**

```
Static File Request (Browser)
    ↓
IOR.resolve('https://host/path/file.css')
    ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Check BrowserScenarioStorage for Scenario<FileModel>            │
│    Key: content hash (SHA-256) or path                             │
│                                                                     │
│ 2. If found, check UnitCacheManager for actual content             │
│    Key: artefactUuid                                               │
│                                                                     │
│ 3. If content cached → return from cache                           │
│    If not cached → fetch from network                              │
│                                                                     │
│ 4. After fetch:                                                    │
│    a. Create Scenario<FileModel> with metadata                     │
│    b. Store in BrowserScenarioStorage                              │
│    c. Create Artefact with content hash                            │
│    d. Store content in UnitCacheManager (Cache API)                │
└─────────────────────────────────────────────────────────────────────┘
    ↓
Return file content (from cache or network)
```
