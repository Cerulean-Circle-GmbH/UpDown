**Focus**: Implement all File methods using TypeScript accessors (P16)

| Accessor | Type | Description |
|----------|------|-------------|
| `get name(): string` | SYNC | Returns `model.name` |
| `set name(value: string)` | SYNC | Sets `model.name`, triggers view update |
| `get size(): number` | SYNC | Returns `model.size` |
| `get content(): Reference<string>` | SYNC | Returns `model.content` |
| `set content(value: string)` | SYNC | Sets `model.content`, computes hash |
| `get parent(): LazyReference<Folder>` | SYNC | Returns `model.parent` |
| `set parent(value: Folder)` | SYNC | Sets `model.parent` IOR or instance |
| `get path(): string` | SYNC | Computes full path from parent chain |
| `get mimetype(): string` | SYNC | Returns `model.mimetype` |

### FS.2: DefaultFolder Operations (Layer 2)

**Focus**: Implement all Folder methods using TypeScript accessors (P16) + ISR

| Method/Accessor | Type | Description |
|-----------------|------|-------------|
| `childAdd(child: File)` | SYNC | Adds to `model.children`, sets child.parent |
| `childRemove(child: File)` | SYNC | Removes from `model.children`, clears parent |
| `childLookup(name: string)` | SYNC | Finds child by name (P16: parameterized lookup) |
| `childLookupByUuid(uuid: string)` | SYNC | Finds child by UUID (P16: parameterized lookup) |
| `get children(): Collection<File>` | SYNC | Returns `model.children` (P16: accessor) |
| `get childCount(): number` | SYNC | Returns `model.children.length` (P16: accessor) |
| `get isEmpty(): boolean` | SYNC | Returns `childCount === 0` (P16: accessor) |
| `pathFromRoot()` | SYNC | Walks parent chain to build path array |

**ISR Consideration**: 
- `model.children` contains LazyReference<File> elements
- When accessed via UcpModel proxy, IOR strings become IOR objects
- IOR objects self-replace with instances when resolved
- Methods that iterate children may see mixed types during resolution

### FS.5: Scenario Persistence (Layer 1/4)

**Focus**: Save/load file scenarios via PersistenceManager + IndexedDB

| Operation | Flow |
|-----------|------|
| Create | `orchestrator.create()` → `IOR.create()` → scenario → `PersistenceManager.save()` → IndexedDB |
| Load | `IOR.resolve()` → `PersistenceManager.load()` → IndexedDB → scenario → instance |
| Update | `model.* = value` → proxy.set → `PersistenceManager.save()` → IndexedDB |
| Delete | `orchestrator.delete()` → `PersistenceManager.delete()` → IndexedDB.remove |
