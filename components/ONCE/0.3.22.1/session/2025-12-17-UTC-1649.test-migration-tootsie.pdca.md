## **P - PLAN**

### **Objective**
Migrate all non-Tootsie tests to Tootsie format per Web4 Principle 25 (Tootsie Tests Only).

### **Background: What is Tootsie?**

**Tootsie** = Radical OOP Test Framework (Web4 P25)
- Tests extend `ONCETestCase` base class
- Tests live in `/test/tootsie/` directory
- Run via: `./once tootsie file <number>` or `./once tootsie file <TestName>`
- Black-box testing only (IOR calls, CLI commands)
- Tests can call shell commands via `this.runOnceCLI()` or `this.execSync()`

**Proper Tootsie test structure:**
```typescript
import { ONCETestCase } from './ONCETestCase.js';

export class TestNN_DescriptiveName extends ONCETestCase {
  protected async executeTestLogic(): Promise<any> {
    // Test logic here
    const req = this.requirement('Name', 'Description');
    req.addCriterion('ID', 'Description');
    // ... test ...
    req.validateCriterion('ID', condition, evidence);
    this.validateRequirement(req);
  }
}
```

**Architecture:**
```
VITEST = Infrastructure Layer
├── Test runner (beforeAll, afterAll, describe, it)
├── Test isolation setup → creates test/data/
├── Copies components (Web4TSComponent, ONCE) into test/data
├── Creates symlinks (latest, dev, test)
└── Creates CLI symlinks (scripts/once, scripts/web4tscomponent)

TOOTSIE = Functional Test Layer (uses test/data SET UP by Vitest)
├── ONCETestCase.serverStartInTestIsolation() → uses test/data/scripts/once
├── this.testDataDir → test/data/
└── Web4Requirement for acceptance criteria
```
