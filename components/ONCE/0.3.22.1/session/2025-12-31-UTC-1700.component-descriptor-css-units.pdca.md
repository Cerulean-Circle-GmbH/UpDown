async componentDescriptorUpdate(): Promise<this> {

### Why ONCE.component.json Missing

The `componentDescriptorUpdate()` command has **never been run** for ONCE 0.3.22.1:

```bash
# This has never been executed:
./once on ONCE 0.3.22.1 componentDescriptorUpdate
```

### Model Structure Expected

The `BrowserOnceOrchestrator` expects:

```typescript
componentJson.model?.units?.css  // Array of CSS IOR paths
```

But `unitsDiscoverForDescriptor` returns a **flat array**:

```typescript
targetScenario.model.units = units;  // Just string[]
```

**Mismatch**: Browser looks for `units.css`, but generator creates `units` (flat).

## P - PLAN (Systematic Build Lifecycle Integration)

### Architecture Analysis

**Current Flow**:
1. `npm start` → `start.sh` → `build.sh`
2. `build.sh`:
   - Compiles TypeScript (`npx tsc`) ← **Shell: minimal bootstrap**
   - Copies CSS assets ← Shell
   - Bundles Service Worker ← Shell
3. `./once build` (explicit CLI call) → `NodeJsOnce.build()` → `DefaultWeb4TSComponent.build()`:
   - Compiles TypeScript ← TypeScript
   - Discovers units via `unitsDiscover()` ← **TypeScript: Web4 OOP**
   - Extracts type descriptors via `typesExtract()` ← TypeScript

**Chicken-and-Egg**: `build.sh` must compile TypeScript BEFORE TypeScript CLI is available.

**Problem**: `unitsDiscoverForDescriptor()` returns flat `string[]`, but browser looks for `units.css`.

**File**: `/Users/Shared/Workspaces/2cuGitHub/UpDown/components/ONCE/0.3.22.1/src/ts/layer4/BrowserOnceOrchestrator.ts`

**BEFORE** (lines 218-230):

```typescript
        const cssUnits = componentJson.model?.units?.css || [];
        // Handle both string IORs and legacy object format
        cssFiles = cssUnits.map(function mapCssUnit(unit: string | { path: string }): string {
          if (typeof unit === 'string') {
            // New format: IOR string (relative path)
            // If it starts with '/' it's already absolute, otherwise prepend basePath
            return unit.startsWith('/') ? unit : `${basePath}/${unit}`;
          } else {
            // Legacy format: { path: "..." }
            return `${basePath}/${unit.path}`;
          }
        });
        console.log(`[Orchestrator] Using component.json units: ${cssFiles.length} CSS files`);
```

**AFTER** (handles both categorized and flat array):

```typescript
        // First try categorized format: units.css
        let cssUnits = componentJson.model?.units?.css || [];
        
        // Fallback: flat array format (generated by unitsDiscoverForDescriptor)
        // Filter CSS files by extension from flat model.units array
        if (cssUnits.length === 0 && Array.isArray(componentJson.model?.units)) {
          cssUnits = componentJson.model.units.filter(function filterCss(u: string): boolean {
            return typeof u === 'string' && u.endsWith('.css');
          });
        }
        
        // Handle both string IORs and legacy object format
        cssFiles = cssUnits.map(function mapCssUnit(unit: string | { path: string }): string {
          if (typeof unit === 'string') {
            // New format: IOR string (relative path)
            // If it starts with '/' it's already absolute, otherwise prepend basePath
            return unit.startsWith('/') ? unit : `${basePath}/${unit}`;
          } else {
            // Legacy format: { path: "..." }
            return `${basePath}/${unit.path}`;
          }
        });
        console.log(`[Orchestrator] Using component.json units: ${cssFiles.length} CSS files`);
```

**Change Summary**:
1. Rename `cssUnits` to `let` (mutable)
2. Add flat array fallback with `.filter()` for `.css` extension
