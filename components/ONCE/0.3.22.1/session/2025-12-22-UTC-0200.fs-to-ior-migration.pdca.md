### FsM.1: Protocol Fallback Logic (1h)

**Problem**: IOR doesn't detect environment and skip unavailable protocols.

**Current** (IOR.ts `performResolution`):

```typescript:296:348:src/ts/layer4/IOR.ts
    private async performResolution(options?: IOROptions): Promise<T | null> {
        const protocols = this.parseProtocolChain();
        
        if (protocols.length === 0) {
            throw new Error('[IOR] No protocols in chain');
        }
        
        // ... no environment check!
        
        // Get the transport protocol (last in chain, e.g., 'https')
        const transportProtocol = protocols[protocols.length - 1];
        const loader = IOR.loaders.get(transportProtocol);
```

**Fix**: Add environment-aware protocol fallback:

```typescript
private async performResolution(options?: IOROptions): Promise<T | null> {
    let protocols = this.parseProtocolChain();
    
    if (protocols.length === 0) {
        throw new Error('[IOR] No protocols in chain');
    }
    
    // I.7.1: Check PWA cache first (browser only)
    const cached = await this.cacheCheck();
    if (cached !== undefined) {
        console.log(`💾 [IOR] Cache hit: ${this.model.uuid}`);
        return cached;
    }
    
    // ═══ ENVIRONMENT-AWARE PROTOCOL FALLBACK ═══
    // For 'ior:fs:https://...' chains:
    // - Node.js: use 'fs' directly
    // - Browser: skip 'fs', use 'https'
    
    const isNodeJs = typeof process !== 'undefined' && process.versions?.node;
    
    if (protocols[0] === 'fs' && !isNodeJs) {
        // Browser context: skip 'fs', fallback to next protocol
        protocols = protocols.slice(1);
        console.log(`📡 [IOR] Browser: skipping 'fs', using: ${protocols.join(' → ')}`);
    }
    
    if (protocols.length === 0) {
        throw new Error('[IOR] No available protocols after fallback');
    }
    // ═══ END FALLBACK LOGIC ═══
    
    // Get the transport protocol (last in chain)
    const transportProtocol = protocols[protocols.length - 1];
    const loader = IOR.loaders.get(transportProtocol);
    
    // ... rest unchanged
}
```

**Checklist**:
- [x] Add `isNodeJs` detection ✅
- [x] Add `protocols.slice(1)` for browser fallback ✅
- [x] Add log message for fallback ✅
- [ ] Test in both Node.js and Browser (Node.js ✅, Browser pending)

                return this.handleNotFound(response);
            }
            // ...

**Fix**: Use IOR with async pattern:

```typescript
// BEFORE (sync, blocking):
if (!fs.existsSync(filePath)) {
    return this.handleNotFound(response);
}
const content = fs.readFileSync(filePath);

// AFTER (async, IOR-based):
const fileIOR = new IOR<Buffer>().initRemote(`ior:fs:https://${host}${urlPath}`);
try {
    const content = await fileIOR.resolve();
    if (!content) {
        return this.handleNotFound(response);
    }
    // ... continue with content
} catch (error) {
    return this.handleNotFound(response);
}
```

**Note**: StaticFileRoute runs on Node.js server, so `fs:` will resolve directly.
The `https://host/path` part is for client-side fallback if code is shared.
