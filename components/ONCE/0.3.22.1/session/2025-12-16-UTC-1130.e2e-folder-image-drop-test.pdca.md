## 📋 PROBLEM ANALYSIS

### E2E Test Failure Investigation

**Test Attempted:** Test23_ImageDropE2E  
**Result:** FAILED

### Key Findings

#### 1. SPA Loading Issue in Headless Browser
The initial test navigated to `/once` which serves a minimal kernel bootstrap page:
```
Page title: ONCE Kernel
Unique tags: div, h1, p, script
Has ucp-router: false
```

**Root Cause:** The `/once` route serves `serveOnceMinimal()` - a minimal HTML page with just kernel initialization, NOT the full SPA.

#### 2. Correct Route is `/demo`
Browser investigation with console messages shows `/demo` works correctly:
- Loads Lit components ✅
- Creates `<once-over-view>` ✅
- CSS preloads work ✅
- WebSocket connects ✅
- Service Worker registers ✅

#### 3. Missing Route: Folder Browser
Current registered UcpRouter routes:
```
/          → <once-peer-default-view>
/demo      → <once-over-view>
/onceCommunicationLog → <once-logger-view>
/server-status → <once-server-status-view>
/peer/:uuid → <once-peer-details-view>
```

**MISSING:** No route for `<folder-over-view>` to browse files!

Expected user flow:
1. Navigate to `/browse` or `/files`
2. See `<folder-over-view>` showing `/EAMD.ucp/components/ONCE/0.3.22.1/src/assets/`
3. Drop `ServiceGlobe.jpg` onto the view
