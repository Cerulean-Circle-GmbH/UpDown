### Scenario Structure
```
{Name}.component.json (Scenario<Web4TSComponentModel>)
тФЬтФАтФА ior: { uuid (UUIDv4), component: "Web4TSComponent", version }
тФЬтФАтФА owner: Base64(JSON(Scenario<UserModel>))  тЖР NOT "system"!
тФВ   тФФтФАтФА Scenario<UserModel> from User.toScenario()
тФВ   тФФтФАтФА JSON.stringify() тЖТ Buffer.from().toString('base64')
тФВ   тФФтФАтФА See: NodeJsOnce.ts:531-534, DefaultCLI.ts:405-428
тФЬтФАтФА model: (Web4TSComponentModel)
тФВ   тФЬтФАтФА name, version, description
тФВ   тФЬтФАтФА units: Collection<IOR>  тЖР ALL units as IOR references, NO categories
тФВ   тФВ   тФФтФАтФА e.g., "/EAMD.ucp/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css"
тФВ   тФФтФАтФА entryPoints, dependencies
тФФтФАтФА unit: Scenario<UnitModel>  тЖР The descriptor IS a unit (full scenario, not just metadata)
```

### IOR Format for Units
All unit references use `/EAMD.ucp` as the default static file route:
```
/EAMD.ucp/components/{Name}/{Version}/{path-to-file}
```

Server-side translation to physical location:
```
/EAMD.ucp/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css
  тЖТ /Users/.../UpDown/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css
```
