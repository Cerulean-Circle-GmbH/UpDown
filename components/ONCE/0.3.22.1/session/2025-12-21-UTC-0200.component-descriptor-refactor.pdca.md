### Target Architecture

```
DefaultWeb4TSComponent (Layer 2)
├── unitsDiscover() — Find all files in component
├── unitCreate() — Create unit scenario for a file
├── unitSave() — Save unit scenario + create .unit symlink
├── descriptorUpdate() — Write {Name}.component.json
└── descriptorRead() — Read existing descriptor

{Name}.component.json (Scenario<Web4TSComponentModel>)
├── ior: { uuid (UUIDv4), component: "Web4TSComponent", version }
├── owner: "system"
├── model: (Web4TSComponentModel)
│   ├── name, version, description
│   ├── units: Collection<IOR>  ← ALL units as IOR references, NO categories
│   │   └── e.g., "/EAMD.ucp/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css"
│   └── entryPoints, dependencies
└── unit: Scenario<UnitModel>  ← The descriptor IS a unit (full scenario, not just metadata)
```

### IOR Format for Units

All unit references use `/EAMD.ucp` as the default static file route:
```
/EAMD.ucp/components/{Name}/{Version}/{path-to-file}
```

This is easy to translate server-side to physical location:
```
/EAMD.ucp/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css
  → /Users/.../UpDown/components/ONCE/0.3.22.1/src/ts/layer5/views/css/FileItemView.css
```
