## **P - PLAN**

### **Objective**
Eliminate 25 scattered `fetch()` calls by centralizing all HTTP transport through `HTTPSLoader` (now in layer4). This ensures:
- **DRY**: Single implementation of fetch logic
- **P7 Compliance**: Async only in layer4
- **IOR Integration**: All requests can use IOR failover profiles

### **Web4 Component Architecture (L17)**
```
Layer 2 (Component) → owns model, manages views, triggers layer4
Layer 5 (Views) → render model, handle events, NEVER fetch
Layer 4 (Orchestrators) → ONLY async layer, fetch() lives HERE

Flow: View needs data → component.method() → layer4.async() → fetch → callback → model update → re-render
```

**Critical Rule**: Views NEVER do direct backend interaction. Only layer4 processes triggered by layer2 methods.

### **F.1: IOR as Unified Entry Point (TRON-Approved)**

**Goal**: IOR becomes THE entry point for ALL remote operations. HTTPSLoader becomes internal.

**Pattern**: `new IOR().init(urlOrIorString).load()`

```typescript
// ✅ NEW: IOR handles everything
const data = await new IOR().init('https://host:port/path').load();
const data = await new IOR().init('ior:scenario:REST:https://host:port/Component/1.0.0/uuid').load();

// IOR internally:
// 1. Parses protocol chain: ior:scenario:REST:https
// 2. Finds/chains loaders: ScenarioLoader → RESTLoader → HTTPSLoader
// 3. Executes through chain
// 4. Returns result
```

**Loader Chain Architecture**:
```
┌────────────────────────────────────────────────────────────────┐
│  IOR.init(urlOrIorString)                                      │
│  ├── parse protocol chain: ior:scenario:REST:https://...       │
│  ├── find loaders for each protocol (internal registry)        │
│  └── IOR.load() chains through loaders                         │
├────────────────────────────────────────────────────────────────┤
│  ScenarioLoader (protocol: 'scenario')                         │
│  ├── Converts JSON ↔ Scenario<T>                               │
│  └── Delegates to next loader in chain                         │
├────────────────────────────────────────────────────────────────┤
│  RESTLoader (protocol: 'REST')                                 │
│  ├── Handles HTTP verbs (GET, POST, PUT, DELETE)               │
│  └── Delegates to transport loader                             │
├────────────────────────────────────────────────────────────────┤
│  HTTPSLoader (protocol: 'https')                               │
│  ├── THE transport layer (fetch lives HERE)                    │
│  └── Handles TLS, TCP/IP, failover profiles                    │
└────────────────────────────────────────────────────────────────┘
```

**Reference<T> Unification**:
```typescript
// Reference<T> can be local OR remote (IOR)
type Reference<T> = T | IOR<T> | null;

// Usage: transparent access
const user: Reference<User> = getUser(); // Could be local or IOR
if (user instanceof IOR) {
    const resolved = await user.load(); // Resolves IOR to User
}
```

### **F.3: Layer5 View Fixes (ARCHITECTURAL VIOLATIONS)**

⚠️ **Views must NEVER fetch directly!** They must ask the component.

```typescript
// ❌ WRONG (current state)
class MyView extends DefaultView<MyComponent> {
    async render() {
        const response = await fetch('/api/endpoint');  // VIOLATION!
        const data = await response.json();
        this.renderData(data);
    }
}

// ✅ CORRECT (target state)
class MyView extends DefaultView<MyComponent> {
    render() {
        // View renders from model (reactive)
        this.renderData(this.model.data);
        
        // If data not loaded yet, ask component
        if (!this.model.data) {
            this.component.loadEndpointData();  // Component handles async
        }
    }
    
    // Optional interceptor (NOT for triggering render - UcpModel does that)
    onDataLoaded?(data: any) {
        // Use only for: logging, transformation, side effects
        // NOT for calling renderData() - that's automatic via model change
    }
}

// Component (layer2)
class MyComponent extends UcpComponent {
    async loadEndpointData() {
        // Component triggers layer4
        const data = await new DefaultIOR().init('/api/endpoint').load();
        
        // Model assignment triggers automatic re-render via UcpModel proxy
        this.model.data = data;
        // ↑ UcpModel detects change → Controller.onModelChange() → View.render()
    }
}
```

**Key Insight**: `this.model.data = data` triggers re-render automatically. No callback needed.
