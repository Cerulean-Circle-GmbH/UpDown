## Problem

Unit symlinks (`.unit` files) are being created in the wrong location:

| Current (Wrong) | Expected (Correct) |
|-----------------|-------------------|
| `components/ONCE/0.3.22.1/src/ts/layer5/views/css/DefaultItemView.css.unit` | `scenarios/type/ONCE/0.3.22.1/src/ts/layer5/views/css/DefaultItemView.css.unit` |

**Root Cause**: `UnitDiscoveryService.unitCreate()` calculates `unitSymlinkPath` relative to `componentRoot`, not `scenarios/type/`.

---

## Analysis

### Current Code

**File**: `src/ts/layer2/UnitDiscoveryService.ts`

**Line 236** — `unitCreate()`:

```typescript
    // Calculate unit symlink path
    const unitSymlinkPath = path.join(this.componentRoot, `${definition.relativePath}.unit`);
```

This creates: `/Users/.../components/ONCE/0.3.22.1/src/ts/layer5/views/css/DefaultItemView.css.unit`

**Line 255** — `unitSave()`:

```typescript
    // Build symlink paths
    const componentUnitPath = `components/${this.componentName}/${this.componentVersion}/${result.definition.relativePath}.unit`;
```

This is also wrong — it uses `components/` instead of `scenarios/type/`.

### Expected Behavior

Unit symlinks should mirror the component structure inside `scenarios/type/`:

```
scenarios/
└── type/
    └── ONCE/
        └── 0.3.22.1/
            └── src/
                └── ts/
                    └── layer5/
                        └── views/
                            └── css/
                                └── DefaultItemView.css.unit → ../../../../../../index/a/b/c/d/e/{uuid}.scenario.json
```

## P - PLAN

### P.1: Fix `unitCreate()` — Calculate Correct Symlink Path

**File**: `src/ts/layer2/UnitDiscoveryService.ts`

**BEFORE** (line 235-236):

```typescript
    // Calculate unit symlink path
    const unitSymlinkPath = path.join(this.componentRoot, `${definition.relativePath}.unit`);
```

**AFTER**:

```typescript
    // Calculate unit symlink path in scenarios/type/ mirror
    // @pdca 2025-12-31-UTC-1800.unit-symlink-location-fix.pdca.md
    const unitSymlinkPath = path.join(
      this.projectRoot,
      'scenarios',
      'type',
      this.componentName,
      this.componentVersion,
      `${definition.relativePath}.unit`
    );
```

### P.2: Fix `unitSave()` — Remove Duplicate Component Path

**File**: `src/ts/layer2/UnitDiscoveryService.ts`

**BEFORE** (line 254-261):

```typescript
    // Build symlink paths
    const componentUnitPath = `components/${this.componentName}/${this.componentVersion}/${result.definition.relativePath}.unit`;
    
    // Save scenario
    await this.scenarioService.scenarioSave(result.scenario, [
      this.scenarioService.typePathBuild('Unit', this.componentVersion),
      componentUnitPath,
    ]);
```

**AFTER**:

```typescript
    // Build type mirror path for scenario symlink
    // Unit symlink: scenarios/type/{Component}/{version}/{relativePath}.unit
    // @pdca 2025-12-31-UTC-1800.unit-symlink-location-fix.pdca.md
    const typeUnitPath = `type/${this.componentName}/${this.componentVersion}/${result.definition.relativePath}.unit`;
    
    // Save scenario with symlinks in scenarios/type/ and scenarios/index/
    await this.scenarioService.scenarioSave(result.scenario, [
      this.scenarioService.typePathBuild('Unit', this.componentVersion),
      typeUnitPath,
    ]);
```
