## Vision Statement (TRON)

> "We need a complete picture from nodejs load file, find or create unit, create descriptor, once browser client, load component, load descriptor, preload css, cache css in PWA via SW, instantiate component, instantiate views, inject css, render views, go offline and get the files as units from the sw cache. ## Key Principles

### P34: IOR as Unified Entry Point

**ALL resource access MUST go through IOR:**

```typescript
// ❌ FORBIDDEN
const data = await fetch('/path/to/resource');
const content = fs.readFileSync('/path/to/file');

// ✅ REQUIRED
const data = await new IOR().init('ior:https://host/path').load();
const content = await new IOR().init('ior:fs:https://host/path/to/file').load();
```

### 🔑 TRUE PEER-TO-PEER: Identical Code on Client AND Server

**CRITICAL INSIGHT (TRON)**: To make code truly peer-to-peer identical:

```typescript
// ✅ SAME CODE ON BROWSER AND NODE.JS:
const content = await new IOR().init('ior:fs:https://host/path/to/file').load();

// On BROWSER: 
//   → IOR detects browser context
//   → Uses 'https://' protocol to fetch from server
//   → Returns file content

// On NODE.JS (SERVER):
//   → IOR detects Node.js context  
//   → SKIPS 'https://' part (it's local!)
//   → Uses direct fs access
//   → Returns file content
```

**The Protocol Chain**: `ior:fs:https://host/path`

| Context | Protocol Resolution |
|---------|---------------------|
| Browser | `fs:` → not available → fallback to `https:` → fetch from server |
| Node.js | `fs:` → available → use directly, skip `https:` |

This means **the same code runs on both client and server** — true P2P architecture!

### 🔄 Sync Cycle Detection (Prevent Infinite Loops)

**CRITICAL**: When UcpController receives a model update, it MUST know the **source**:

```typescript
// Update source detection
enum UpdateSource {
  LOCAL,        // User action in this browser
  WEBSOCKET,    // From server broadcast
  PERSISTENCE   // From cache/IndexedDB load
}

// UcpController logic:
if (source === UpdateSource.LOCAL) {
  // 1. Save to IndexedDB ✅
  // 2. Send via WebSocket ✅
  // 3. Update views ✅
}

if (source === UpdateSource.WEBSOCKET) {
  // 1. Save to IndexedDB ✅
  // 2. DO NOT send via WebSocket ❌ (would cause loop!)
  // 3. Update views ✅
}
```

This prevents the infinite broadcast loop: `Client A → Server → Client A → Server → ...`

### No fetch(), No fs Functions — IOR ONLY

| Context | Old Way | Web4 Way |
|---------|---------|----------|
| Browser HTTP | `fetch(url)` | `IOR.load('ior:https://...')` |
| Browser Cache | `caches.match()` | `IOR.load('ior:cache://...')` |
| **File Access** | `fs.readFile()` / `fetch()` | `IOR.load('ior:fs:https://...')` ← **SAME CODE!** |
| Scenario | `JSON.parse(fs.read)` | `IOR.load('ior:scenario://...')` |
| WebSocket | `ws.send()` | `IOR.save('ior:wss://...')` |

### Protocol Fallback Logic (NEEDED)

```typescript
// In IOR.performResolution():
const protocols = this.parseProtocolChain();  // e.g., ['fs', 'https']

// Check if first protocol is available
const firstProtocol = protocols[0];  // 'fs'
const loader = IOR.loaders.get(firstProtocol);

if (firstProtocol === 'fs') {
    // Node.js context check
    const isNodeJs = typeof process !== 'undefined' && process.versions?.node;
    if (isNodeJs) {
        // Use FileLoader directly
        return await loader.load(path);
    } else {
        // Browser: skip 'fs', use next protocol ('https')
        protocols.shift();
        return await this.resolveWithProtocols(protocols);
    }
```
