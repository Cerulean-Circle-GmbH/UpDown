## Problem Statement

Browser serves **stale JavaScript** even after:
1. Setting `Cache-Control: no-cache` on JS files ✅
2. Implementing network-first Service Worker strategy ✅
3. Manual cache clear ✅

**Symptom**: Console shows `cwd: /EAMD.ucp/components/ONCE/0.3.22.1/src/assets` when this value was removed from the source code and compiled JS.

**Root Cause**: 
- ESM modules are cached in the browser's **module loader** (separate from HTTP cache and SW cache)
- Once a module is imported, it's cached for the browser session
- Even `no-cache` headers don't invalidate already-loaded modules

---

## CMM4 Analysis: Three Caching Layers

| Layer | Type | Cache Clear Method |
|-------|------|-------------------|
| 1 | **HTTP Cache** | `Cache-Control: no-cache` ✅ Done |
| 2 | **Service Worker Cache** | Network-first strategy ✅ Done |  
| 3 | **ESM Module Cache** | **Page reload with new URL** ❌ Not done |

**The Missing Piece**: ESM modules stay cached until the page is **fully reloaded** with a different module URL.

---

## Plan: Version-Based Cache Busting

### Option A: Build Timestamp in Import URLs (RECOMMENDED)

**Strategy**: Append `?v={timestamp}` to all dynamically loaded module URLs.

**Location**: `ServerHierarchyManager.ts` — the HTML template generator

**Current Code** (line ~2480):

```typescript
await import('/EAMD.ucp/components/ONCE/${version}/dist/ts/layer5/views/UcpView.js');
```

**Fixed Code**:

```typescript
// ${buildTimestamp} is set at server start or compile time
await import('/EAMD.ucp/components/ONCE/${version}/dist/ts/layer5/views/UcpView.js?v=${buildTimestamp}');
```

**Implementation Steps**:

1. **CB.1**: Add `buildTimestamp` property to `NodeJsOnce` (set at server start)
2. **CB.2**: Pass `buildTimestamp` to all HTML template generators
3. **CB.3**: Append `?v=${buildTimestamp}` to all `import()` URLs in HTML templates
4. **CB.4**: Update `StaticFileRoute` to ignore query params when resolving file paths

### Option B: ETag-Based Validation

**Strategy**: Use ETags with content hashing for validation.

**Pros**: Standard HTTP caching, no URL changes needed
**Cons**: Requires browser to make validation request; ESM modules may bypass this

### Option C: Service Worker Version Mismatch Detection

**Strategy**: SW compares cached version vs server version, forces refresh on mismatch.

**Current** `OnceServiceWorker.ts`:

```typescript
this.model = {
  version: '0.0.0',  // ❌ Wrong: should match ONCE version
  cacheName: 'once-cache-v0.0.0',
  // ...
};
```

**Corrected Approach**: 

- **Production**: Use ONCE component version: `0.3.22.1`
- **Development**: Use version + timestamp: `0.3.22.1-1704214800000` or `0.3.22.1-2026-01-02T14:00:00Z`

This ensures:
1. SW cache is tied to actual component version
2. Development builds get fresh code on every restart
3. Production builds have stable caching

**Enhanced Logic**:

```typescript
async versionCheck(): Promise<void> {
  // Fetch server version from /health or /version endpoint
  const serverResponse = await fetch('/health');
  const { version } = await serverResponse.json();
  
  // Compare with SW's cached version
  if (version !== this.model.version) {
    console.log('[SW] Version mismatch, clearing cache...');
    
    // Clear all caches
    const cacheNames = await caches.keys();
    await Promise.all(cacheNames.map(name => caches.delete(name)));
    
    // Update SW version
    this.model.version = version;
    this.model.cacheName = `once-cache-v${version}`;
    
    // Signal clients to reload
    const clients = await self.clients.matchAll();
    clients.forEach(client => {
      client.postMessage({ type: 'CACHE_CLEARED', version });
    });
  }
}
```

**Client Side** (in HTML template):

```javascript
navigator.serviceWorker.addEventListener('message', (event) => {
  if (event.data.type === 'CACHE_CLEARED') {
    console.log('[Client] SW cleared cache for version:', event.data.version);
    window.location.reload();
  }
});
```

**Why Both?**

### CB.4: SW Version Mismatch Detection

```typescript
/**
 * Check if server version differs from cached version
 * Clear cache and signal clients if mismatch detected
 * @pdca 2026-01-02-UTC-1400.cache-busting-systematic-fix.pdca.md CB.4
 */
async versionCheck(): Promise<boolean> {
  try {
    const response = await fetch('/health?_=' + Date.now());  // Bypass cache
    const { version } = await response.json();
    
    if (version !== this.model.version) {
      console.log(`[SW] Version mismatch: cached=${this.model.version}, server=${version}`);
      
      // Clear all caches
      const cacheNames = await caches.keys();
      await Promise.all(cacheNames.map(name => caches.delete(name)));
      
      // Update version
      this.model.version = version;
      this.model.cacheName = `once-cache-v${version}`;
      
      // Signal all clients to reload
      const clients = await (self as any).clients.matchAll();
      clients.forEach((client: any) => {
        client.postMessage({ type: 'VERSION_UPDATED', version });
      });
      
      return true;  // Version changed
    }
    
    return false;  // Version same
  } catch (error: any) {
    console.warn('[SW] Version check failed:', error.message);
    return false;
  }
}
```
