## 📋 PLAN

### Problem Statement
The `/browse` route was created as a quick solution, but the proper design should:
1. Use `/EAMD.ucp` as the route (matching the static file serving pattern)
2. Support `defaultPath` as root boundary (can't navigate above)
3. Support `cwd` as initial current working directory
4. Show breadcrumb from root to cwd
5. Enable navigation both upward (to root) and downward (into folders)

### Design

```
Route: /EAMD.ucp
Options:
  defaultPath: "/EAMD.ucp/"           # Root boundary - cannot navigate above
  cwd: "/EAMD.ucp/components/ONCE/0.3.22.1/src/assets"  # Initial folder to show

Breadcrumb Display:
  [EAMD.ucp] > [components] > [ONCE] > [0.3.22.1] > [src] > [assets]
   (root)      (clickable)   (clickable)  ...          ...    (current)

Navigation:
  - Click breadcrumb item → navigate to that folder
  - Click folder item → navigate into folder
  - Click ">" on folder → navigate into folder (alternative)
  - Cannot navigate above defaultPath (root boundary)
```

### Changes Required

#### R.2: Add `cwd` Parameter Support

- Load `cwd` initially, use `defaultPath` as navigation boundary

#### R.3: Enhanced Breadcrumb Navigation
- **FolderOverView.ts**: Calculate breadcrumb from `defaultPath` to `currentPath`
- Make breadcrumb items clickable (except current)
- Respect `defaultPath` as upper boundary

#### R.4: Navigation Boundary Enforcement
- Prevent navigation above `defaultPath`
- Hide "up" navigation when at root boundary
