# 🧱 Web4 Component Anatomy Checklist

**Purpose:** Verify component structure before release. UCP = Unit-Component-Package.

---

## Component Criteria (4 MUST HAVE)

- [ ] **C1: Black Box** — Internal implementation hidden, only interfaces exposed
- [ ] **C2: Interfaces Exposed** — Public API defined in `layer3/*.interface.ts`
- [ ] **C3: Self-Contained** — All code in versioned folder `/components/{Name}/{Version}/`
- [ ] **C4: Machine-Readable Descriptor** — `{Name}.component.json` exists and is valid

---

## Component Descriptor (`xyz.component.json`)

- [ ] **D1: IS a Scenario** — Has `{ ior, owner, model }` structure
- [ ] **D2: Has Own Unit** — `unit: Scenario<UnitModel>` describes the descriptor itself
- [ ] **D3: Lists ALL Units** — Every file needed for the component lifecycle
- [ ] **D4: Generator Created** — NOT manually edited (generated by `UnitDiscoveryService`)

---

---

## Unit Structure (.unit symlinks)

- [ ] **U1: Symlink Exists** — `{file}.unit` symlink next to original file
- [ ] **U2: Points to Scenario** — `→ scenarios/index/{uuid-path}/{uuid}.scenario.json`
- [ ] **U3: Scenario Has IOR** — `{ uuid, component: "Unit", version }`
- [ ] **U4: Model Has Origin** — Git IOR: `ior:git:github.com/.../path/to/file`
- [ ] **U5: Model Has References** — `symlinkPaths[]` for deduplication

---

## Lifecycle Coverage

- [ ] **L1: Download** — Can be fetched via IOR
- [ ] **L2: Install** — `src/sh/install-deps.sh` handles dependencies
- [ ] **L3: Build** — `src/sh/build.sh` compiles TypeScript
- [ ] **L4: Test** — `./once tootsie` runs all tests
- [ ] **L5: Start** — `npm start` or `./once` starts component
- [ ] **L6: Clean** — `src/sh/clean.sh` removes build artifacts
- [ ] **L7: Uninstall** — Can be removed without breaking system
