## P - PLAN

### TRON Feedback (Critical Correction)

> "The one layer of references look ahead loading is also not specific for folder or file only. It's the **default behaviour for model**.
> 
> An IOR references a scenario that gets instantiated. The instance has references that might be IORs or local instances. On dereferencing the instance, **all references are prefetched**. This could mean to load multiple classes and instantiate them, because the scenario model references them."

### What We Missed in FFM.3.6

**FFM.3.6 implemented folder-specific prefetching**:
- `DefaultFolder.resolve()` — prefetches `model.children`
- `DefaultFolder.childSelect()` — triggers next layer

**But this is WRONG architecture!**

ONE LAYER LOOKAHEAD should be:
- **Universal** — applies to ALL `Reference<T>` fields in ANY model
- **Automatic** — happens in `IOR.resolveInstance()` / `IOR.instantiate()`
- **Not folder-specific** — `DefaultFolder.resolve()` becomes a thin wrapper

### Correct Architecture

```
IOR.resolveInstance<C>() or IOR.instantiate<C>(scenario)
│
├── 1. Load scenario (if IOR string)
├── 2. Instantiate component: new Class().init(scenario)
├── 3. 🔴 PREFETCH all Reference<T> fields in model ← NEW!
│      │
│      ├── Scan model for Reference<T> fields (IOR strings)
│      ├── For each IOR string: resolve to instance
│      ├── Replace IOR string with instance in model
│      └── (Nested references stay as IOR strings — ONE LAYER only)
│
└── 4. Return fully prefetched instance
```

### Example: What Should Happen

```typescript
// Scenario with nested references
const folderScenario = {
  ior: { uuid: 'folder-1', component: 'Folder', version: '0.3.22.1' },
  model: {
    uuid: 'folder-1',
    name: 'Documents',
    children: [
      'ior:scenario:file-1',     // IOR string
      'ior:scenario:subfolder-1' // IOR string
    ],
    parent: 'ior:scenario:root-folder'  // IOR string
  }
};

// After IOR.instantiate(folderScenario):
const folder = await IOR.instantiate<DefaultFolder>(folderScenario);

// NOW folder.model should be:
folder.model = {
  uuid: 'folder-1',
  name: 'Documents',
  children: [
    DefaultFile { model: { uuid: 'file-1', ... } },       // INSTANCE!
    DefaultFolder { model: { uuid: 'subfolder-1', ... } } // INSTANCE!
  ],
  parent: DefaultFolder { model: { uuid: 'root-folder', ... } } // INSTANCE!
};

// BUT subfolder's children are still IOR strings (ONE LAYER only):
folder.model.children[1].model.children = [
  'ior:scenario:grandchild-1',  // Still IOR string!
  'ior:scenario:grandchild-2'   // Still IOR string!
];
```

## Research: How to Identify Reference<T> Fields

### Option A: Runtime Introspection via TypeDescriptor

TypeDescriptor already tracks field types. We can:
1. Get TypeDescriptor for the component class
2. Iterate over fields marked as `Reference<T>`
3. Resolve each IOR string to instance

**Pros**: Clean, DRY, uses existing infrastructure
**Cons**: Requires TypeDescriptor to be populated with field info

### Option B: Convention-Based Detection

Scan model for values that are:
- Strings starting with `'ior:'`
- Arrays of strings starting with `'ior:'`

**Pros**: Simple, no infrastructure changes
**Cons**: Less type-safe, might catch non-Reference fields

### Option C: Model Interface Annotation

Add `_referenceFields` to Model interface listing which fields are references.

**Pros**: Explicit, no guessing
**Cons**: Manual maintenance, could get out of sync

### DECISION: Option B (Convention-Based)

For MVP, use convention-based detection:
- Any string value starting with `'ior:'` is a reference
- Arrays are handled recursively
- Objects are scanned for string properties

This is simple and works with existing models without changes.

## 🔴 TRON Research Feedback: Lazy Prefetch with Promise Memoization

> "Research what it means when the model proxy in UcpModel triggers the prefetch but does not await the result, but a subsequent trigger would await until the reference is finally resolved."

### Current Approach (Blocking)

```
IOR.instantiate(scenario)
├── new Class().init(scenario)
├── await prefetchReferences(model)  ← BLOCKING! Waits for ALL children
└── return instance
```

**Problem**: Every instantiation blocks until ALL references are resolved. Slow if you only need one child.

### Proposed Approach: Lazy Prefetch via UcpModel Proxy

```
IOR.instantiate(scenario)
├── new Class().init(scenario)       ← model has IOR strings
└── return instance (immediately!)   ← NO BLOCKING

Later: component.model.children[0]
├── UcpModel.get() detects 'ior:...' string
├── Fire-and-forget: triggers IOR.dereference() (non-blocking)
├── Returns IOR string (first access)
└── Stores promise in pendingReferences map

Even later: component.model.children[0] (while resolving)
├── UcpModel.get() sees pending promise
├── AWAIT the same promise (deferred blocking)
└── Replace IOR string with instance in rawModel
└── Return instance

After resolved: component.model.children[0]
├── UcpModel.get() sees instance (not string)
└── Return instance immediately
```

### Key Insight: UcpModel Proxy `get` Handler

The `get` handler in `UcpModel.modelProxyCreate()` currently just returns the raw value:

```typescript
get: function(target, property) {
  return (target as Record<string | symbol, unknown>)[property];
}
```

**Enhanced version with lazy prefetch**:

```typescript
/** Map of pending resolution promises */
private pendingReferences: Map<string, Promise<unknown>> = new Map();

get: function(target, property) {
  const value = (target as Record<string | symbol, unknown>)[property];
  
  // 1. Already an instance? Return it
  if (typeof value !== 'string' || !value.startsWith('ior:')) {
    return value;
  }
  
  // 2. IOR string detected — check if resolution already pending
  const iorString = value as string;
  const pendingKey = `${String(property)}:${iorString}`;
  
  if (ucpModel.pendingReferences.has(pendingKey)) {
    // Resolution in progress — this is a SUBSEQUENT access
    // AWAIT the pending promise (deferred blocking)
    // Note: This makes `get` async, which requires careful handling
    return ucpModel.pendingReferences.get(pendingKey);
  }
  
  // 3. First access — trigger prefetch (fire-and-forget)
  const resolutionPromise = IOR.dereference(iorString).then(function onResolved(instance) {
    // Replace IOR string with instance in rawModel
    (target as Record<string | symbol, unknown>)[property] = instance;
    // Remove from pending
    ucpModel.pendingReferences.delete(pendingKey);
    return instance;
  });
  
  // Store the pending promise
  ucpModel.pendingReferences.set(pendingKey, resolutionPromise);
  
  // Return IOR string for now (first access is non-blocking)
  return value;
}
```

### Challenge: Proxy `get` Cannot Be Async

**Problem**: JavaScript Proxy `get` handlers MUST be synchronous. Cannot `await` inside.

**Solutions**:

#### Solution A: Return Promise on Pending (caller must await)

```typescript
// Caller code:
const child = await component.model.children[0];  // Caller awaits
```

**Pros**: Clean, explicit
**Cons**: Breaks existing code that expects sync access

#### Solution B: Dual Access Pattern (sync + async)

```typescript
// Sync access (returns IOR string if unresolved)
const child = component.model.children[0];

// Async access (awaits if pending, returns instance)
const child = await component.resolveProperty('children', 0);
```

**Pros**: Backward compatible
**Cons**: Two different APIs

#### Solution C: Layer 4 Orchestrator for Async Access

Move all Reference<T> access to Layer 4 orchestrators:

```typescript
// Layer 4 orchestrator (async allowed)
const child = await component.childGet(0);  // Handles resolution
```

**Pros**: Clean separation, follows P7 (async only in Layer 4)
**Cons**: Requires orchestrator methods for every reference access

### DECISION: Solution C — Layer 4 Orchestrators (Web4 P7 Compliant)

- UcpModel proxy triggers prefetch (fire-and-forget) on `get`
- Returns IOR string immediately (sync — Layer 3)
- Layer 4 orchestrator methods (`childGet()`, `parentGet()`) await resolution
- Follows P7: Async only in Layer 4

### Revised Architecture

```
Layer 3 (UcpModel proxy — SYNC):
  get children[0]
  ├── Value is IOR string 'ior:...'
  ├── Fire-and-forget: trigger resolution
  ├── Return IOR string (non-blocking)
  └── Subsequent get: still returns string until resolved

Layer 4 (Orchestrator — ASYNC):
  await folder.childGet(0)
  ├── Check model.children[0]
  ├── If IOR string: await resolution
  ├── If pending: await same promise
  ├── If instance: return immediately
  └── Return instance
```

### URP.2: Add Lazy Prefetch to UcpModel.get (1.5h)

Enhance `modelProxyCreate()` with fire-and-forget prefetch:

```typescript
/** Map of pending resolution promises: "propertyKey:iorString" → Promise */
private pendingReferences: Map<string, Promise<unknown>> = new Map();

private modelProxyCreate(model: T): T {
  const ucpModel = this;
  
  return new Proxy(model, {
    set: function(target, property, value) {
      // ... existing set logic ...
    },
    get: function(target, property) {
      const value = (target as Record<string | symbol, unknown>)[property];
      
      // 1. Not a string or not IOR? Return as-is
      if (typeof value !== 'string' || !value.startsWith('ior:')) {
        return value;
      }
      
      // 2. IOR string detected — trigger prefetch (fire-and-forget)
      const iorString = value as string;
      const pendingKey = String(property) + ':' + iorString;
      
      if (!ucpModel.pendingReferences.has(pendingKey)) {
        // First access — trigger async resolution (non-blocking)
        const { IOR } = require('../layer4/IOR.js');  // Dynamic import to avoid circular
        const resolutionPromise = IOR.dereference(iorString).then(
          function onResolved(instance: unknown): unknown {
            // Replace IOR string with instance
            (target as Record<string | symbol, unknown>)[property] = instance;
            ucpModel.pendingReferences.delete(pendingKey);
            return instance;
          }
        );
        ucpModel.pendingReferences.set(pendingKey, resolutionPromise);
      }
      
      // 3. Return IOR string (sync — Layer 3)
      //    Caller must use Layer 4 orchestrator to await
      return value;
    }
  });
}
```

### URP.3: Add Layer 4 Reference Resolution Helper (30min)

Add to `IOR.ts` (Layer 4) — helper for orchestrators:

```typescript
/**
 * Await a Reference<T> that may be IOR string, pending promise, or instance.
 * 
 * P7 Compliant: This is Layer 4 async code.
 * 
 * @param value The reference value from model
 * @param pendingPromise Optional pending promise from UcpModel
 * @returns Resolved instance
 */
static async awaitReference<T>(
  value: T | string | null, 
  pendingPromise: Reference<Promise<T>>
): Promise<Reference<T>> {
  // 1. Null/undefined
  if (value === null || value === undefined) {
    return null;
  }
  
  // 2. Already an instance
  if (typeof value !== 'string') {
    return value;
  }
  
  // 3. IOR string — check for pending promise
  if (pendingPromise) {
    return await pendingPromise;
  }
  
  // 4. No pending — resolve now
  return await IOR.dereference<T>(value);
}
```

async resolve(): Promise<this> {

  // Skip if this UUID already visited

    return model;
  }
  visited.add(model.uuid);

}
