## Method Analysis: Questions for TRON

### Q1: `start()` (external line 58-89)

External purpose: CLI command that runs `npm start` in a loaded component's directory
```typescript
async start(): Promise<this> {
  execSync('npm start', { cwd: componentPath, stdio: 'inherit' });
}
```

**Question**: Do we need `./once start` as a CLI command? 
- This is DIFFERENT from `BrowserOnce.start()` (browser init)
- This is DIFFERENT from `DefaultCLI.start()` (CLI entry point)

### Q2: `tootsie()` (external lines 2280-2644)

External `tootsie()` is COMPLEX:
1. Sets up test isolation context
2. Calls `discoverTestFile()` (private helper)
3. Calls `loadTootsieComponent()` (private helper)
4. Calls `executeTestFile()` (private helper)
5. Uses TootsieTestRunner from Tootsie/0.3.20.6

ONCE's inline `test()` (lines 160-196) is SIMPLE:
```typescript
async test(scope?: string, ...references: string[]): Promise<this> {
  execSync(`npx tsx test/tootsie/Test${ref.padStart(2, '0')}*.ts`, { cwd: componentRoot });
}
```

**Question**: 
- Should `./once tootsie` call the inline `test()` method?
- OR should we migrate the full `tootsie()` with helpers?

**Option A**: Add alias `tootsie = test` in ONCECLI (simple, ~5 lines)
**Option B**: Migrate full `tootsie()` + 4 private helpers (~400 lines)

### Q3: `componentStart()` (external lines 1632-1729)

Called at `NodeJsOnce.ts:3069`:
```typescript
await web4ts.componentStart(componentName, version);
```

**Question**: Should `componentStart()` become a NodeJsOnce method?
- It's about building/preparing other components
- Currently delegated via web4ts

### Q4: `componentDescriptorUpdate()` vs `manifestUpdate()`

ONCE has `manifestUpdate()` which writes `asset-manifest.json` (OLD).
External has `componentDescriptorUpdate()` which writes `{Name}.component.json` (NEW).

**Action**: Replace `manifestUpdate()` with `componentDescriptorUpdate()`
