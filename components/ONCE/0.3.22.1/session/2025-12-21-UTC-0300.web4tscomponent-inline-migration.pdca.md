## TRON Decisions (Confirmed 2025-12-21)

### Q1: `start()` — ✅ DEFER
Not needed now. ONCE already has `BrowserOnce.start()` and `DefaultCLI.start()` for different purposes.

### Q2: `tootsie()` — ✅ OPTION B (Full Migrate)

**TRON**: "tootsie is fundamentally different from test"

| Aspect | ONCE's `test()` | External `tootsie()` |
|--------|-----------------|---------------------|
| Test Isolation | ❌ None | ✅ Switches to `test/data` |
| Discovery | Simple glob | Pattern matching with scopes |
| Tootsie Component | ❌ Direct exec | ✅ Loads dynamically |
| TootsieTestRunner | ❌ No | ✅ Yes |
| Quality Consciousness | ❌ Simple runner | ✅ Full framework |

**Action**: Migrate full `tootsie()` + 4 private helpers (~400 lines)

### Q3: `componentStart()` — ✅ INLINE
Migrate into ONCE's `DefaultWeb4TSComponent` to reduce external dependency.

### Q4: `componentDescriptorUpdate()` — ✅ CONFIRMED

**Key Insight**: `{Name}.component.json` IS the manifest (better version)!
- `manifestUpdate()` writes obsolete `asset-manifest.json`
- `componentDescriptorUpdate()` writes proper `{ ior, owner, model, unit }` structure
- **Action**: Replace `manifestUpdate()` with `componentDescriptorUpdate()`
- **Future CD PDCA**: Identify and remove all obsolete Manifest code
