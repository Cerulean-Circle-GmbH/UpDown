## 📋 PLAN

### Problem Statement

The E2E test (Test23_ImageDropE2E) fails because:
1. **Wrong route**: `/once` serves a minimal kernel boot page (just status text), not the full SPA
2. **No folder-overview**: Expected `<folder-overview>` showing assets directory doesn't exist yet
3. **SPA not loading**: Playwright headless browser shows static HTML, not dynamic Lit components

### Expected Test Flow

1. Navigate to a route with `<folder-overview>`
2. Show `/EAMD.ucp/components/ONCE/0.3.22.1/src/assets/*`
3. Drop `ServiceGlobe.jpg` onto the view
4. Verify Unit scenario created on server
5. Verify image displayed in `<image-default-view>`

### Blockers Identified

| ID | Blocker | Status | Solution |
|----|---------|--------|----------|
| B1 | No route shows folder-overview | 🔴 OPEN | Add `/files` client route to UcpRouter |
| B2 | Playwright used wrong route `/once` | ✅ FIXED | Use `/` or `/demo` instead |
| B3 | Playwright snapshots empty | 🟢 RESOLVED | SPA works, snapshots limited by tool |
| B4 | No assets folder route | 🔴 OPEN | Register `/files` → `<folder-over-view>` |

### Root Cause Analysis

The Playwright test was navigating to `/once` which serves a **minimal kernel boot page** with no SPA views. The correct routes are `/` or `/demo` which serve the full SPA with UcpRouter.
