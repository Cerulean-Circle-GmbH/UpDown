## Research Findings (Pre-Plan)

### Current State (ONCE 0.3.22.1)

```json
// ONCE.component.json (generated by inline DefaultWeb4TSComponent)
{
  "ior": { "uuid": "...", "component": "ONCE", "version": "0.3.22.1" },
  "owner": "Base64(Scenario<UserModel>)",  // ✅ Fixed in CD.3
  "model": {
    "units": ["/EAMD.ucp/...", ...]  // ✅ Flat IOR paths (502 items)
    // ❌ MISSING: unit attribute (Scenario<UnitModel> for descriptor itself)
  }
  // ❌ MISSING: unit attribute at root level
}
```

### Target State (Web4TSComponent descriptor format)

```json
// {Name}.component.json (generated by real Web4TSComponent)
{
  "ior": { "uuid": "...", "component": "...", "version": "..." },
  "owner": "Base64(Scenario<UserModel>)",
  "model": {
    // ... component model fields
    "implementationClassName": "..."
  },
  "unit": {  // ← MISSING: Fourth attribute
    "ior": { "uuid": "...", "component": "Unit", "version": "0.3.0.5" },
    "owner": "...",
    "model": {
      "uuid": "...",
      "name": "ONCE.component.json",
      "origin": "ior:local:file://.../ONCE.component.json",
      "definition": "...",
      "typeM3": "CLASS",
      "indexPath": "scenarios/index/a/b/uuid.scenario.json",
      "references": [...],
      "createdAt": "...",
      "updatedAt": "..."
    }
  }
}
```

### Scenario Index Structure

```
scenarios/
├── index/
│   ├── a/b/{uuid}.scenario.json  ← Primary storage (by UUID prefix)
│   └── ...
└── type/
    └── ONCE/
        └── 0.3.22.1/
            ├── {uuid1}.scenario.json → ../../../index/a/b/{uuid1}.scenario.json
            ├── {uuid2}.scenario.json → ../../../index/c/d/{uuid2}.scenario.json
            └── ...  ← Symlinks to index
```

### DefaultUnit Key Methods

From `Unit/0.3.0.5/src/ts/layer2/DefaultUnit.ts`:
- `toScenario()` — Creates `Scenario<UnitModel>` from unit state
- `storage.saveScenario(uuid, scenario, symlinkPaths)` — Saves to index with symlinks
- `createFromFilesystemEntity()` — Creates unit from file/folder
