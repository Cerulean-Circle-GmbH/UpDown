# PDCA: Fix Demo Hub Server Count - Notify Primary on Client Shutdown

**CMM Level**: CMM3 (Defined Process with PDCA)

## PLAN

### Problem Statement
The demo hub shows 45 servers running when only 5 processes exist. Root cause analysis reveals:

1. **Test servers don't unregister from primary when stopped**
   - Tests call `await client.stopServer()` internally
   - This closes HTTP server and updates scenario to `SHUTDOWN`
   - BUT doesn't notify primary server to remove from `serverRegistry`

2. **Primary's `serverRegistry` accumulates dead servers**
   - `/servers` endpoint returns all registered servers
   - Dead servers remain in registry until housekeeping runs
   - Demo hub fetches from `/servers` every 10s and shows stale data

3. **Housekeeping only runs when explicitly triggered**
   - Not automatic on server startup
   - Tests don't trigger housekeeping
   - Dead server entries accumulate indefinitely

### Root Cause Analysis
**5 Why Analysis**:
1. **Why** does demo hub show 45 servers? → `/servers` endpoint returns 45 entries
2. **Why** does `/servers` return 45? → `serverRegistry` has 45 entries
3. **Why** does registry have 45? → Servers register but never unregister
4. **Why** don't they unregister? → `stopServer()` doesn't notify primary
5. **Why** doesn't it notify? → CMM1 oversight - only `/stop` HTTP endpoint unregisters

**CMM3 Checklist Review**:
- ❌ **Lifecycle Management**: Client shutdown doesn't complete full unregistration workflow
- ❌ **State Synchronization**: Primary doesn't know when clients stop internally
- ❌ **Housekeeping**: Not automatic, requires manual trigger
- ✅ **Scenario Persistence**: Shutdown state IS saved to scenario
- ❌ **Test Cleanup**: Tests don't trigger housekeeping after stopping servers

### Impact Assessment
- **Severity**: HIGH - Demo hub unusable, shows false server counts
- **Scope**: All client server shutdowns (tests + manual stops)
- **User Impact**: Confusing UI, can't trust server counts
- **Technical Debt**: Memory leak in `serverRegistry` Map

### Solution Design

#### Option A: Client Notifies Primary on Shutdown ✅
**Approach**:
- In `stopServer()`, if client server, notify primary via WebSocket or HTTP
- Send unregister message before closing connections
- Primary removes from registry immediately

**Pros**:
- ✅ Immediate cleanup, no delay
- ✅ Works for all shutdown scenarios (graceful, crash recovery handled separately)
- ✅ Follows publish-subscribe pattern
- ✅ No polling required

**Cons**:
- Requires network call during shutdown (could fail if primary down)

#### Option B: Automatic Housekeeping on Primary Startup
**Approach**:
- Run housekeeping automatically when primary starts
- Check all registered servers' health
- Remove dead ones from registry

**Pros**:
- ✅ Catches all dead servers, even from crashes
- ✅ No client-side changes needed

**Cons**:
- ❌ Only runs on primary restart
- ❌ Dead servers accumulate between restarts
- ❌ Adds startup delay

#### Option C: Periodic Automatic Housekeeping
**Approach**:
- Run housekeeping every N minutes automatically
- Check server health and clean up dead entries

**Pros**:
- ✅ Automatic cleanup without manual trigger
- ✅ Catches crashes and unclean shutdowns

**Cons**:
- ❌ Delay between shutdown and cleanup (up to N minutes)
- ❌ Unnecessary overhead if no dead servers

**Decision**: **Implement A + B**
- **Option A** for immediate cleanup on graceful shutdown
- **Option B** for crash recovery and startup cleanup

async stopServer(): Promise<void> {
    this.serverModel.state = LifecycleState.STOPPING;
    await this.updateScenarioState(LifecycleState.STOPPING);

    // ✅ NEW: Notify primary server to unregister this client
    if (!this.serverModel.isPrimaryServer && this.primaryWsClient) {
        try {
            const unregisterMsg = {
                type: 'server-unregister',
                data: {
                    uuid: this.serverModel.uuid,
                    port: this.serverModel.capabilities.find(c => c.capability === 'httpPort')?.port
                }
            };
            this.primaryWsClient.send(JSON.stringify(unregisterMsg));
            // Give primary time to process
            await new Promise(resolve => setTimeout(resolve, 100));
        } catch (error) {
            console.warn('⚠️  Failed to notify primary of shutdown:', error);
        }
    }

    // ... rest of stopServer() logic
}
