# CMM3 PDCA: Fix Domain/Hostname Structure in Scenario Paths
**Date**: 2025-11-17  
**Process**: 1f (Complete PDCA Cycle)  
**Version**: 0.3.20.5  
**Issue**: Domain and hostname are reversed in scenario file paths

## 1f.1 PLAN: Define Problem & Solution

### Problem Statement

**Current (WRONG)**:
- FQDN: `McDonges-3.fritz.box`
- Detection reverses entire string: `box.fritz.McDonges-3`
- Scenario path: `scenarios/box.fritz.McDonges-3/ONCE/0.3.20.5/`
- This mixes domain and hostname into a single reversed string

**Correct Interpretation**:
- FQDN: `McDonges-3.fritz.box`
- Hostname: `McDonges-3` (first component)
- Domain parts: `box`, `fritz` (remaining components in reverse order)
- Scenario path: `scenarios/box/fritz/McDonges-3/ONCE/0.3.20.5/`

### Root Cause

The `detectDomain()` method in `ServerHierarchyManager.ts` currently:
1. Takes the full hostname (e.g., `McDonges-3.fritz.box`)
2. Reverses ALL parts: `['McDonges-3', 'fritz', 'box'].reverse()` → `['box', 'fritz', 'McDonges-3']`
3. Joins with dots: `box.fritz.McDonges-3`
4. Uses this as domain name in paths

This violates the semantic separation of:
- **Domain**: The network domain (organization/location)
- **Hostname**: The specific machine identifier

### Impact Analysis

**Files Affected**:

1. **Source Code**:
   - `src/ts/layer2/ServerHierarchyManager.ts`:
     - `detectDomain()` - needs to extract domain only, not full reversed hostname
     - `detectHostname()` - already correct, returns full FQDN
     - `loadOrCreateScenario()` - uses domain in path construction
     - `performHousekeeping()` - scans scenario directories
     - `updateScenarioState()` - saves to scenario path
     - `ensureCapabilitySymlink()` - creates symlinks in scenario path
   
   - `src/ts/layer2/ScenarioManager.ts`:
     - `loadScenarioByUUID()` - constructs path with domain
   
   - `src/ts/layer3/ONCEServerModel.ts`:
     - Model stores both `domain` and `host` fields

2. **Tests**:
   - `test/lifecycle-management.test.ts` - domain detection in `beforeEach`
   - `test/primary-housekeeping.test.ts` - domain detection in `beforeEach`
   - All tests that verify scenario file locations

3. **Existing Scenario Files**:
   - Files under `scenarios/box.fritz.McDonges-3/` need migration to `scenarios/box/fritz/McDonges-3/`
   - Files under `scenarios/local.once/` need migration to `scenarios/local/once/localhost/`

### Design Decision

**New Structure**:
```
scenarios/
  {domain-part-n}/    # e.g., "box" (TLD)
    {domain-part-n-1}/# e.g., "fritz" (subdomain)
      ...             # Additional domain levels as needed
        {hostname}/   # e.g., "McDonges-3" (machine name)
          ONCE/
            {version}/
              uuid.scenario.json
              capability/
                httpPort/
                  {port}/
                    uuid.scenario.json -> ../../../../../uuid.scenario.json
```

**Special Cases**:
1. **FQDN** (e.g., `McDonges-3.fritz.box`):
   - Hostname: `McDonges-3` (first part)
   - Domain parts: `box`, `fritz` (remaining parts, reversed)
   - Path: `scenarios/box/fritz/McDonges-3/ONCE/`

2. **localhost**:
   - Hostname: `localhost`
   - Domain parts: `local`, `once` (default)
   - Path: `scenarios/local/once/localhost/ONCE/`

3. **Simple hostname** (e.g., `myserver`):
   - Hostname: `myserver`
   - Domain parts: `local`, `myserver`
   - Path: `scenarios/local/myserver/myserver/ONCE/`

## 1f.2 DO: Implement Solution

### Changes to Make

**1. `ServerHierarchyManager.ts`**:

```typescript
/**
 * Detect domain from FQDN (without hostname) as separate path components
 * Examples:
 *   - "McDonges-3.fritz.box" -> domain path: ["box", "fritz"]
 *   - "localhost" -> domain path: ["local", "once"]
 *   - "myserver" -> domain path: ["local", "myserver"]
 */
private detectDomainPath(): string[] {
    const fqdn = this.detectHostname(); // Returns full FQDN from OS
    
    // Special case: localhost
    if (fqdn === 'localhost') {
        return ['local', 'once']; // Default domain path
    }
    
    // If FQDN has dots, extract domain parts (all except first)
    if (fqdn.includes('.')) {
        const parts = fqdn.split('.');
        const domainParts = parts.slice(1); // Skip hostname (first part)
        return domainParts.reverse(); // Reverse for proper domain hierarchy
    }
    
    // Simple hostname without domain
    return ['local', fqdn];
}

/**
 * Get domain as string (for model storage, backward compatibility)
 */
private detectDomain(): string {
    return this.detectDomainPath().join('.');
}

/**
 * Extract hostname from FQDN (first component only)
 * Examples:
 *   - "McDonges-3.fritz.box" -> hostname: "McDonges-3"
 *   - "localhost" -> hostname: "localhost"
 *   - "myserver" -> hostname: "myserver"
 */
private extractHostname(): string {
    const fqdn = this.detectHostname(); // Returns full FQDN from OS
    
    // If FQDN has dots, return only first part
    if (fqdn.includes('.')) {
        return fqdn.split('.')[0];
    }
    
    // Otherwise return as-is
    return fqdn;
}
```

**2. Add hostname field to model**:
```typescript
this.serverModel = {
    ...defaultModel,
    uuid: uuidv4(),
    pid: process.pid,
    state: LifecycleState.CREATED,
    platform: this.detectEnvironment(),
    domain: this.detectDomain(),        // e.g., "box.fritz" (for backward compatibility)
    hostname: this.extractHostname(),    // e.g., "McDonges-3" (NEW)
    host: this.detectHostname(),        // e.g., "McDonges-3.fritz.box" (full FQDN)
    ip: this.detectIPAddress(),
    capabilities: [],
    isPrimaryServer: false
};
```

**3. Update path construction** (all scenario path locations):
```typescript
// OLD: scenarios/${domain}/ONCE/${version}
// NEW: scenarios/{domain-parts}/{hostname}/ONCE/${version}

const domainPath = this.detectDomainPath(); // ['box', 'fritz']
const hostname = this.serverModel.hostname || this.extractHostname();

const mainScenarioDir = path.join(
    this.projectRoot, 
    'scenarios',
    ...domainPath,              // Spread domain parts as separate directories
    hostname,
    'ONCE',
    this.version
);
```

**4. Update symlink relative paths** (deeper directory hierarchy):
```typescript
// OLD: ../../../../uuid.scenario.json (4 levels up from capability/httpPort/{port}/)
// NEW: ../../../../../uuid.scenario.json (6 levels up from scenarios/box/fritz/hostname/ONCE/version/capability/httpPort/port/)

const relativePathToMain = path.relative(
    path.dirname(capabilitySymlink),
    mainScenarioPath
);
```

**5. Update housekeeping** to scan `scenarios/{domain-parts}/{hostname}/ONCE/{version}/`
