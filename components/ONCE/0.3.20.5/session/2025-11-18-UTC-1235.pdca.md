# PDCA: Fix tests deleting entire scenario directory tree

**Date**: 2025-11-18 UTC 12:35  
**Version**: 0.3.20.5  
**Requirement**: Never delete scenario base directories - only individual files  
**Type**: Critical Bug Fix - Data Safety

---

## **📋 PLAN**

### **Problem Statement**

User correctly identified that tests were using `fs.rmSync(scenarioBaseDir, { recursive: true })` which deletes ENTIRE directory trees including:
- `scenarios/box/fritz/McDonges-3/ONCE/0.3.20.5/`
- All subdirectories
- ALL files from ANY component

**Why this is dangerous**:
- ✅ Multiple components may store data in parallel paths
- ✅ Different databases may coexist in scenarios directory
- ✅ Deleting directory tree destroys OTHER components' data
- ✅ Tests should only clean up THEIR OWN files

### **Root Cause**

**Files with dangerous deletions**:
1. `test/primary-housekeeping.test.ts` (line 161): `fs.rmSync(scenarioBaseDir, { recursive: true })`
2. `test/primary-housekeeping.test.ts` (line 133): `fs.rmSync(capabilityDir, { recursive: true })`
3. `test/lifecycle-management.test.ts` (line 193): `fs.rmSync(scenarioBaseDir, { recursive: true })`
4. `test/lifecycle-management.test.ts` (line 186): `fs.rmSync(capabilityDir, { recursive: true })`

**Impact**: Tests could destroy persistent data from other components/databases sharing the scenarios directory.

### **Solution Design**

**Replace recursive directory deletion with selective file deletion**:

```typescript
// ❌ DANGEROUS - Deletes everything:
fs.rmSync(scenarioBaseDir, { recursive: true, force: true });

// ✅ SAFE - Only deletes test files:
const files = fs.readdirSync(scenarioBaseDir).filter(f => f.endsWith('.scenario.json'));
for (const file of files) {
    fs.unlinkSync(path.join(scenarioBaseDir, file));
}
// Clean symlinks in capability directories
// Remove only empty port directories after cleaning
```

### **Lessons Learned**

**What Could Have Gone Wrong**:
- Multiple components storing persistent data in `scenarios/` directory
- Different databases coexisting (e.g., `scenarios/domain/host/ComponentA/`, `scenarios/domain/host/ComponentB/`)
- Test runs destroying production data
- Loss of persistent configuration/state

**Why This Matters** (Web4 Multi-Component Architecture):
- ✅ Components must coexist peacefully in shared directories
- ✅ Each component responsible for ITS OWN files only
- ✅ Never assume you own an entire directory tree
- ✅ Use selective deletion, not recursive

**Rule**: **NEVER use `rmSync(recursive:true)` or `rm -rf` on shared directories**
