### **Problem Statement**

**Current Architecture**: ServerHierarchyManager is written in **functional/procedural style** with:
- ❌ 1497 lines in a single file
- ❌ Direct `require()` and `import` calls scattered throughout
- ❌ File system operations (`fs`, `path`) used directly everywhere
- ❌ HTTP requests using raw `http` module in methods
- ❌ WebSocket handling mixed with business logic
- ❌ Tight coupling to Node.js APIs (`os`, `child_process`, `fs`)
- ❌ Hard to test (requires actual filesystem, network, OS)
- ❌ Hard to mock dependencies
- ❌ Violates Dependency Inversion Principle

**Example Violations**:

```typescript
// ❌ FUNCTIONAL/PROCEDURAL STYLE:
private detectHostname(): string {
    return os.hostname();  // Direct dependency on os module
}

private detectIPAddress(): string {
    const networkInterfaces = os.networkInterfaces();  // Direct dependency
    // ... 20 lines of imperative logic
}

private async performHousekeeping(): Promise<{deleted: number, discovered: number}> {
    const scenarioBaseDir = path.join(...);  // Direct path manipulation
    const items = fs.readdirSync(dir);  // Direct fs dependency
    const stat = fs.lstatSync(fullPath);  // More fs coupling
    // ... 200+ lines of imperative logic mixed with I/O
}
```

### **TRUE Radical OOP Principles Violated**

**1. Model-Driven Design** ❌
- Should have: All state in models, all logic operates on models
- Actually have: Scattered state, imperative I/O mixed with logic

**2. Dependency Injection** ❌
- Should have: Dependencies passed via constructor
- Actually have: Hard-coded `require()` and `import` dependencies

**3. Single Responsibility** ❌
- Should have: One class = one responsibility
- Actually have: ServerHierarchyManager does EVERYTHING (hierarchy, housekeeping, HTTP, WebSocket, file I/O, scenario management, port management)

**4. Testability** ❌
- Should have: Easy to unit test with mocks
- Actually have: Requires real filesystem, network, OS APIs

**5. Separation of Concerns** ❌
- Should have: Infrastructure (I/O) separated from domain logic
- Actually have: I/O operations mixed into every method

### **TRUE Radical OOP Solution Design**

**Core Principle**: **"Empty Constructor + Scenario Init + Model-Driven + Dependency Injection"**

**Web4 EAM 5-Layer Architecture**:

```
EAM Layer 1: Infrastructure
├── NodeOSInfrastructure (os.hostname(), os.networkInterfaces())
├── NodeFileSystemInfrastructure (fs, path operations)
├── NodeNetworkInfrastructure (http, ws servers)
├── NodeProcessInfrastructure (child_process)
└── All physical/platform dependencies isolated here

EAM Layer 2: Implementation (Default Classes)
├── DefaultONCEServer (main implementation)
├── DefaultScenarioManager (scenario persistence)
├── DefaultHousekeeping (housekeeping logic)
├── DefaultServerLifecycle (lifecycle management)
└── All concrete algorithm implementations

EAM Layer 3: Interface/Service Contracts
├── ONCEServer interface (service contract)
├── ScenarioManager interface (capability contract)
├── HousekeepingService interface (service contract)
├── ServerLifecycle interface (capability contract)
└── All type contracts, no implementation

EAM Layer 4: Business Processes
├── ServerStartupProcess (workflow)
├── ServerShutdownProcess (rollbackable transaction chain)
├── HousekeepingProcess (automated workflow)
└── All transaction chains with rollback capability

EAM Layer 5: User Experience
├── CLI (command-line interface)
├── Demo Hub (web UI)
├── Browser Client (interactive UI)
└── All user/agent interaction points
```

/**
 * DefaultONCEServer - Main server implementation
 * Web4 Pattern: Empty constructor, scenario-driven state
 */
class DefaultONCEServer implements ONCEServer {
    model: ONCEServerModel;
    
    // Infrastructure dependencies (injected via scenario)
    private infrastructure?: {
        os: NodeOSInfrastructure;
        fs: NodeFileSystemInfrastructure;
        network: NodeNetworkInfrastructure;
    };
    
    // Service dependencies (injected via scenario)
    private scenarioManager?: ScenarioManager;
    private housekeepingService?: HousekeepingService;
    
    // ✅ Web4 Pattern: Empty Constructor
    constructor() {
        this.model = {
            uuid: '',
            domain: '',
            hostname: '',
            host: '',
            ip: '',
            port: 0,
            lifecycle: LifecycleState.CREATED,
            capabilities: []
        };
    }
    
    // ✅ Web4 Pattern: Scenario Initialization
    init(scenario: ONCEServerScenario): this {
        // Restore model from scenario
        this.model = { ...scenario.model };
        
        // Inject infrastructure (Layer 1)
        if (scenario.infrastructure) {
            this.infrastructure = {
                os: new NodeOSInfrastructure().init(scenario.infrastructure.os),
                fs: new NodeFileSystemInfrastructure().init(scenario.infrastructure.fs),
                network: new NodeNetworkInfrastructure().init(scenario.infrastructure.network)
            };
        }
        
        // Inject services (Layer 3)
        if (scenario.services) {
            this.scenarioManager = new DefaultScenarioManager().init(scenario.services.scenarioManager);
            this.housekeepingService = new DefaultHousekeepingService().init(scenario.services.housekeeping);
        }
        
        return this;
    }
    
    async start(): Promise<this> {
        // 1. Detect environment (via infrastructure)
        if (this.infrastructure) {
            const env = await this.infrastructure.os.detectEnvironment();
            this.model.hostname = env.hostname;
            this.model.ip = env.primaryIP;
            this.model.domain = this.parseDomain(env.fqdn);
        }
        
        // 2. Update lifecycle
        this.model.lifecycle = LifecycleState.STARTING;
        
        // 3. Start HTTP/WebSocket servers (via infrastructure)
        if (this.infrastructure) {
            await this.infrastructure.network.startHTTPServer(this.model.port);
            await this.infrastructure.network.startWebSocketServer();
        }
        
        // 4. Save scenario (via service)
        if (this.scenarioManager) {
            await this.scenarioManager.saveScenario(this.model);
        }
        
        // 5. Perform housekeeping (via service)
        await this.performHousekeeping();
        
        // 6. Update lifecycle
        this.model.lifecycle = LifecycleState.RUNNING;
        
        return this;
    }
    
    async stop(): Promise<this> {
        this.model.lifecycle = LifecycleState.STOPPING;
        
        // Stop infrastructure
        if (this.infrastructure) {
            await this.infrastructure.network.stopAllServers();
        }
        
        // Delete scenario
        if (this.scenarioManager) {
            await this.scenarioManager.deleteScenario(this.model.uuid);
        }
        
        this.model.lifecycle = LifecycleState.STOPPED;
        return this;
    }
    
    async performHousekeeping(): Promise<this> {
        if (this.housekeepingService) {
            const results = await this.housekeepingService.cleanup();
            console.log(`✅ Housekeeping complete: ${results.deleted} deleted, ${results.discovered} discovered`);
        }
        return this;
    }
    
    // ✅ Web4 Pattern: Scenario Hibernation
    toScenario(): ONCEServerScenario {
        return {
            model: { ...this.model },
            infrastructure: this.infrastructure ? {
                os: this.infrastructure.os.toScenario(),
                fs: this.infrastructure.fs.toScenario(),
                network: this.infrastructure.network.toScenario()
            } : undefined,
            services: {
                scenarioManager: this.scenarioManager?.toScenario(),
                housekeeping: this.housekeepingService?.toScenario()
            }
        };
    }
    
    private parseDomain(fqdn: string): string {
        // Pure logic - no I/O
        if (fqdn === 'localhost') return 'local.once';
        if (fqdn.includes('.')) {
            const parts = fqdn.split('.');
            return parts.slice(1).reverse().join('.');
        }
        return `local.${fqdn}`;
    }
}

/**
 * DefaultScenarioManager - Scenario persistence implementation
 */
class DefaultScenarioManager implements ScenarioManager {
    model: ScenarioManagerModel;
    private fs?: NodeFileSystemInfrastructure;
    
    constructor() {
        this.model = { projectRoot: '', version: '' };
    }
    
    init(scenario: ScenarioManagerScenario): this {
        this.model = { ...scenario.model };
        if (scenario.infrastructure?.fs) {
            this.fs = new NodeFileSystemInfrastructure().init(scenario.infrastructure.fs);
        }
        return this;
    }
    
    async saveScenario(serverModel: ONCEServerModel): Promise<this> {
        if (!this.fs) throw new Error('FileSystem infrastructure not initialized');
        
        // Build path (pure logic)
        const scenarioPath = this.buildScenarioPath(serverModel);
        
        // Ensure directory exists (via infrastructure)
        await this.fs.ensureDirectory(path.dirname(scenarioPath));
        
        // Write file (via infrastructure)
        const content = JSON.stringify(serverModel, null, 2);
        await this.fs.writeFile(scenarioPath, content);
        
        // Create symlink for capability indexing
        const symlinkPath = this.buildCapabilitySymlinkPath(serverModel);
        await this.fs.createSymlink(scenarioPath, symlinkPath);
        
        return this;
    }
    
    async loadScenario(uuid: string): Promise<ONCEServerModel | null> {
        // Implementation
        return null;
    }
    
    async deleteScenario(uuid: string): Promise<this> {
        // Implementation
        return this;
    }
    
    toScenario(): ScenarioManagerScenario {
        return {
            model: { ...this.model },
            infrastructure: this.fs ? { fs: this.fs.toScenario() } : undefined
        };
    }
    
    private buildScenarioPath(serverModel: ONCEServerModel): string {
        // Pure function - domain logic
        const domainParts = serverModel.domain.split('.').reverse();
        return path.join(
            this.model.projectRoot,
            'scenarios',
            ...domainParts,
            serverModel.hostname,
            'ONCE',
            this.model.version,
            `${serverModel.uuid}.scenario.json`
        );
    }
    
    private buildCapabilitySymlinkPath(serverModel: ONCEServerModel): string {
        // Pure function - capability indexing logic
        const domainParts = serverModel.domain.split('.').reverse();
        return path.join(
            this.model.projectRoot,
            'scenarios',
            ...domainParts,
            serverModel.hostname,
            'ONCE',
            this.model.version,
            'capability',
            'httpPort',
            serverModel.port.toString(),
            `${serverModel.uuid}.scenario.json`
        );
    }
}

/**
 * NodeOSInfrastructure - Encapsulates Node.js os module
 */
class NodeOSInfrastructure {
    private model: { hostname: string; fqdn: string; primaryIP: string };
    
    constructor() {
        this.model = { hostname: '', fqdn: '', primaryIP: '' };
    }
    
    init(scenario: any): this {
        this.model = { ...scenario };
        return this;
    }
    
    async detectEnvironment(): Promise<EnvironmentModel> {
        // ✅ ONLY place where os module is imported
        const os = await import('os');
        
        const fqdn = os.hostname();
        const hostname = fqdn.split('.')[0];
        const primaryIP = this.detectPrimaryIP(os);
        
        this.model = { hostname, fqdn, primaryIP };
        
        return { hostname, fqdn, primaryIP };
    }
    
    private detectPrimaryIP(os: any): string {
        const interfaces = os.networkInterfaces();
        for (const name of Object.keys(interfaces)) {
            for (const iface of interfaces[name]) {
                if (iface.family === 'IPv4' && !iface.internal) {
                    return iface.address;
                }
            }
        }
        return '127.0.0.1';
    }
    
    toScenario(): any {
        return { ...this.model };
    }
}

/**
 * NodeFileSystemInfrastructure - Encapsulates Node.js fs/path modules
 */
class NodeFileSystemInfrastructure {
    constructor() {}
    
    init(scenario: any): this {
        return this;
    }
    
    async writeFile(path: string, content: string): Promise<void> {
        // ✅ ONLY place where fs is imported
        const fs = await import('fs/promises');
        await fs.writeFile(path, content, 'utf-8');
    }
    
    async readFile(path: string): Promise<string> {
        const fs = await import('fs/promises');
        return fs.readFile(path, 'utf-8');
    }
    
    async deleteFile(path: string): Promise<void> {
        const fs = await import('fs/promises');
        await fs.unlink(path);
    }
    
    async ensureDirectory(dirPath: string): Promise<void> {
        const fs = await import('fs/promises');
        await fs.mkdir(dirPath, { recursive: true });
    }
    
    async createSymlink(target: string, linkPath: string): Promise<void> {
        const fs = await import('fs/promises');
        const path = await import('path');
        
        // Ensure symlink directory exists
        await this.ensureDirectory(path.dirname(linkPath));
        
        // Create symlink
        try {
            await fs.symlink(target, linkPath);
        } catch (err: any) {
            if (err.code !== 'EEXIST') throw err;
        }
    }
    
    toScenario(): any {
        return {};
    }
}

### **QA Decisions**

**D1: Use Web4 EAM 5-Layer Architecture (NOT Hexagonal/Ports & Adapters)**
- **Why**: Web4 has a standardized 5-layer model aligned with Enterprise Architecture Management
- **Alternative**: Hexagonal Architecture (too generic, not Web4-specific)
- **Decision**: EAM layers provide clear separation and align with Web4 principles

**D2: Web4 Empty Constructor + Scenario Init Pattern (NOT Constructor Injection)**
- **Why**: Web4 mandates `constructor(){}` with `init(scenario)` for hibernation/restoration
- **Alternative**: Traditional Dependency Injection via constructor params (violates Web4)
- **Decision**: Empty constructor enables complete scenario serialization/deserialization

**D3: Infrastructure Encapsulation in Layer 1 (NOT Scattered Imports)**
- **Why**: All platform dependencies (Node.js APIs) isolated in one layer
- **Alternative**: Imports scattered throughout (current problem)
- **Decision**: Layer 1 is ONLY place where `os`, `fs`, `http`, etc. are imported

**D4: Scenario-Driven Dependency Injection (NOT Runtime Service Locator)**
- **Why**: Dependencies injected via scenario JSON enables testability and portability
- **Alternative**: Service Locator pattern (hidden dependencies)
- **Decision**: Scenario contains complete infrastructure + service configuration
