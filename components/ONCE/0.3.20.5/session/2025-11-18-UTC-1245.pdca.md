### **Problem Statement**

**Current Architecture**: ServerHierarchyManager is written in **functional/procedural style** with:
- ❌ 1497 lines in a single file
- ❌ Direct `require()` and `import` calls scattered throughout
- ❌ File system operations (`fs`, `path`) used directly everywhere
- ❌ HTTP requests using raw `http` module in methods
- ❌ WebSocket handling mixed with business logic
- ❌ Tight coupling to Node.js APIs (`os`, `child_process`, `fs`)
- ❌ Hard to test (requires actual filesystem, network, OS)
- ❌ Hard to mock dependencies
- ❌ Violates Dependency Inversion Principle

**Example Violations**:

```typescript
// ❌ FUNCTIONAL/PROCEDURAL STYLE:
private detectHostname(): string {
    return os.hostname();  // Direct dependency on os module
}

private detectIPAddress(): string {
    const networkInterfaces = os.networkInterfaces();  // Direct dependency
    // ... 20 lines of imperative logic
}

private async performHousekeeping(): Promise<{deleted: number, discovered: number}> {
    const scenarioBaseDir = path.join(...);  // Direct path manipulation
    const items = fs.readdirSync(dir);  // Direct fs dependency
    const stat = fs.lstatSync(fullPath);  // More fs coupling
    // ... 200+ lines of imperative logic mixed with I/O
}
```

### **TRUE Radical OOP Principles Violated**

**1. Model-Driven Design** ❌
- Should have: All state in models, all logic operates on models
- Actually have: Scattered state, imperative I/O mixed with logic

**2. Dependency Injection** ❌
- Should have: Dependencies passed via constructor
- Actually have: Hard-coded `require()` and `import` dependencies

**3. Single Responsibility** ❌
- Should have: One class = one responsibility
- Actually have: ServerHierarchyManager does EVERYTHING (hierarchy, housekeeping, HTTP, WebSocket, file I/O, scenario management, port management)

**4. Testability** ❌
- Should have: Easy to unit test with mocks
- Actually have: Requires real filesystem, network, OS APIs

**5. Separation of Concerns** ❌
- Should have: Infrastructure (I/O) separated from domain logic
- Actually have: I/O operations mixed into every method

### **TRUE Radical OOP Solution Design**

**Core Principle**: **"Model contains all state, Infrastructure handles I/O, Domain contains pure logic"**

**Architecture Layers**:

```
Layer 1: Models (Pure Data)
├── ServerModel
├── ScenarioModel
├── NetworkModel
├── FileSystemModel
└── EnvironmentModel

Layer 2: Infrastructure (I/O Adapters)
├── OSAdapter (hostname, IP detection)
├── FileSystemAdapter (read, write, delete)
├── NetworkAdapter (HTTP, WebSocket)
├── ProcessAdapter (spawn, kill)
└── PathAdapter (path manipulation)

Layer 3: Domain Logic (Pure Business Logic)
├── HostnameDetector (operates on EnvironmentModel)
├── ScenarioManager (operates on ScenarioModel)
├── HousekeepingService (operates on models, uses adapters)
└── ServerLifecycleManager (operates on ServerModel)

Layer 4: Application (Composition)
└── ONCEServer (composes everything with DI)
```

// LAYER 1: MODELS (Pure Data Structures)
// ========================================

interface EnvironmentModel {
    hostname: string;
    ip: string;
    networkInterfaces: NetworkInterfaceModel[];
}

interface NetworkInterfaceModel {
    name: string;
    family: 'IPv4' | 'IPv6';
    address: string;
    internal: boolean;
}

interface ServerModel {
    uuid: string;
    domain: string;
    hostname: string;
    host: string;
    ip: string;
    port: number;
    state: LifecycleState;
    capabilities: CapabilityModel[];
}

interface ScenarioModel {
    uuid: string;
    path: string;
    content: any;
    exists: boolean;
}

// ========================================
// LAYER 2: PORTS (Interfaces for Adapters)
// ========================================

interface OSPort {
    getEnvironment(): Promise<EnvironmentModel>;
}

interface FileSystemPort {
    readFile(path: string): Promise<string>;
    writeFile(path: string, content: string): Promise<void>;
    deleteFile(path: string): Promise<void>;
    createDirectory(path: string): Promise<void>;
    exists(path: string): Promise<boolean>;
    listDirectory(path: string): Promise<string[]>;
    getStats(path: string): Promise<FileStatsModel>;
}

interface NetworkPort {
    startHTTPServer(port: number): Promise<HTTPServerModel>;
    startWebSocketServer(httpServer: HTTPServerModel): Promise<WebSocketServerModel>;
    makeHTTPRequest(options: HTTPRequestOptions): Promise<HTTPResponseModel>;
}

interface PathPort {
    join(...paths: string[]): string;
    dirname(path: string): string;
    basename(path: string): string;
    relative(from: string, to: string): string;
}

// ========================================
// LAYER 3: DOMAIN LOGIC (Pure Functions on Models)
// ========================================

class HostnameDetector {
    constructor(
        private osPort: OSPort,
        private pathPort: PathPort
    ) {}
    
    async detectHostnameDetails(): Promise<HostnameDetailsModel> {
        const env = await this.osPort.getEnvironment();
        
        // Pure logic - no I/O
        const fqdn = env.hostname;
        const parts = this.parseFQDN(fqdn);
        
        return {
            fqdn: fqdn,
            hostname: parts.hostname,
            domain: parts.domain,
            domainPath: parts.domainPath
        };
    }
    
    private parseFQDN(fqdn: string): ParsedFQDN {
        // Pure function - no side effects
        if (fqdn === 'localhost') {
            return {
                hostname: 'localhost',
                domain: 'local.once',
                domainPath: ['local', 'once']
            };
        }
        
        if (fqdn.includes('.')) {
            const parts = fqdn.split('.');
            const hostname = parts[0];
            const domainParts = parts.slice(1);
            return {
                hostname,
                domain: domainParts.reverse().join('.'),
                domainPath: domainParts.reverse()
            };
        }
        
        return {
            hostname: fqdn,
            domain: `local.${fqdn}`,
            domainPath: ['local', fqdn]
        };
    }
}

class ScenarioService {
    constructor(
        private fsPort: FileSystemPort,
        private pathPort: PathPort
    ) {}
    
    async loadScenario(scenarioPath: string): Promise<ScenarioModel> {
        // Uses port for I/O
        const exists = await this.fsPort.exists(scenarioPath);
        if (!exists) {
            return { uuid: '', path: scenarioPath, content: null, exists: false };
        }
        
        const content = await this.fsPort.readFile(scenarioPath);
        const parsed = JSON.parse(content);
        
        return {
            uuid: parsed.uuid,
            path: scenarioPath,
            content: parsed,
            exists: true
        };
    }
    
    async saveScenario(scenario: ScenarioModel): Promise<void> {
        // Pure domain logic + port for I/O
        const directory = this.pathPort.dirname(scenario.path);
        
        // Ensure directory exists
        if (!await this.fsPort.exists(directory)) {
            await this.fsPort.createDirectory(directory);
        }
        
        // Write file
        const content = JSON.stringify(scenario.content, null, 2);
        await this.fsPort.writeFile(scenario.path, content);
    }
    
    buildScenarioPath(
        projectRoot: string,
        hostnameDetails: HostnameDetailsModel,
        version: string,
        uuid: string
    ): string {
        // Pure function using PathPort
        return this.pathPort.join(
            projectRoot,
            'scenarios',
            ...hostnameDetails.domainPath,
            hostnameDetails.hostname,
            'ONCE',
            version,
            `${uuid}.scenario.json`
        );
    }
}

class HousekeepingService {
    constructor(
        private fsPort: FileSystemPort,
        private pathPort: PathPort,
        private networkPort: NetworkPort
    ) {}
    
    async performHousekeeping(
        scenarioBaseDir: string
    ): Promise<HousekeepingResultModel> {
        // 1. Find scenarios (uses port)
        const scenarioFiles = await this.findScenarios(scenarioBaseDir);
        
        // 2. Process each (domain logic + port for I/O)
        const results = {
            deleted: 0,
            discovered: 0
        };
        
        for (const file of scenarioFiles) {
            const scenario = await this.loadAndCheckScenario(file);
            
            if (scenario.shouldDelete) {
                await this.fsPort.deleteFile(file);
                results.deleted++;
            } else if (scenario.shouldDiscover) {
                // Discover logic
                results.discovered++;
            }
        }
        
        return results;
    }
    
    private async findScenarios(dir: string): Promise<string[]> {
        // Recursive search using port
        const results: string[] = [];
        const items = await this.fsPort.listDirectory(dir);
        
        for (const item of items) {
            const fullPath = this.pathPort.join(dir, item);
            const stats = await this.fsPort.getStats(fullPath);
            
            if (stats.isDirectory) {
                const subResults = await this.findScenarios(fullPath);
                results.push(...subResults);
            } else if (item.endsWith('.scenario.json')) {
                results.push(fullPath);
            }
        }
        
        return results;
    }
}

        return '127.0.0.1';

    private mapNetworkInterfaces(interfaces: any): NetworkInterfaceModel[] {

}

class NodeFileSystemAdapter implements FileSystemPort {
    async readFile(path: string): Promise<string> {
        const fs = await import('fs/promises');
        return fs.readFile(path, 'utf-8');
    }
    
    async writeFile(path: string, content: string): Promise<void> {
        const fs = await import('fs/promises');
        await fs.writeFile(path, content, 'utf-8');
    }
    
    async deleteFile(path: string): Promise<void> {
        const fs = await import('fs/promises');
        await fs.unlink(path);
    }
    
    async exists(path: string): Promise<boolean> {
        const fs = await import('fs/promises');
        try {
            await fs.access(path);
            return true;
        } catch {
            return false;
        }
    }
    
    // ... other methods
}

class NodePathAdapter implements PathPort {
    private path: any;
    
    constructor() {
        this.path = require('path');  // Encapsulated, not spread everywhere
    }
    
    join(...paths: string[]): string {
        return this.path.join(...paths);
    }
    
    dirname(path: string): string {
        return this.path.dirname(path);
    }
    
    // ... other methods
}

// LAYER 5: APPLICATION (Composition Root)
// ========================================

class ONCEServer {
    private hostnameDetector: HostnameDetector;
    private scenarioService: ScenarioService;
    private housekeepingService: HousekeepingService;
    private serverModel: ServerModel;
    
    constructor(
        osPort: OSPort,
        fsPort: FileSystemPort,
        networkPort: NetworkPort,
        pathPort: PathPort
    ) {
        // ✅ Dependency Injection!
        this.hostnameDetector = new HostnameDetector(osPort, pathPort);
        this.scenarioService = new ScenarioService(fsPort, pathPort);
        this.housekeepingService = new HousekeepingService(fsPort, pathPort, networkPort);
        
        this.serverModel = this.createDefaultModel();
    }
    
    async start(): Promise<void> {
        // 1. Detect environment (via injected ports)
        const hostnameDetails = await this.hostnameDetector.detectHostnameDetails();
        
        // 2. Update model
        this.serverModel.domain = hostnameDetails.domain;
        this.serverModel.hostname = hostnameDetails.hostname;
        this.serverModel.host = hostnameDetails.fqdn;
        
        // 3. Start servers (via injected ports)
        // ...
        
        // 4. Perform housekeeping (via service)
        await this.housekeepingService.performHousekeeping('/path/to/scenarios');
        
        // 5. Save scenario (via service)
        const scenarioPath = this.scenarioService.buildScenarioPath(
            '/project/root',
            hostnameDetails,
            '0.3.20.5',
            this.serverModel.uuid
        );
        
        await this.scenarioService.saveScenario({
            uuid: this.serverModel.uuid,
            path: scenarioPath,
            content: this.serverModel,
            exists: false
        });
    }
    
    private createDefaultModel(): ServerModel {
        return {
            uuid: crypto.randomUUID(),
            domain: 'local.once',
            hostname: 'localhost',
            host: 'localhost',
            ip: '127.0.0.1',
            port: 42777,
            state: LifecycleState.CREATED,
            capabilities: []
        };
    }
}

**1. Testability** ✅

**3. Separation of Concerns** ✅
- Models: Pure data
- Ports: Interfaces (contracts)
- Adapters: I/O implementation
- Domain Services: Pure business logic
- Application: Composition

**4. Single Responsibility** ✅
- HostnameDetector: ONLY detects hostname
- ScenarioService: ONLY manages scenarios
- HousekeepingService: ONLY performs housekeeping
- Each class < 200 lines

**5. Dependency Inversion** ✅
- High-level modules depend on abstractions (Ports)
- Low-level modules implement abstractions (Adapters)
- No direct dependencies on concrete implementations

### **QA Decisions**

**D1: Use Hexagonal Architecture (Ports & Adapters)**
- **Why**: Clear separation between domain and infrastructure
- **Alternative**: Repository Pattern (too data-centric)
- **Decision**: Hexagonal better fits our multi-concern system

**D2: Async/Await for All I/O Operations**
- **Why**: All file and network I/O is inherently async
- **Alternative**: Sync operations (blocks event loop)
- **Decision**: Full async for scalability

**D3: Dependency Injection via Constructor**
- **Why**: Makes dependencies explicit and testable
- **Alternative**: Service Locator (hidden dependencies)
- **Decision**: Constructor injection for transparency
