**Current Issue**: Server logs are **not useful or readable** for debugging because:

1. **UUID Truncation in UI**: Demo hub shows `UUID: 4e683fdb...` instead of full UUID
2. **Missing Sender Context**: Logs don't show WHO is performing actions
3. **Too Verbose**: Repetitive "WebSocket connection established/closed" spam
4. **No Action Context**: "Broadcasting" doesn't say what is being broadcast or why
5. **Duplicate Logs**: "Scenario saved" logged twice for same operation

### **Root Cause Analysis**

**Why are logs not useful?**

1. **No Logging Standard**: Each log message written ad-hoc without pattern
2. **Missing Context**: Logs don't include WHO is performing action
3. **No UUID Prefix**: Can't trace actions to specific servers
4. **Duplicate Logging**: Same action logged at multiple layers
5. **WebSocket Noise**: Low-level connection events without context

**Where are the problems?**

From grep analysis:
- `ServerHierarchyManager.ts`: All the "Broadcasting", "Registering" logs
- `DefaultONCE.ts`: Scenario save logs
- Demo hub HTML: UUID truncation in UI

### **Solution Design**

**Pattern 1: Standard Log Format**
```typescript
console.log(`${emoji} [${shortUUID(this.model.uuid)}] ${action}: ${context}`);
```

**Pattern 2: Directional Logging for Communication**
```typescript
// Broadcast
console.log(`📤 [${shortUUID(sender)} → ${targetCount} ${targetType}] ${messageType}`);

// Acknowledgment
console.log(`✅ [${shortUUID(sender)} → ${shortUUID(receiver)}] Acknowledged: ${messageType}`);
```

**Pattern 3: Server Identity in Logs**
```typescript
const serverIdentity = `${this.model.hostname}:${this.model.port}`;
console.log(`📋 [${shortUUID(uuid)}] Client registered: ${serverIdentity} (${uuid})`);
```

**Pattern 4: Remove Noise**
- Remove generic "WebSocket connection established/closed"
- Only log meaningful events: "Browser client connected", "Server registered", etc.
- Consolidate duplicate logs

### **QA Decisions**

**D1: Use Short UUID (8 chars) in logs (NOT full 36 chars)**
- **Why**: `[fc1493ee]` is readable, `[fc1493ee-0814-4c4a-99b5-eba4dec32c15]` is not
- **Alternative**: Full UUID (too long)
- **Decision**: 8 chars for logs, full UUID in UI on hover/click

**D2: Include sender UUID in ALL server logs (NOT just some)**
- **Why**: Essential for tracing actions in multi-server environment
- **Alternative**: Only log UUID for some events (inconsistent)
- **Decision**: Every log line gets `[uuid]` prefix

**D3: Show full UUID in demo hub UI (NOT truncated)**
- **Why**: User explicitly requested, needed for debugging
- **Alternative**: Keep truncation (user complained)
- **Decision**: Show full UUID, use smaller font or monospace

**D4: Remove low-level WebSocket noise (NOT log every connection)**
- **Why**: Not useful without context, creates log spam
- **Alternative**: Keep all WebSocket logs (too verbose)
- **Decision**: Only log meaningful connection events with WHO

### **Phase 1: Create Logging Utilities**

#### **File: `components/ONCE/0.3.20.5/src/ts/utils/LoggingUtils.ts` (NEW)**

```typescript
/**
 * Logging utilities for consistent, context-aware logging
 * Web4 Pattern: Include sender UUID in all logs
 */

/**
 * Shorten UUID to first 8 characters for readable logs
 */
export function shortUUID(uuid: string): string {
    return uuid ? uuid.substring(0, 8) : 'unknown';
}

/**
 * Format server identity as hostname:port
 */
export function serverIdentity(hostname: string, port: number): string {
    return `${hostname}:${port}`;
}

/**
 * Standard log format: [uuid] action: context
 */
export function logAction(emoji: string, uuid: string, action: string, context: string): void {
    console.log(`${emoji} [${shortUUID(uuid)}] ${action}: ${context}`);
}

/**
 * Broadcast log: [sender → targets] message
 */
export function logBroadcast(
    senderUUID: string, 
    targetCount: number, 
    targetType: string, 
    messageType: string
): void {
    console.log(`📤 [${shortUUID(senderUUID)} → ${targetCount} ${targetType}] ${messageType}`);
}

/**
 * Acknowledgment log: [sender → receiver] Acknowledged: message
 */
export function logAcknowledgment(
    senderUUID: string, 
    receiverUUID: string, 
    messageType: string
): void {
    console.log(`✅ [${shortUUID(senderUUID)} → ${shortUUID(receiverUUID)}] Acknowledged: ${messageType}`);
}

/**
 * Registration log: [uuid] Server registered: identity (fullUUID)
 */
export function logRegistration(
    uuid: string, 
    hostname: string, 
    port: number
): void {
    console.log(`📋 [${shortUUID(uuid)}] Server registered: ${serverIdentity(hostname, port)} (${uuid})`);
}
```
