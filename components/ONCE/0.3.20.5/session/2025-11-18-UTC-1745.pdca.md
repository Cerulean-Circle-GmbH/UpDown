## PLAN

### Problem Statement
The current `message-exchange.test.ts` tests do not adequately verify:
1. **Dynamic primary server discovery** from `/health` endpoint (just implemented in browser client)
2. **Relay pattern initiated from client servers** (tests only verify primary → client broadcasts)
3. **Cross-server relay routing** (client A → primary → client B)
4. **Server hostname/domain** in message `from` fields (just added to browser client)

### Root Cause Analysis
**Current Test Coverage Gaps**:
1. ❌ No test for client server discovering primary address via `/health`
2. ❌ No test for relay message including `host` field in `from` object
3. ❌ Relay test only checks logs, not actual message routing
4. ❌ No verification of `/servers` endpoint requiring primary server address
5. ❌ Tests use `demoMessages` which internally creates messages, not via HTTP endpoints like browser does

**CMM3 Checklist Review**:
- ❌ **Test Coverage**: Missing tests for new relay functionality
- ❌ **Path Authority**: Tests don't verify dynamic discovery of primary address
- ✅ **Git Protocol**: Following proper commit message format
- ❌ **DRY**: Tests duplicate server startup logic instead of shared helpers
- ✅ **Radical OOP**: Tests use model-based message tracking

### Impact Assessment
- **Severity**: MEDIUM - Tests pass but don't cover critical new functionality
- **Scope**: Test suite only (`message-exchange.test.ts`)
- **User Impact**: Low (tests are internal), but risks regressions going undetected
- **Technical Debt**: Test gaps create false confidence in relay functionality
