# Scenario-Based Black-Box Test Pattern (Web4 Compliant)

**Date**: 2025-11-21 UTC 15:00  
**Component**: ONCE v0.3.21.2  
**Principle**: Environment Independent - NO `process.env`!

---

## Core Insight

**❌ WRONG** (Environment-dependent):
```typescript
// Violates Web4: Uses process.env
process.env.ONCE_PROJECT_ROOT = testDataDir;
execSync('./once startServer', { cwd: projectRoot });
```

**✅ CORRECT** (Scenario-based):
```typescript
// Web4 compliant: Scenario-based configuration
const testScenario: Scenario<CLIModel> = {
    ior: { uuid: uuidv4(), component: 'ONCE', version: '0.3.21.2' },
    owner: 'test-runner',
    model: {
        projectRoot: testDataDir, // ✅ Isolated test path
        componentRoot: path.resolve(__dirname, '..'),
        targetDirectory: testDataDir,
        isTestIsolation: true
    }
};

const cli = new ONCECLI();
cli.init(testScenario); // ✅ Pass scenario to init
await cli.component.startServer(); // ✅ Programmatic invocation
```

## Test Pattern Architecture

### Level 1: Pure Black-Box (Shell Commands)
**Use Case**: Integration tests, user acceptance tests  
**Method**: Execute `./once` bash wrapper with shell commands

```typescript
// Level 1: Shell command execution
execSync('./once startServer', { cwd: componentRoot });
await waitForServer(42777, 5000);
const response = await fetch('http://localhost:42777/health');
```

**Pros**: Exact user experience  
**Cons**: Hard to isolate paths (bash wrapper uses git to find projectRoot)

### Level 2: Programmatic CLI with Test Scenario ⭐ **RECOMMENDED**
**Use Case**: Component tests, lifecycle tests, scenario tests  
**Method**: Create ONCECLI instance, pass test scenario, invoke methods

```typescript
// Level 2: Programmatic CLI + Test Scenario
const testScenario = createTestScenario(testDataDir);
const cli = new ONCECLI();
cli.init(testScenario); // ✅ Scenario-based configuration

// Now component.model.projectRoot points to test/data
await cli.component.startServer();
```

**Pros**: Perfect isolation, Web4 compliant, programmatic control  
**Cons**: Not pure black-box (imports ONCECLI)

### Level 3: Direct Component (White-Box) ❌ **AVOID**
**Use Case**: Unit tests for specific component methods  
**Method**: Direct `DefaultONCE` instantiation

```typescript
// Level 3: White-box (AVOID in lifecycle tests)
const component = new DefaultONCE();
await component.init();
await component.startServer(); // ❌ Bypasses CLI delegation
```

**Pros**: Fine-grained control  
**Cons**: Violates black-box principle, bypasses CLI delegation

## Recommended Pattern for Lifecycle Tests

**Use Level 2 (Programmatic CLI + Test Scenario)**

### Setup Pattern

```typescript
import { ONCECLI } from '../src/ts/layer5/ONCECLI.js';
import { Scenario } from '../src/ts/layer3/Scenario.js';
import { CLIModel } from '../src/ts/layer3/CLIModel.interface.js';
import { v4 as uuidv4 } from 'uuid';
import * as path from 'path';
import * as fs from 'fs';

describe('Server Lifecycle Management (Black-Box)', () => {
    let testDataDir: string;
    let scenarioDir: string;
    let testScenario: Scenario<CLIModel>;
    let cli: ONCECLI;
    
    beforeEach(async () => {
        // Setup: Create test/data directory
        const componentRoot = path.resolve(__dirname, '..');
        testDataDir = path.join(componentRoot, 'test/data');
        scenarioDir = path.join(testDataDir, 'scenarios');
        
        // Clean test data
        if (fs.existsSync(testDataDir)) {
            fs.rmSync(testDataDir, { recursive: true, force: true });
        }
        fs.mkdirSync(scenarioDir, { recursive: true });
        
        // ✅ Create test scenario (Web4 pattern)
        testScenario = {
            ior: {
                protocol: 'web4',
                host: 'localhost',
                uuid: uuidv4(),
                component: 'ONCE',
                version: '0.3.21.2'
            },
            owner: 'test-runner-uuid',
            model: {
                projectRoot: testDataDir, // ✅ Test isolation
                componentRoot: componentRoot,
                targetDirectory: testDataDir,
                isTestIsolation: true, // ✅ Flag for test mode
                version: '0.3.21.2'
            }
        };
        
        // ✅ Create CLI with test scenario
        cli = new ONCECLI();
        cli.init(testScenario);
    });
    
    afterEach(async () => {
        // Cleanup: Stop all test servers
        try {
            await cli.component.stopServer();
        } catch (e) {
            // Server may not be running
        }
        
        // Clean test data
        if (fs.existsSync(testDataDir)) {
            fs.rmSync(testDataDir, { recursive: true, force: true });
        }
    });
});
```

### Test Example

```typescript
it('should start primary server and create scenario file', async () => {
    // ACT: Start server via CLI component
    await cli.component.startServer();
    
    // Wait for server to be ready
    await waitForServer(42777, 5000);
    
    // ASSERT: Check observable outcomes
    
    // 1. HTTP endpoint should be reachable
    const response = await fetch('http://localhost:42777/health');
    expect(response.status).toBe(200);
    
    // 2. Scenario file should exist in test/data/scenarios
    const scenarioFiles = fs.readdirSync(scenarioDir)
        .filter(f => f.endsWith('.scenario.json'));
    expect(scenarioFiles.length).toBeGreaterThan(0);
    
    // 3. Scenario should have correct state
    const scenarioPath = path.join(scenarioDir, scenarioFiles[0]);
    const scenarioData = JSON.parse(fs.readFileSync(scenarioPath, 'utf8'));
    
    // Handle both legacy and Web4 formats
    const state = scenarioData.model?.state?.state || scenarioData.state?.state;
    expect(state).toBe('running' || 'primary_server');
    
    // 4. Scenario should have correct port
    const port = scenarioData.model?.port || scenarioData.port;
    expect(port).toBe(42777);
    
    // CLEANUP
    await cli.component.stopServer();
});
```

## Key Principles

### 1. NO `process.env` Usage
```typescript
// ❌ WRONG
process.env.ONCE_PROJECT_ROOT = testDataDir;

// ✅ RIGHT
testScenario.model.projectRoot = testDataDir;
```

### 2. Scenario-Based Configuration
```typescript
// ❌ WRONG
const cli = new ONCECLI();
cli.model.projectRoot = testDataDir; // Direct mutation

// ✅ RIGHT
const testScenario = createTestScenario(testDataDir);
cli.init(testScenario); // Configuration via scenario
```

### 3. Observable Outcomes Only
```typescript
// ❌ WRONG (white-box)
const internalState = (cli.component as any).serverHierarchyManager.getServerModel();

// ✅ RIGHT (black-box)
const response = await fetch('http://localhost:42777/api/servers');
const servers = await response.json();
```

### 4. Test Isolation Verification
```typescript
afterAll(() => {
    // Verify NO production pollution
    const productionScenarios = path.join(projectRoot, 'scenarios');
    const testArtifacts = fs.readdirSync(productionScenarios, { recursive: true })
        .filter(f => f.includes('test'));
    
    expect(testArtifacts.length).toBe(0); // ✅ Zero test artifacts in production
});
```
