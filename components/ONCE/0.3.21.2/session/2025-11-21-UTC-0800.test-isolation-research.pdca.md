## **📋 PLAN - Research Findings**

### **1. Web4 Test Isolation Pattern (from Web4TSComponent)**

#### **Core Principle: Automatic Detection via CWD**

```typescript
// From DefaultCLI.ts:164-179
private calculateProjectRootInternal(): string {
    const cwd = process.cwd();
    
    // 1. Check for test isolation FIRST (highest priority for test context)
    // CRITICAL: Works for both Web4TSComponent tests AND generated component tests
    // Example: /Users/.../components/Web4TSComponent/0.3.17.1/test/data/components/TestIsolatedComponent/0.1.0.0
    // Result: /Users/.../components/Web4TSComponent/0.3.17.1/test/data
    if (cwd.includes('/test/data')) {
      // Find test/data directory by walking up from cwd
      const parts = cwd.split('/');
      const testIndex = parts.indexOf('test');
      if (testIndex !== -1 && testIndex + 1 < parts.length && parts[testIndex + 1] === 'data') {
        // Return path UP TO AND INCLUDING test/data
        return parts.slice(0, testIndex + 2).join('/');
      }
    }
    // ... fallback to git root, etc.
}
```

**Key Insight**: `projectRoot` is **calculated automatically** based on `process.cwd()`, NOT manually set!

#### **The `test shell` Command**

```typescript
// From DefaultWeb4TSComponent.ts:1938-2037
async testShell(version?: string): Promise<this> {
    const componentRoot = this.model.componentRoot;
    const testDataDir = path.join(componentRoot, 'test', 'data');
    
    // 1. Clean test/data directory for fresh start
    // 2. Run baseline tests to populate test/data
    // 3. Source component's source.env
    // 4. Start interactive shell with CWD = test/data
    
    // The MAGIC: When shell runs, process.cwd() = test/data
    // Therefore, calculateProjectRootInternal() detects test isolation!
}
```

**Pattern**:
- `test shell` **changes directory** to `{componentRoot}/test/data`
- All test commands run **with CWD = test/data**
- `DefaultCLI.calculateProjectRootInternal()` **automatically detects** `/test/data` in path
- Result: `projectRoot` = `{componentRoot}/test/data` (no manual override needed!)

#### **The Model Variables**

From `ONCEModel.interface.ts` and `CLIModel.interface.ts`:


**Key Insight**: `testDataDir` is NOT a separate variable for scenarios! It's `path.join(componentRoot, 'test', 'data')` - the DIRECTORY where tests run from.

### **2. What I Did Wrong**

#### **Incorrect Approach (My Implementation)**

```typescript
// ❌ WRONG: Manual override in test setup
const productionRoot = path.resolve(...);
testProjectRoot = path.join(productionRoot, 'components/ONCE', '0.3.21.2', 'test/data');
server.model.projectRoot = testProjectRoot; // ❌ Manual override!
```

**Problems**:
1. Violates Path Authority - CLI should calculate, not tests
2. Breaks when component methods call `calculateProjectRoot()` again
3. Doesn't leverage automatic `/test/data` detection
4. `testProjectRoot` used as "project root for scenarios" - **WRONG CONCEPT!**

#### **Correct Approach (Web4 Pattern)**

```bash
# Run tests FROM test/data directory
cd components/ONCE/0.3.21.2/test/data
npx vitest run ../test/lifecycle-management.test.ts

# OR use test shell:
once test shell  # Automatically cd to test/data and source env
```

**Result**:
1. `process.cwd()` = `.../components/ONCE/0.3.21.2/test/data`
2. `calculateProjectRootInternal()` detects `/test/data` in path
3. Returns `.../components/ONCE/0.3.21.2/test/data` as `projectRoot`
4. All scenarios written to `{projectRoot}/scenarios/...` = `.../test/data/scenarios/...`
5. **NO MANUAL OVERRIDES NEEDED!**
