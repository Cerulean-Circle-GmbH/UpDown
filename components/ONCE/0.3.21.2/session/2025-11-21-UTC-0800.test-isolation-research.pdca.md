## **🎯 Web4 Principles - Path Authority & DRY**

### **Principle 1: Paths from Source Code, Never CWD**

> **All directory paths MUST be derived from component source code location (`import.meta.url`), NEVER from `process.cwd()`.**
> 
> **Rationale**: `process.cwd()` is random and depends on where the command was run from. Component structure is well-defined and deterministic. Source code location is the single source of truth.

### **Principle 2: DRY - Use Model Paths, Never Recalculate**

> **All path calculations are done ONCE by the CLI and stored in the model. Components MUST use `this.model.*` paths, NEVER recalculate them.**
>
> **Rationale**: Path Authority - CLI is responsible for calculating paths. Components receive paths through their model (dependency injection). Recalculating violates DRY and creates inconsistencies.

**CLIModel Path Variables** (from `src/ts/layer3/CLIModel.interface.ts`):
```typescript
interface CLIModel {
  projectRoot: string;              // e.g., /Users/.../Web4Articles (or test/data in test isolation)
  componentsDir: string;            // e.g., projectRoot/components
  scriptsDir: string;               // e.g., projectRoot/scripts  
  scriptsVersionDir: string;        // e.g., projectRoot/scripts/versions
  testDataDir: string;              // e.g., componentRoot/test/data
}
```

**Correct Usage**:
```typescript
// ✅ CORRECT: Use model paths (injected by CLI)
const projectRoot = this.model.projectRoot;
const testDataDir = this.model.testDataDir;
const componentsDir = this.model.componentsDir;

// ❌ WRONG: Recalculating paths
const projectRoot = path.resolve(dirname(import.meta.url), '../../../..');
const testDataDir = path.join(projectRoot, 'test/data');
```
