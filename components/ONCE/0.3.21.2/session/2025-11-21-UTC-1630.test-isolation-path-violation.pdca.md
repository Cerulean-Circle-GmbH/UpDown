## Act

### Solution Implemented ✅

**Approach**: Follow Web4TSComponent pattern for path discovery

**Key Finding**: ONCE was missing the synchronous path discovery pattern that Web4TSComponent uses!

**Comparison Analysis**:
1. **Web4TSComponent** (CORRECT):
   - Constructor is truly empty (no path calculations)
   - `init()` method discovers paths from `import.meta.url` with `realpathSync()`
   - CLI calls `component.init()` THEN overrides `projectRoot`
   
2. **ONCE** (BROKEN):
   - Constructor calculated paths without `realpathSync()`
   - No `init()` path discovery
   - ONCECLI tried to manually set all paths

**Solution Applied**:

1. **Added `discoverPathsFromFilesystem()` method to DefaultONCE** (@bash 1054-1070):
   ```typescript
   private discoverPathsFromFilesystem(): void {
     try {
       const currentFilePath = path.dirname(new URL(import.meta.url).pathname);
       const currentVersionDir = realpathSync(path.resolve(currentFilePath, '..', '..', '..'));
       const componentDirName = path.basename(currentVersionDir);
       const isVersionDir = /^\d+\.\d+\.\d+\.\d+$/.test(componentDirName);
       
       const versionString = isVersionDir ? componentDirName : '0.0.0.0';
       
       this.model.componentRoot = currentVersionDir;
       this.model.version = versionString;
     } catch (error) {
       console.warn('⚠️ Path discovery failed, using defaults:', error);
     }
   }
   ```

2. **Updated DefaultONCE constructor**:
   - Calls `discoverPathsFromFilesystem()` during construction
   - Uses `realpathSync()` to resolve symlinks correctly
   - Sets `componentRoot` and `version` from actual filesystem location

3. **Simplified ONCECLI**:
   - Creates component: `new DefaultONCE()` (paths already discovered)
   - Overrides only CLI-specific paths: `projectRoot`, `targetDirectory`, `isTestIsolation`
   - Follows Web4TSComponent delegation pattern exactly

4. **Web4 Naming Violation Fixed**:
   - ❌ Originally named: `_discoverPaths()` (FORBIDDEN underscore prefix!)
   - ✅ Renamed to: `discoverPathsFromFilesystem()` (Web4 compliant)
   - Added underscore prohibition to Web4 principles in main PDCA
