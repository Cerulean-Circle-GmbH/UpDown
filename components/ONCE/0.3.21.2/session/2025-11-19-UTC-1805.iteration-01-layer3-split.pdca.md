## **📋 CMM3 PDCA Structure**

### **P - PLAN**

#### **1. Problem Statement**
Layer 3 (Interface/Service Contracts) currently violates the Web4 principle "one type per file". Multiple interfaces, enums, and types are co-located in single files, making the codebase harder to navigate, test, and maintain. This iteration will split all Layer 3 files to achieve strict one-type-per-file compliance.

#### **2. Current Situation (AS-IS)**

**Layer 3 Audit Results**:

| File | Types Defined | Status | Action Required |
|------|---------------|--------|-----------------|
| `LifecycleEvents.ts` | 5 types | ❌ VIOLATES | Split into 5 files |
| `ONCE.ts` | 4 interfaces | ❌ VIOLATES | Split into 4 files |
| `ONCEServerModel.ts` | 2 types + 1 const | ❌ VIOLATES | Split into 2 files, remove const |
| `ONCEModel.interface.ts` | 3 interfaces | ❌ VIOLATES | Split into 3 files |
| `Component.ts` | 1 interface (duplicate) | ❌ DEPRECATED | Mark for removal |

4. `ONCEModel.interface.ts` contains:
   - `ONCEModel` (interface)

#### **4. Root Cause Analysis**

**Why does Layer 3 have multiple types per file?**
- **Historical**: Incremental development without strict enforcement
- **Convenience**: Co-locating related types seemed practical
- **Unawareness**: Web4 "one type per file" principle not consistently applied
- **Legacy**: Some files inherited from earlier versions

**5W1H Analysis**:
- **What**: Split 4 files into 14+ files

- **When**: Iteration 1 (foundation for all future iterations)
- **Where**: `src/ts/layer3/` directory
- **Who**: AI Assistant with automated testing
- **How**: File splitting + import updates + comprehensive testing

4. **`LifecycleObserver.interface.ts`** (NEW - replaces functional LifecycleEventHandler):
```typescript
import { LifecycleEvent } from './LifecycleEvent.interface.js';

/**
 * Lifecycle observer interface (Observer Pattern)
 * ✅ Web4: TRUE Radical OOP - no functional types
 * ✅ Replaces: type LifecycleEventHandler = (event: LifecycleEvent) => void | Promise<void>
 */
export interface LifecycleObserver {
  /**
   * Called before component initialization
   */
  onBeforeInit(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called after component initialization
   */
  onAfterInit(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called before component start
   */
  onBeforeStart(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called after component start
   */
  onAfterStart(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called before component stop
   */
  onBeforeStop(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called after component stop
   */
  onAfterStop(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called on error
   */
  onError(event: LifecycleEvent): void | Promise<void>;
  
  /**
   * Called on state change
   */
  onStateChange(event: LifecycleEvent): void | Promise<void>;
}
```

5. **`LifecycleSubject.interface.ts`** (NEW - replaces functional LifecycleHooks):
```typescript
import { LifecycleObserver } from './LifecycleObserver.interface.js';
import { LifecycleEvent } from './LifecycleEvent.interface.js';

/**
 * Lifecycle subject interface (Observer Pattern)
 * ✅ Web4: TRUE Radical OOP - Observer Pattern
 * ✅ Replaces: interface LifecycleHooks (functional approach)
 */
export interface LifecycleSubject {
  /**
   * Attach an observer
   */
  attach(observer: LifecycleObserver): void;
  
  /**
   * Detach an observer
   */
  detach(observer: LifecycleObserver): void;
  
  /**
   * Notify all observers of an event
   */
  notify(event: LifecycleEvent): Promise<void>;
}
```
