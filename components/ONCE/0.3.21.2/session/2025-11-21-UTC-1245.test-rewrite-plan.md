# Test Rewrite Plan - White Box to Black Box

**Date**: 2025-11-21  
**Component**: ONCE 0.3.21.2  
**Task**: Rewrite lifecycle-management.test.ts from white-box to black-box

## Problem Statement

The current tests directly instantiate `DefaultONCE` class (white-box), which:
1. Breaks the component delegation chain
2. Doesn't test the actual user experience (CLI commands)
3. Causes `componentRoot` and path authority issues
4. Doesn't match how users interact with ONCE

## What Each Test Currently Tests (White Box)

### Test 1: "should delete shutdown server scenarios on primary startup"
**Current Approach**:
- Manually creates a shutdown scenario JSON file
- Creates symlink manually
- Directly instantiates `DefaultONCE` and calls `startServer()`
- Checks that files are deleted

**What It Tests**: Housekeeping deletes shutdown scenarios on startup

### Test 2: "should discover and keep running client server scenarios"
**Current Approach**:
- Starts primary server via `new DefaultONCE()` 
- Starts client server via another `new DefaultONCE()`
- Stops client, manually modifies scenario to "running"
- Restarts primary
- Checks that scenario still exists (because server is "reachable")

**What It Tests**: Housekeeping discovers and preserves running client scenarios

### Test 3: "should update scenario state to STOPPING then SHUTDOWN"
**Current Approach**:
- Starts server via `new DefaultONCE()`
- Calls `stopServer()`
- Checks scenario file transitions: RUNNING → STOPPING → SHUTDOWN

**What It Tests**: Graceful shutdown updates scenario state correctly

### Test 4: "should register new client server dynamically"
**Current Approach**:
- Starts primary via `new DefaultONCE()`
- Starts two client servers dynamically
- Checks `getRegisteredServers()` count

**What It Tests**: Client servers register with primary dynamically

### Test 5: "should handle client disconnect and cleanup"
**Current Approach**:
- Starts primary and client via `new DefaultONCE()`
- Stops client
- Checks that primary's registry no longer contains client

**What It Tests**: Client unregistration and cleanup

### Test 6: "should persist server state across restarts"
**Current Approach**:
- Starts server via `new DefaultONCE()`
- Stops it
- Restarts it
- Checks scenario UUID is different but port is reused

**What It Tests**: Scenario persistence and port reuse

### Test 7: "should restart primary and rediscover running clients after crash"
**Current Approach**:
- Spawns primary and client processes via `spawn-server.mjs`
- Kills primary process (simulate crash)
- Restarts primary
- Checks that clients are rediscovered

**What It Tests**: Crash recovery and client rediscovery

## Black Box Rewrite Strategy

### General Principles
1. **Use CLI commands only**: `once startServer`, `once stopServer`, `once status`
2. **Check observable outcomes**: HTTP endpoints, scenario files, process state
3. **No direct class instantiation**: All interaction via `./once` wrapper
4. **Test from user perspective**: What would a user see/do?

### Test 1 (Black Box): Housekeeping deletes shutdown scenarios
```typescript
it('should delete shutdown server scenarios on primary startup', async () => {
    // 1. Manually create a shutdown scenario file
    const shutdownUuid = uuidv4();
    const scenarioPath = `${scenarioDir}/${shutdownUuid}.scenario.json`;
    fs.writeFileSync(scenarioPath, JSON.stringify({
        ior: { uuid: shutdownUuid, component: 'ONCE', version: '0.3.21.2' },
        owner: 'test',
        model: { state: { state: 'shutdown' }, capabilities: [{ capability: 'httpPort', port: 8080 }] }
    }));
    
    // 2. Start primary via CLI
    execSync('./once startServer', { cwd: projectRoot });
    await waitForServer(42777, 5000);
    
    // 3. Check that shutdown scenario was deleted
    expect(fs.existsSync(scenarioPath)).toBe(false);
    
    // 4. Cleanup
    execSync('./once stopServer', { cwd: projectRoot });
});
```

### Test 3 (Black Box): Graceful shutdown state transitions
```typescript
it('should update scenario state to STOPPING then SHUTDOWN', async () => {
    // 1. Start server
    execSync('./once startServer', { cwd: projectRoot });
    const uuid = await getServerUuid(42777); // Get via HTTP /servers endpoint
    const scenarioPath = `${scenarioDir}/${uuid}.scenario.json`;

    const beforeScenario = JSON.parse(fs.readFileSync(scenarioPath));
    expect(beforeScenario.model.state.state).toBe('running');
    
    // 3. Stop server
    execSync('./once stopServer', { cwd: projectRoot });
    
    // 4. Check final state is SHUTDOWN
    const afterScenario = JSON.parse(fs.readFileSync(scenarioPath));
    expect(afterScenario.model.state.state).toBe('shutdown');
});
```

### Test 7 (Black Box): Crash recovery
```typescript
it('should restart primary and rediscover running clients after crash', async () => {
    // 1. Start primary as background process
    const primary = spawn('node', ['./once', 'startServer'], { 
        cwd: projectRoot, 
        detached: true,
        stdio: 'ignore'
    });
    await waitForServer(42777, 5000);
    
    // 2. Start client servers
    const client1 = spawn('node', ['./once', 'startServer'], { cwd: projectRoot, detached: true });
    const client2 = spawn('node', ['./once', 'startServer'], { cwd: projectRoot, detached: true });
    await waitForServer(8080, 5000);
    await waitForServer(8081, 5000);
    
    // 3. Verify clients are registered
    const before = await fetch('http://localhost:42777/servers').then(r => r.json());
    expect(before.servers.length).toBe(2);
    
    // 4. Kill primary (simulate crash)
    process.kill(primary.pid, 'SIGKILL');
    await new Promise(resolve => setTimeout(resolve, 1000));
    
    // 5. Verify primary is dead
    await expect(fetch('http://localhost:42777/servers')).rejects.toThrow();
    
    // 6. Restart primary
    execSync('./once startServer', { cwd: projectRoot });
    await waitForServer(42777, 5000);
    
    // 7. Verify clients are rediscovered (they're still running)
    // Note: This depends on whether clients try to reconnect
    // May need to poll /servers endpoint with retry logic
    await waitFor(async () => {
        const after = await fetch('http://localhost:42777/servers').then(r => r.json());
        return after.servers.length === 2;
    }, 10000);
    
    // 8. Cleanup
    [client1, client2].forEach(p => process.kill(p.pid, 'SIGTERM'));
    execSync('./once stopServer', { cwd: projectRoot });
});
```

## Helper Functions Needed

```typescript
// Wait for HTTP server to be available
async function waitForServer(port: number, timeout: number): Promise<void> {
    const start = Date.now();
    while (Date.now() - start < timeout) {
        try {
            await fetch(`http://localhost:${port}/`);
            return;
        } catch {
            await new Promise(resolve => setTimeout(resolve, 100));
        }
    }
    throw new Error(`Server on port ${port} not available after ${timeout}ms`);
}

// Get server UUID from /servers endpoint
async function getServerUuid(port: number): Promise<string> {
    const response = await fetch(`http://localhost:${port}/servers`);
    const data = await response.json();
    return data.primaryServer.uuid;
}

// Generic wait with condition
async function waitFor(condition: () => Promise<boolean>, timeout: number): Promise<void> {
    const start = Date.now();
    while (Date.now() - start < timeout) {
        if (await condition()) return;
        await new Promise(resolve => setTimeout(resolve, 100));
    }
    throw new Error(`Condition not met after ${timeout}ms`);
}
```

## Implementation Notes

1. **All tests must use `execSync('./once startServer')` or `spawn('node', ['./once', 'startServer'])`**
2. **No direct `new DefaultONCE()` - this is white-box**
3. **Verify behavior via HTTP endpoints (`/servers`, `/status`) and filesystem (scenario files)**
4. **Each test must be independent - cleanup thoroughly in afterEach**
5. **Tests should be resilient to timing issues - use retry logic where needed**
