### 3. Target Situation (TO-BE)

#### Black-Box Test Architecture

**Principles**:
1. ✅ **CLI-Only Interaction**: All server operations via `./once` CLI wrapper
2. ✅ **Test Isolation**: All artifacts in `test/data/` directory
3. ✅ **Observable Outcomes**: Verify via HTTP endpoints, scenario files, process state
4. ✅ **No Internal Access**: No `(instance as any).internalProperty`
5. ✅ **Production Matching**: Tests use same commands as users

#### Black-Box Test Pattern

```typescript
describe('Server Lifecycle Management (Black-Box)', () => {
    let testDataDir: string;
    let scenarioDir: string;
    
    beforeEach(async () => {
        // Setup: Create test/data directory
        const componentRoot = path.resolve(__dirname, '..');
        testDataDir = path.join(componentRoot, 'test/data');
        scenarioDir = path.join(testDataDir, 'scenarios');
        
        // Clean test data
        if (fs.existsSync(testDataDir)) {
            fs.rmSync(testDataDir, { recursive: true, force: true });
        }
        fs.mkdirSync(scenarioDir, { recursive: true });
        
        // Override ONCE_PROJECT_ROOT for tests
        process.env.ONCE_PROJECT_ROOT = testDataDir;
    });
    
    afterEach(async () => {
        // Cleanup: Stop all test servers
        execSync('pkill -f "once.*startServer" || true', { stdio: 'ignore' });
        
        // Clean test data
        if (fs.existsSync(testDataDir)) {
            fs.rmSync(testDataDir, { recursive: true, force: true });
        }
        
        delete process.env.ONCE_PROJECT_ROOT;
    });
    
    it('should start primary server on port 42777', async () => {
        // ACT: Start server via CLI (black-box)
        execSync('./once startServer primary &', {
            cwd: path.resolve(__dirname, '..'),
            env: { ...process.env, ONCE_PROJECT_ROOT: testDataDir }
        });
        
        // Wait for server to start
        await waitForServer(42777, 5000);
        
        // ASSERT: Check observable outcomes
        const isHealthy = await checkServerHealth(42777);
        expect(isHealthy).toBe(true);
        
        // ASSERT: Check scenario file exists
        const scenarioFiles = fs.readdirSync(scenarioDir)
            .filter(f => f.endsWith('.scenario.json'));
        expect(scenarioFiles.length).toBeGreaterThan(0);
        
        // ASSERT: Verify scenario contains correct port
        const scenarioData = JSON.parse(fs.readFileSync(
            path.join(scenarioDir, scenarioFiles[0]), 
            'utf8'
        ));
        expect(scenarioData.model?.port || scenarioData.port).toBe(42777);
    });
});
```

**Key Differences from White-Box**:
- ✅ Uses `execSync('./once startServer')` instead of `new DefaultONCE()`
- ✅ Uses `test/data/` instead of production `scenarios/`
- ✅ Verifies via HTTP `/health` endpoint (observable)
- ✅ Verifies via scenario file content (observable)
- ✅ No access to internal component state

### 4. Root Cause Analysis

**Why did white-box tests happen?**

**Root Cause 1: Misunderstanding of Test Scope**
- **Issue**: Tests were written to verify internal component logic, not user-facing behavior
- **Example**: Checking `serverHierarchyManager.getServerModel()` instead of HTTP responses
- **Impact**: Tests became brittle and dependent on implementation details

**Root Cause 2: Missing Test Isolation Pattern**
- **Issue**: Web4TSComponent's `test/data` pattern wasn't documented in ONCE
- **Example**: No `ONCE_PROJECT_ROOT` environment variable support
- **Impact**: Tests polluted production `scenarios/` directory

**Root Cause 3: No CLI Test Pattern**
- **Issue**: No examples of testing via CLI wrapper
- **Example**: All examples showed direct `DefaultONCE` instantiation
- **Impact**: Tests bypassed CLI delegation (path authority)

**Root Cause 4: Context Loss During Iteration 1.5**
- **Issue**: Got lost in test fixes, forgot main refactoring goal
- **Example**: Spent 6+ hours on path override hacks
- **Impact**: Deviated from MAIN PDCA objectives (TRUE Radical OOP)
