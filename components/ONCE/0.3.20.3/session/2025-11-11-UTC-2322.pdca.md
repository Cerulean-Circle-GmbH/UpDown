## ACT

### Lessons Learned

**What Worked Well:**
1. **Web4 Scenario Format** - Using TRUE Radical OOP with component/version/data structure provides perfect message format
2. **Model-Driven Tracking** - All message state in `model.messageTracker` (no private properties)
3. **Auto-Acknowledgments** - Automatic ACK on message receipt creates clear communication trail
4. **Dual Output** - Terminal for human, JSON for machines - perfect for both demo and CI
5. **Pattern Separation** - Clear broadcast/relay/P2P separation makes communication patterns explicit

**Challenges Overcome:**
1. **TypeScript Any Types** - Needed `(c: any)` for capability filtering (acceptable for dynamic model access)
2. **WebSocket Connections** - P2P creates temporary connections (close after send)
3. **Async Coordination** - Used `sleep()` to let servers register before sending messages

**Web4 Principles Applied:**
- Component-based messaging (ONCE↔ONCE scenarios)
- Model-driven state (all tracking in model)
- Path authority (ports derived from model)
- Scenario format (standard Web4 structure)
- No environment variables (all config in model)

### Standardization

**Reusable Patterns:**
1. **ONCEScenarioMessage Interface** - Can be adopted by other Web4 components
2. **Message Tracker Pattern** - Model-driven message tracking template
3. **Auto-Acknowledgment** - Automatic ACK generation on receipt
4. **JSON Report Export** - Standard reporting format for message exchanges
