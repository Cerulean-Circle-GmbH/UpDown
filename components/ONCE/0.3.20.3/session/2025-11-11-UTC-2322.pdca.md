## ACT

### Lessons Learned

**What Worked Well:**
1. **Scenario Format Compliance** - ONCEScenarioMessage now extends Scenario interface - TRUE Web4 format
2. **Model-Driven Tracking** - All message state in `model.messageTracker` (no private properties)
3. **Auto-Acknowledgments** - Automatic ACK on message receipt creates clear communication trail
4. **ACK Loop Prevention** - Simple `startsWith('ACK:')` check prevents infinite recursion
5. **Dual Output** - Terminal for human, JSON for machines - perfect for both demo and CI
6. **Pattern Separation** - Clear broadcast/relay/P2P separation makes communication patterns explicit
7. **Demo Integration** - Numeric parameter in `demo()` delegates to `demoMessages()` seamlessly

**Challenges Overcome:**
1. **ACK Infinite Loop** - Initial implementation caused ACKs to trigger new ACKs indefinitely
   - **Fix:** Added check to skip ACK generation for messages with content starting with "ACK:"
2. **Scenario Format Migration** - Changed from flat `component/version/data` to proper Scenario structure
   - **Fix:** Extended Scenario interface, added `objectType`, `state`, and `metadata` fields
3. **TypeScript Any Types** - Needed `(c: any)` for capability filtering (acceptable for dynamic model access)
4. **WebSocket Connections** - P2P creates temporary connections (close after send)
5. **Async Coordination** - Used `sleep()` to let servers register before sending messages

**Web4 Principles Applied:**
- **TRUE Scenario Format:** Messages extend Scenario interface with objectType, state, metadata
- **Component-based messaging:** ONCE↔ONCE scenarios
- **Model-driven state:** All tracking in model
- **Path authority:** Ports derived from model
- **No environment variables:** All config in model

### Standardization

**Reusable Patterns:**
1. **ONCEScenarioMessage Interface** - Extends Scenario with message-specific state - adoptable by other Web4 components
2. **Message Tracker Pattern** - Model-driven message tracking template
3. **Auto-Acknowledgment** - Automatic ACK generation on receipt (with loop prevention)
4. **JSON Report Export** - Standard reporting format for message exchanges
5. **Demo Command Extension** - Pattern for adding automated modes to interactive commands
