### **Radical OOP Principles to Enforce**

Based on [Radical OOP Analysis PDCA](../../../../../Web4Articles/components/Web4TSComponent/0.3.18.1/session/2025-11-06-UTC-0030.functional-vs-radical-oop-analysis.pdca.md):

| Principle | Violation Pattern | Detectable? | ESLint Rule |
|-----------|-------------------|-------------|-------------|
| **1. No CommonJS** | `require()` calls | ✅ Yes | `no-restricted-syntax` |
| **2. ES Modules Only** | `module.exports` | ✅ Yes | `no-restricted-syntax` |
| **3. Async/Await** | `.then()` chains | ⚠️ Partial | `prefer-promise-reject-errors`, `no-floating-promises` |
| **4. No `__dirname`/`__filename`** | Node.js globals | ✅ Yes | `no-restricted-globals` |
| **5. `import.meta.url`** | Path detection pattern | ⚠️ Complex | Custom rule or docs |
| **6. Empty Constructor** | Complex constructor logic | ❌ Complex | Manual review |
| **7. Scenario Initialization** | `init()` pattern | ❌ Semantic | Manual review |
| **8. Model-Driven State** | Direct calculations | ❌ Semantic | Manual review |
| **9. Method Chaining** | Return type `this` | ⚠️ Partial | TypeScript strict |
| **10. No Functional Helpers** | Pure functions in class | ❌ Semantic | Manual review |

**Legend:**
- ✅ Yes = Fully detectable with ESLint
- ⚠️ Partial = Can detect some cases, not all
- ❌ Complex/Semantic = Requires human review or custom tooling

      },

        message: '🚫 CommonJS exports is PROHIBITED. Use ES module export instead.',

        name: '__dirname',
        message: '🚫 __dirname is PROHIBITED. Use import.meta.url with path.dirname() instead.',
      },
      {
        name: '__filename',
        message: '🚫 __filename is PROHIBITED. Use import.meta.url instead.',
      },

    '@typescript-eslint/no-floating-promises': 'error',
    '@typescript-eslint/await-thenable': 'error',

    'prefer-promise-reject-errors': 'error',

**Linting vs Manual Review:**

| Aspect | Linting (Automated) | Manual Review (Human) |
|--------|---------------------|----------------------|
| Speed | Milliseconds | Hours |
| Consistency | 100% | Variable |
| Coverage | Syntax patterns | Semantic meaning |
| Cost | One-time setup | Every PR |
| Scalability | Infinite | Limited |

- Linter catches syntax violations (CommonJS, globals)
- Code review focuses on architecture (model-driven, init patterns)

- ✅ "This code uses `require()`"
- ❌ "This method recalculates instead of reading from model"

For semantic Radical OOP principles, we need:
1. Architecture documentation (PDCAs)
2. Code review guidelines
3. Pair programming / mentoring
4. (Future) Custom AST-based tools
