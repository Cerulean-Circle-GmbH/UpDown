### **2. New Web4 Principles to Add**

#### **Web4 Principle 17: Component Instance Pattern (startComponent)**
> "Components are versioned, self-contained modules in `components/{Name}/{version}/`. The `static start(config)` method **LITERALLY runs `npm start`** to ensure the component is built, compiled, and all dependencies are installed before creating an instance. Instances are configured scenarios for that component. Use lazy getters to load instances on demand. ALWAYS store instances using `Reference<T>` (Web4 Principle 5), never `Type | null`."

**Critical Understanding - What `static start()` REALLY Does**:
```typescript
// Example: HTMLRoute.start() (FUTURE - when HTMLRoute is its own component)

    // 2. Run: cd components/HTMLRoute/0.3.21.7 && npm install
    // 3. Run: cd components/HTMLRoute/0.3.21.7 && npm start (compile TypeScript)
    // 4. Lazy import: const { HTMLRoute } = await import('components/HTMLRoute/0.3.21.7/dist/...');
    // 5. Create instance: const instance = new HTMLRoute().init(config.scenario);
    // 6. Return instance
}
```

#### **Web4 Principle 18: Black-Box Testing Only**
> "Tests must be protocol-agnostic. Tests use IOR calls and don't know if communication happens via HTTP, WebSocket, or any other loader. White-box tests that check implementation details are forbidden."

**Pattern**:
```typescript
// ✅ BLACK-BOX: Protocol-agnostic
const health = await kernel.loadViaIOR('ior:scenario:REST:https://localhost:42777/health');
expect(health.model.status).toBe('healthy');

// ❌ WHITE-BOX: Knows about HTTP
const response = await fetch('http://localhost:42777/health');
expect(response.status).toBe(200);
```

#### **Web4 Principle 19: Server Binding vs URL Host**
> "Distinguish between 'host for URLs' (FQDN for scenarios/IORs) and 'interface to bind to' (IP address/interface name for server.listen()). Use `0.0.0.0` or specific IP for binding, use FQDN for URLs."

**Pattern**:
```typescript
// ✅ CORRECT: Separate concerns
model.urlHost = 'mcdonges-3.fritz.box';  // For URLs, scenarios, IORs
model.bindInterface = '0.0.0.0';         // For server.listen()
server.listen(port, model.bindInterface); // Bind to interface
const ior = `https://${model.urlHost}:${port}/path`; // Use FQDN in IOR
```

#### **Web4 Principle 20: Radical OOP ID Generation**
> "Never use functional ID generation (e.g., `uuidv4()`). Always use object-oriented ID providers. Define `IDProvider.interface.ts` and implement providers like `UUIDProvider.ts`. IDs are created via `idProvider.create()`, making generation testable, mockable, and replaceable."

**Pattern**:
```typescript
// IDProvider.interface.ts
export interface IDProvider {
    create(): string;
    validate(id: string): boolean;
}

// UUIDProvider.ts
export class UUIDProvider implements IDProvider {
    public create(): string {
        return randomUUID();  // Node.js crypto.randomUUID()
    }
    
    public validate(id: string): boolean {
        const uuidRegex = /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i;
        return uuidRegex.test(id);
    }
}

// Usage in Component
export class MyComponent {
    private idProvider: IDProvider;
    
    constructor(idProvider: IDProvider = new UUIDProvider()) {
        this.idProvider = idProvider;
    }
    
    public createInstance(): this {
        this.model.uuid = this.idProvider.create();  // ✅ OOP!
        return this;
    }
}

// ❌ WRONG: Functional (not testable, not mockable)
import { v4 as uuidv4 } from 'uuid';
this.model.uuid = uuidv4();

// ✅ CORRECT: Radical OOP (testable, mockable, replaceable)
this.model.uuid = this.idProvider.create();
```

**Benefits**:
- ✅ Testable (can inject mock provider)
- ✅ Replaceable (can swap UUID → ULID → custom)
- ✅ Radical OOP (no functional calls)
- ✅ Centralized (one place to change ID strategy)

### **New Web4 Principles Added**

**Principle 17**: Component Instance Pattern (startComponent)
- Components are versioned folders with package.json
- `static start()` **LITERALLY runs `npm start`**
- Instances created via scenarios
- Use `Reference<T>` for storage

**Principle 18**: Black-Box Testing Only
- Tests use IOR calls (protocol-agnostic)
- No knowledge of HTTP/WS/etc
- White-box tests forbidden

**Principle 19**: Server Binding vs URL Host
- `bindInterface` for `server.listen()` (0.0.0.0)
- `urlHost` for URLs/IORs (FQDN)

**Principle 20**: Radical OOP ID Generation
- Never use `uuidv4()` (functional)
- Always use `idProvider.create()` (OOP)
- Testable, replaceable, centralized
