## **📋 P - PLAN**

### **1. Problem Statement**

**Current Issues Identified**:

1. **Hostname Binding Error**:
   - HTTPServer binds to `this.serverModel.host` (FQDN like `ea9b5422-587c-4f74-8fed-f90ff03fd4ee.fritz.box`)
   - Node.js `server.listen()` expects IP address or interface name, not hostname
   - Scenarios are written correctly with FQDN for URLs
   - **Root Cause**: Confusion between "host for URLs" vs "interface to bind to"

2. **Header Repetition** (DRY Violation):
   - Headers like `'Access-Control-Allow-Origin': '*'` repeated in multiple routes
   - Should be configured once in HTTPServer model
   - **Web4 Principle 8 Violation**: DRY - Don't Repeat Yourself

3. **Functional Route Instantiation**:
   ```typescript
   // ❌ CURRENT: Functional instantiation
   const homeRoute = new HTMLRoute();
   homeRoute.model.uuid = uuidv4();  // ← Forbidden functional code
   homeRoute.setPattern('/', HttpMethod.GET);
   homeRoute.setProvider(() => this.component!.serveStatus());
   ```

4. **Missing Route Component Pattern**:
   - Routes should be **components** (following `once startComponent` pattern)
   - Routes should have their own version, dependencies, build process
   - Routes should be lazy-loaded via getters
   - Routes should follow: `HomeRoute.start()` ≈ `§/components/HomeRoute/0.3.21.7 npm start`

5. **No Black-Box Tests**:
   - Current tests are white-box (know implementation details)
   - Need protocol-agnostic tests (don't care if HTTP, WS, or other loader)
   - Should test via IOR calls only (true Web4 testing)

### **2. New Web4 Principles to Add**

#### **Web4 Principle 17: Route Component Pattern**
> "Routes are components. Each route has its own version, dependencies, and lifecycle. Routes are loaded via `startComponent()` which ensures dependencies are installed, code is compiled, and route is ready to handle requests."

**Pattern**:
```typescript
// ✅ Routes as lazy-loaded components
get homeRoute(): HomeRoute {
    if (!this._homeRoute) {
        // Like: once startComponent HomeRoute 0.3.21.7
        this._homeRoute = HomeRoute.start({
            pattern: '/',
            method: HttpMethod.GET,
            provider: () => this.component!.serveStatus()
        });
    }
    return this._homeRoute;
}

// Register routes (now just calls getters)
this.httpRouter.registerRoute(this.homeRoute);
```

#### **Web4 Principle 18: Black-Box Testing Only**
> "Tests must be protocol-agnostic. Tests use IOR calls and don't know if communication happens via HTTP, WebSocket, or any other loader. White-box tests that check implementation details are forbidden."

**Pattern**:
```typescript
// ✅ BLACK-BOX: Protocol-agnostic
const health = await kernel.loadViaIOR('ior:scenario:REST:https://localhost:42777/health');
expect(health.model.status).toBe('healthy');

// ❌ WHITE-BOX: Knows about HTTP
const response = await fetch('http://localhost:42777/health');
expect(response.status).toBe(200);
```

#### **Web4 Principle 19: Server Binding vs URL Host**
> "Distinguish between 'host for URLs' (FQDN for scenarios/IORs) and 'interface to bind to' (IP address/interface name for server.listen()). Use `0.0.0.0` or specific IP for binding, use FQDN for URLs."

**Pattern**:
```typescript
// ✅ CORRECT: Separate concerns
model.urlHost = 'mcdonges-3.fritz.box';  // For URLs, scenarios, IORs
model.bindInterface = '0.0.0.0';         // For server.listen()
server.listen(port, model.bindInterface); // Bind to interface
const ior = `https://${model.urlHost}:${port}/path`; // Use FQDN in IOR
```
