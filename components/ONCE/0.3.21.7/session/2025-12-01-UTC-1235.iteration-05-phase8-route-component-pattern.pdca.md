## **📋 P - PLAN**

### **1. Problem Statement**

**Current Issues Identified**:

1. **Hostname Binding Error**:
   - HTTPServer binds to `this.serverModel.host` (FQDN like `ea9b5422-587c-4f74-8fed-f90ff03fd4ee.fritz.box`)
   - Node.js `server.listen()` expects IP address or interface name, not hostname
   - Scenarios are written correctly with FQDN for URLs
   - **Root Cause**: Confusion between "host for URLs" vs "interface to bind to"

2. **Header Repetition** (DRY Violation):
   - Headers like `'Access-Control-Allow-Origin': '*'` repeated in multiple routes
   - Should be configured once in HTTPServer model
   - **Web4 Principle 8 Violation**: DRY - Don't Repeat Yourself

3. **Functional Route Instantiation**:
   ```typescript
   // ❌ CURRENT: Functional instantiation
   const homeRoute = new HTMLRoute();
   homeRoute.model.uuid = uuidv4();  // ← Forbidden functional code
   homeRoute.setPattern('/', HttpMethod.GET);
   homeRoute.setProvider(() => this.component!.serveStatus());
   ```

4. **Missing Route Component Pattern**:
   - Routes should be **components** (following `once startComponent` pattern)
   - Routes should have their own version, dependencies, build process
   - Routes should be lazy-loaded via getters
   - Routes should follow: `HomeRoute.start()` ≈ `§/components/HomeRoute/0.3.21.7 npm start`

5. **No Black-Box Tests**:
   - Current tests are white-box (know implementation details)
   - Need protocol-agnostic tests (don't care if HTTP, WS, or other loader)
   - Should test via IOR calls only (true Web4 testing)

### **2. New Web4 Principles to Add**

#### **Web4 Principle 17: Component Instance Pattern (startComponent)**
> "Components (like `HTMLRoute`) are classes. Instances (like `homeRoute`) are created from components via `startComponent(scenario)`. This ensures dependencies are installed, code is compiled, and the instance is ready. Use lazy getters to load instances on demand. ALWAYS store instances using `Reference<T>` (Web4 Principle 5), never `Type | null`."

**Correct Understanding**:
- `HTMLRoute` = **Component** (the class, versioned, in `components/HTMLRoute/0.3.21.7/`)
- `homeRoute` = **Instance** (configured scenario for that component)
- `startComponent()` = Creates instances from components (like `npm start`)
- `Reference<T>` = **MANDATORY** for all nullable references (Web4 Principle 5)

**Pattern**:
```typescript
// ✅ CORRECT: Store instance using Reference<T> (Web4 Principle 5!)
private homeRouteInstance: Reference<HTMLRoute> = null;

// ✅ CORRECT: Lazy getter creates instance from component
get homeRoute(): HTMLRoute {
    if (!this.homeRouteInstance) {
        // Like: once startComponent HTMLRoute 0.3.21.7
        this.homeRouteInstance = HTMLRoute.start({
            scenario: {
                ior: { uuid: uuidv4(), component: 'HTMLRoute', version: '0.3.21.7' },
                owner: 'system',
                model: {
                    pattern: '/',
                    method: HttpMethod.GET,
                    provider: () => this.component!.serveStatus(),
                    priority: 50
                }
            }
        });
    }
    return this.homeRouteInstance;
}

// ❌ WRONG: Using Type | null (violates Web4 Principle 5!)
private homeRouteInstance: HTMLRoute | null = null;

// ❌ WRONG: Using underscore (forbidden in Web4/Radical OOP)
private _homeRoute: Reference<HTMLRoute> = null;
get homeRoute(): HTMLRoute {
    if (!this._homeRoute) {  // ← NO UNDERSCORES!
        ...
    }
}

// ❌ WRONG: Treating route as component (it's an instance!)
get HTMLRoute(): HTMLRoute {  // ← HTMLRoute is the COMPONENT, not the instance
    ...
}

// Register routes (just calls getters to create instances)
this.httpRouter.registerRoute(this.homeRoute);  // ← Creates instance via getter
```

**Clarification**:
- **Component**: `HTMLRoute`, `ScenarioRoute`, `IORRoute` (classes in versioned folders)
- **Instance**: `homeRoute`, `healthRoute`, `demoRoute` (configured scenarios)
- **No underscores**: Never use `_variable` names (already forbidden in Web4/Radical OOP)
- **Use Reference<T>**: ALWAYS use `Reference<T>` instead of `Type | null` (Web4 Principle 5)
- **`startComponent()`**: Factory method that ensures component is built and ready

#### **Web4 Principle 18: Black-Box Testing Only**
> "Tests must be protocol-agnostic. Tests use IOR calls and don't know if communication happens via HTTP, WebSocket, or any other loader. White-box tests that check implementation details are forbidden."

**Pattern**:
```typescript
// ✅ BLACK-BOX: Protocol-agnostic
const health = await kernel.loadViaIOR('ior:scenario:REST:https://localhost:42777/health');
expect(health.model.status).toBe('healthy');

// ❌ WHITE-BOX: Knows about HTTP
const response = await fetch('http://localhost:42777/health');
expect(response.status).toBe(200);
```

#### **Web4 Principle 19: Server Binding vs URL Host**
> "Distinguish between 'host for URLs' (FQDN for scenarios/IORs) and 'interface to bind to' (IP address/interface name for server.listen()). Use `0.0.0.0` or specific IP for binding, use FQDN for URLs."

**Pattern**:
```typescript
// ✅ CORRECT: Separate concerns
model.urlHost = 'mcdonges-3.fritz.box';  // For URLs, scenarios, IORs
model.bindInterface = '0.0.0.0';         // For server.listen()
server.listen(port, model.bindInterface); // Bind to interface
const ior = `https://${model.urlHost}:${port}/path`; // Use FQDN in IOR
```

#### **Phase 8.3: Component Instance Pattern (NEXT ITERATION)**
**Impact**: Medium - Architectural improvement  
**Effort**: 4 hours

**Correct Pattern (Web4 Principle 5: Reference<T>)**:
```typescript
// ServerHierarchyManager.ts

// ✅ CORRECT: Use Reference<T> for nullable instance storage
private homeRouteInstance: Reference<HTMLRoute> = null;
private demoRouteInstance: Reference<HTMLRoute> = null;
private healthRouteInstance: Reference<ScenarioRoute> = null;

// ✅ CORRECT: Lazy getter creates instance via startComponent
get homeRoute(): HTMLRoute {
    if (!this.homeRouteInstance) {
        // Like: once startComponent HTMLRoute 0.3.21.7 scenario.json
        this.homeRouteInstance = HTMLRoute.start({
            scenario: {
                ior: { 
                    uuid: uuidv4(), 
                    component: 'HTMLRoute',  // ← Component name
                    version: '0.3.21.7' 
                },
                owner: 'system',
                model: {
                    uuid: uuidv4(),
                    name: 'HomeRoute',  // ← Instance name
                    pattern: '/',
                    method: HttpMethod.GET,
                    priority: 50,
                    statistics: { /* ... */ }
                }
            },
            provider: () => this.component!.serveStatus()
        });
    }
    return this.homeRouteInstance;
}

// ❌ WRONG: Using Type | null (should use Reference<Type>)
private homeRouteInstance: HTMLRoute | null = null;

// ❌ WRONG: Using underscore (forbidden!)
private _homeRoute: Reference<HTMLRoute> = null;

// ✅ CORRECT: Registration just calls getters
private registerRoutes(): void {
    // Initialize IOR router
    if (!this.iorRouter.model.uuid) {
        this.iorRouter.init(undefined, this.component);
    }
    
    // Register routes via getters (creates instances on demand)
    this.httpRouter.registerRoute(this.iorRoute);      // ← IORRoute instance
    this.httpRouter.registerRoute(this.homeRoute);     // ← HTMLRoute instance
    this.httpRouter.registerRoute(this.demoRoute);     // ← HTMLRoute instance
    this.httpRouter.registerRoute(this.healthRoute);   // ← ScenarioRoute instance
    
    console.log(`📍 Registered ${this.httpRouter.model.routes.length} route instances`);
}
```

**Changes**:
1. Add `HTMLRoute.start(config)` static method
2. Convert route instantiation to lazy getters
3. Remove functional `new HTMLRoute()` + `uuidv4()` calls
4. Use `Reference<HTMLRoute>` (not `HTMLRoute | null`)
5. Each getter implements `startComponent` pattern

**Files**:
- Update: `HTMLRoute.ts`, `ScenarioRoute.ts`, `IORRoute.ts` (add `static start()`)
- Update: `ServerHierarchyManager.ts` (add lazy getters, use Reference<T>)
