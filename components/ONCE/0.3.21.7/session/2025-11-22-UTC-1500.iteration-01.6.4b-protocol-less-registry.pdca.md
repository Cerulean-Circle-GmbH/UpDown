## **P - PLAN**

### **1. Problem Statement**

**Current Situation**:
- ServerRegistry uses **protocol-based messages** for registration
- WebSocket messages: `server-register`, `server-unregister`, `server-list-request`
- Violates **Web4 Principle 11**: Protocol-Less Communication
- State is transmitted via messages instead of scenario replication

**Example of Current Violation**:
```typescript
// ❌ PROTOCOL-BASED: Message passing
ws.send(JSON.stringify({
    type: 'server-register',
    data: {
        uuid: this.serverModel.uuid,
        host: this.serverModel.host,
        port: this.serverModel.port,
        // ... more data
    }
}));

// Primary receives and parses message
case 'server-register':
    this.serverRegistry.set(data.uuid, {
        model: data,
        lastSeen: new Date().toISOString()
    });
```

**Why This Violates Web4**:
- Objects should **replicate state** (scenarios), not **send messages**
- No protocols - just object state hibernation/rehydration
- Instances should have SAME UUID (clones), not different IDs

### **2. Target Situation (TO-BE)**

**Protocol-Less Registry**:
```typescript
// ✅ SCENARIO-BASED: State replication
// Client server saves its scenario (already happening!)
await this.saveScenario(this.getScenario());

// Primary discovers scenarios from filesystem
const scenarios = await this.discoverServerScenarios();

// Registry stores scenarios, not messages
this.serverRegistry.set(uuid, {
    scenario: loadedScenario,  // Full scenario object

```

```typescript
/**
 * Discover server scenarios from filesystem
 * Replaces WebSocket registration messages
 * @returns Array of discovered scenarios
 */
async discoverServerScenarios(): Promise<Scenario<LegacyONCEScenario>[]> {
    const scenarioDir = path.join(
        this.projectRoot,
        'scenarios',
        ...this.getDetectDomainPath(),
        this.serverModel.hostname,
        'ONCE',
        this.versionFromComponent
    );
    
    // Find all scenario files
    const files = glob.sync(`${scenarioDir}/*.scenario.json`);
    
    const scenarios = [];
    for (const file of files) {
        try {
            const content = fs.readFileSync(file, 'utf8');
            const scenario = JSON.parse(content);
            scenarios.push(scenario);
        } catch (error) {
            console.warn(`⚠️  Failed to load scenario: ${file}`, error);
        }
    }
    
    return scenarios;
}
```

#### **Task 4: Update performHousekeeping**

Enhance existing housekeeping to register discovered servers:

```typescript
async performHousekeeping(): Promise<{ deleted: number; discovered: number }> {
    console.log('🧹 Performing primary server housekeeping...');
    
    const scenarios = await this.discoverServerScenarios();
    let discovered = 0;
    
    for (const scenario of scenarios) {
        const uuid = scenario.ior.uuid;
        
        // Skip self
        if (uuid === this.serverModel.uuid) continue;
        
        // Check if already registered
        if (!this.serverRegistry.has(uuid)) {
            console.log(`📡 Discovered new server: ${uuid}`);
            this.serverRegistry.set(uuid, {
                scenario: scenario,
                lastSeen: new Date().toISOString()
            });
            discovered++;
            
            // Broadcast to browser clients
            this.broadcastToBrowserClients({
                type: 'server-discovered',
                data: scenario
            });
        }
    }
    
    // ... existing deletion logic ...
    
    return { deleted, discovered };
}
```

1. **Deprecation > Deletion**
   - Marking methods as deprecated allows gradual migration
   - Warnings guide developers without breaking builds
   - Keeps browser client functionality intact

   - No need to parse/reconstruct models
   - Aligns perfectly with Web4 "protocol-less" principle

3. **Filesystem = Source of Truth**
   - Scenario files persist across restarts
   - No need for in-memory registration protocol
   - Automatic discovery via housekeeping
