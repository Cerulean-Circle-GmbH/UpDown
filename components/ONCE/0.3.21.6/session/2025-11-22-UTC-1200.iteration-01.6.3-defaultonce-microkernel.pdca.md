## **P - PLAN**

### **1. Problem Statement**

**Current Situation**:
- Route handlers in `ServerHierarchyManager` contain **functional logic** (inline JSON, closures)
- State scattered across handlers instead of being in `this.model`
- Violates **Radical OOP** principle: methods should operate on model, not create state
- Routes are **protocol-based** (HTTP endpoints) not **IOR-based method invocations**

**Example of Current Violation**:
```typescript
// ❌ FUNCTIONAL: State created in handler
} else if (url.pathname === '/health') {
    res.end(JSON.stringify({
        status: 'running',
        uuid: this.serverModel.uuid,
        // ... inline data creation
    }));
}
```

**Web4 Architecture Violation**:
- DefaultONCE should be a **micro kernel**
- Routes should **delegate to methods**
- Methods should **return scenarios** (state hibernation format)
- Template methods exist but are in wrong class (ServerHierarchyManager)

### **2. Target Situation (TO-BE)**

**DefaultONCE as Micro Kernel**:
```typescript
class DefaultONCE {
    // ✅ Empty constructor (Web4 pattern)
    constructor() { /* minimal */ }
    
    // ✅ All state in model
    model: ONCEModel;
    
    // ✅ Methods return state/scenarios
    getHealth(): HealthModel { /* operates on this.model */ }
    getServers(): ServerListModel { /* operates on this.model */ }
    
    // ✅ Template rendering methods
    serveOnceClient(): string { /* renders from this.model */ }
    serveDemoHub(): string { /* renders from this.model */ }
    
    // ✅ Action methods
    async startServer(): Promise<void> { /* uses this.model */ }
    async discoverServers(): Promise<DiscoveryResult> { /* uses this.model */ }
    async shutdownAll(): Promise<void> { /* uses this.model */ }
}
```

**Route Handlers Delegate to Methods**:
```typescript
// ✅ RADICAL OOP: Route delegates to method
if (url.pathname === '/health') {
    const health = this.component.getHealth();
    res.end(JSON.stringify(health));
}
```

**IOR-based Method Invocation Alignment**:
```
HTTP GET /health
  ↓
IOR: ior:https://localhost:42777/ONCE/0.3.21.6/{uuid}/getHealth
  ↓
Method: once.getHealth()
  ↓
Returns: HealthModel (state)
```

### **3. Root Cause Analysis**

**Why are routes functional instead of OOP?**

**5W1H**:
- **What**: Route handlers contain inline logic instead of delegating to methods
- **Why**: ServerHierarchyManager evolved from v0.2.0.0 without Radical OOP principles
- **When**: During migration from 0.2.0.0 → 0.3.20.0
- **Where**: `ServerHierarchyManager.ts` lines 265-380 (HTTP request handler)
- **Who**: Migration preserved functional patterns for speed
- **How**: Need to extract logic into DefaultONCE methods

**Fishbone (Cause & Effect)**:
```
Methods:                         People:
- No getHealth()        ──┐      ┌── Migration speed priority
- No getServers()         │      │
                          ▼      ▼
Materials:                 FUNCTIONAL
- Inline JSON        ──►   ROUTE     ◄── Fast implementation
- No model state           HANDLERS
                          ▲      ▲
Process:                   │      │
- Direct response    ──┘      └── No OOP review
```

```typescript
getServers(): ServerListModel {
```

```typescript
async startClientServer(): Promise<void> {
    const { exec } = await import('child_process');
    const componentDir = this.model.componentRoot;
    
    if (!componentDir) {
        throw new Error('Component not initialized');
    }

}
```

```typescript
/**
 * Shutdown all servers (infrastructure method)
 * Called by DefaultONCE.shutdownAll()
 */
async shutdownAllServers(): Promise<void> {
    // Shutdown all client servers first
    for (const [uuid, entry] of this.serverRegistry.entries()) {
        if (entry.websocket && entry.websocket.readyState === WebSocket.OPEN) {
            console.log(`🛑 Sending shutdown to client: ${uuid}`);
            entry.websocket.send(JSON.stringify({ type: 'shutdown-command' }));
        }
    }
    
    await this.sleep(2000); // Let clients shutdown
    
    // Shutdown self
    await this.stopServer();
}
```
