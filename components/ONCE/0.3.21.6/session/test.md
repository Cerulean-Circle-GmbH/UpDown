## PROBLEM STATEMENT

### Critical CMM3 Violation

The assistant manually created files in ONCE 0.3.21.7 instead of using the CMM3 toolchain, causing:

1. **Missing CLI Script**: `scripts/versions/once-v0.3.21.7` does not exist
2. **Wrong File Permissions**: `components/ONCE/0.3.21.7/once` has incorrect permissions
3. **Process Violation**: Manual file creation bypasses CMM3 quality gates and automation
4. **Wrong Version**: Created descriptor in 0.3.21.7 instead of 0.3.21.6

### Root Cause

The chicken-and-egg problem was mishandled:
- ONCE 0.3.21.7 was created manually using `once upgrade nextBuild` from 0.3.21.6
- But 0.3.21.7 already exists (created by `upgrade nextBuild`)
- The assistant should have created `ONCE.component.json` in 0.3.21.6 BEFORE upgrading
- Then the upgrade process would have copied it to 0.3.21.7 correctly

## ROOT CAUSE ANALYSIS

### Why Manual Creation Failed

1. **Process Bypassed**: Manual file creation skips CMM3 automation
2. **Missing Steps**: Upgrade tool handles:
   - CLI script generation (`scripts/versions/once-v0.3.21.X`)
   - File permission setting (executable `once` script)
   - Symlink copying from previous version
   - Package.json updates
   - README updates
3. **Wrong Version Target**: Should have worked in 0.3.21.6, not 0.3.21.7

### CMM3 Principle Violated

**Web4 Principle 0: CMM3 Process Authority**
> "Never manually create artifacts that are generated by CMM3 tools. Tools ensure consistency, permissions, and process adherence."

### Lessons Learned

**What Went Wrong**:
1. Manual file creation bypassed CMM3 quality gates
2. Chicken-and-egg problem solved in wrong order
3. Worked in future version (0.3.21.7) instead of current (0.3.21.6)

**Correct Approach**:
1. Always work in CURRENT version
2. Create artifacts in current version BEFORE upgrading
3. Let upgrade tool copy/migrate artifacts
4. Regenerate version-specific artifacts after upgrade
5. Trust CMM3 tools for permissions and scripts

**New Rule**:
> "The ONLY manual creation allowed is a bootstrap scenario in the CURRENT version to enable initial loading. All other artifacts MUST be created by CMM3 tools."
