## **📊 SUMMARY**

**OBJECTIVE**: Reimplement component lifecycle events using TRUE Web4 Radical OOP pattern, replacing functional event handlers with proper Observer Pattern implementation.

**SCOPE**: 
- Define lifecycle interface in `DefaultOnceKernel` base class
- Implement Observer Pattern for lifecycle subscribers (NO arrow functions/callbacks)
- Map lifecycle state transitions to scenario state
- Override lifecycle methods in environment-specific kernels as needed
- Event flow: `init() → start() → ready() → stop() → destroy()`

**CURRENT STATE** (Problems):
- ❌ `emitEvent()` method uses **functional callbacks** (`LifecycleEventHandler = (event: any) => void`)
- ❌ `eventHandlers` stored as `Map<string, Function[]>` (functional anti-pattern)
- ❌ Lifecycle events exist but NO proper Observer Pattern
- ❌ `LifecycleObserver` interface defined but NOT used
- ❌ State transitions not mapped to `model.state`
- ❌ No validation of lifecycle state transitions

**TARGET STATE** (Solutions):
- ✅ `LifecycleManager` interface with proper OOP methods
- ✅ Observer Pattern with `attachObserver()`, `detachObserver()`, `notifyObservers()`
- ✅ Lifecycle methods update `model.state: LifecycleState`
- ✅ Scenario state reflects current lifecycle state
- ✅ NO callbacks, NO arrow functions, just OOP methods
- ✅ State transition validation (e.g., can't START from STOPPED without INIT)

**ARCHITECTURAL PRINCIPLE**:
```typescript
// ❌ BEFORE (Functional Anti-Pattern):
private eventHandlers: Map<LifecycleEventType, ((event: any) => void)[]>;
await this.emitEvent(LifecycleEventType.BEFORE_START);

// ✅ AFTER (Radical OOP with Observer Pattern):
private observers: LifecycleObserver[] = [];
this.notifyObservers(LifecycleEventType.BEFORE_START);
// Observers implement: onBeforeStart(kernel: OnceKernel): void
```

**Lifecycle State Machine**:
```
CREATED (constructor)
    ↓
    init()
    ↓
INITIALIZING → INITIALIZED
    ↓
    start()
    ↓
STARTING → RUNNING
    ↓
    pause() (optional)
    ↓
PAUSING → PAUSED
    ↓
    resume() (optional)
    ↓
RESUMING → RUNNING
    ↓
    stop()
    ↓
STOPPING → STOPPED
    ↓
    destroy()
    ↓
SHUTTING_DOWN → SHUTDOWN

ERROR (from any state on failure)
```

#### **Task 1.13.2: LifecycleManager Interface Design**

**New Interface** (`src/ts/layer3/LifecycleManager.interface.ts`):
```typescript
import type { LifecycleObserver } from './LifecycleObserver.interface.js';
import type { LifecycleEventType } from './LifecycleEventType.enum.js';
import type { LifecycleState } from './LifecycleState.enum.js';

/**
 * LifecycleManager - Manages lifecycle events and observers
 * Radical OOP Pattern - NO callbacks, just observer registration
 */
export interface LifecycleManager {
    /**
     * Attach lifecycle observer
     * Observer must implement LifecycleObserver interface
     */
    attachObserver(observer: LifecycleObserver): void;
    
    /**
     * Detach lifecycle observer
     */
    detachObserver(observer: LifecycleObserver): void;
    
    /**
     * Notify all observers of lifecycle event
     * Calls the appropriate method on each observer (convention-over-configuration)
     * 
     * Example: notifyObservers(LifecycleEventType.BEFORE_START)
     *          → calls observer.onBeforeStart(this)
     */
    notifyObservers(eventType: LifecycleEventType, data?: any): void;
    
    /**
     * Get current lifecycle state
     */
    getLifecycleState(): LifecycleState;
    
    /**
     * Validate state transition
     * Returns true if transition is allowed, false otherwise
     */
    canTransitionTo(targetState: LifecycleState): boolean;
    
    /**
     * Transition to new state
     * Validates transition, updates model.state, notifies observers
     * 
     * @throws Error if transition invalid
     */
    transitionTo(targetState: LifecycleState, eventType: LifecycleEventType, data?: any): void;
}
```

#### **Task 1.13.3: Implementation Strategy**

**Implementation Plan**:

1. **Create `LifecycleManager.interface.ts`** (Layer 3) ✅ **NEW**
   - Interface for lifecycle management
   - No implementation, just contract

2. **Implement Lifecycle in `DefaultOnceKernel`** (Layer 2)
   - Add `private observers: LifecycleObserver[] = []`
   - Add `protected state: LifecycleState = LifecycleState.CREATED`
   - Implement `attachObserver()`, `detachObserver()`, `notifyObservers()`
   - Implement `canTransitionTo()`, `transitionTo()`
   - Add state validation logic

3. **Define Lifecycle Methods in `DefaultOnceKernel`**
   - `init()` - transition CREATED → INITIALIZING → INITIALIZED
   - `start()` - transition INITIALIZED → STARTING → RUNNING
   - `pause()` - transition RUNNING → PAUSING → PAUSED
   - `resume()` - transition PAUSED → RESUMING → RUNNING
   - `stop()` - transition RUNNING → STOPPING → STOPPED
   - `destroy()` - transition STOPPED → SHUTTING_DOWN → SHUTDOWN

4. **Update `NodeJsOnce`**
   - Remove `emitEvent()` functional implementation
   - Replace with `notifyObservers()` calls
   - Update `startServer()`, `init()` to use new lifecycle methods
   - Override lifecycle methods if Node.js-specific behavior needed

5. **Update `BrowserOnce`**
   - Implement browser-specific lifecycle if needed
   - Otherwise inherit from `DefaultOnceKernel`

6. **Convention-over-Configuration Event Dispatch**
   - `LifecycleEventType.BEFORE_START` → `observer.onBeforeStart(kernel)`
   - `LifecycleEventType.AFTER_START` → `observer.onAfterStart(kernel)`
   - Method name derived from event type (camelCase conversion)

## **📋 Risk Assessment**

**High Priority Risks**:

1. **Risk**: Breaking existing code that depends on functional `emitEvent()`
   - **Mitigation**: Comprehensive test coverage, gradual migration
   - **Contingency**: Keep functional implementation temporarily, add deprecation warnings

2. **Risk**: Observer Pattern more complex than functional approach
   - **Mitigation**: Good documentation, clear examples
   - **Contingency**: Provide helper methods to simplify observer creation

3. **Risk**: State transition validation too restrictive
   - **Mitigation**: Thorough state machine design, allow override for special cases
   - **Contingency**: Add escape hatch for forced transitions (with warnings)

**Medium Priority Risks**:

4. **Risk**: Convention-over-configuration method dispatch fails at runtime
   - **Mitigation**: Validate observer implements required methods
   - **Contingency**: Fallback to generic `onLifecycleEvent(type, data)` method

5. **Risk**: Performance impact from observer notifications
   - **Mitigation**: Benchmark tests, async notifications where appropriate
   - **Contingency**: Add option to disable lifecycle notifications in production
