# Scenario Location Bug Fix

**Date**: 2025-11-27 UTC 11:30  
**Iteration**: 1.15  
**Status**: ✅ FIXED

## Problem

Scenarios were found in WRONG location:
- ❌ `§/components/ONCE/0.3.21.6/test/data/components/ONCE/0.3.21.6/scenarios/`
- ✅ CORRECT: `§/components/ONCE/0.3.21.6/test/data/scenarios/`

## Root Cause

Component was copied with scenarios/ directory during test isolation setup.
When tests ran, old scenarios from previous test runs existed there.

## The Code is CORRECT

`ScenarioManager.ts` line 52-71 correctly derives projectRoot:
```typescript
private get projectRoot(): string {
    // ... derives from component path properly
    return resolve(componentsDir, '..'); // -> project root
}
```

Scenarios are ALWAYS saved to `${projectRoot}/scenarios/`.

## Fix Applied

1. ✅ Removed wrong scenario directory:
   ```bash
   rm -rf test/data/components/ONCE/0.3.21.6/scenarios
   ```

2. ✅ Verified correct structure:
   - Scenarios: `test/data/scenarios/` ✅
   - Component: `test/data/components/ONCE/0.3.21.6/` (NO scenarios/) ✅

## Prevention

The rsync copy command in test isolation should exclude scenarios/:
```bash
rsync -a --exclude='test/data' --exclude='scenarios' ...
```
