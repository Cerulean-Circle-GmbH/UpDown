### **Context**

**User Question**: "when will once-client.html be using the once kernel instead of the hardcoded functional shit scripts?"

**Current State**: Browser HTML has ~400 lines of hardcoded functional JavaScript:
- Manual WebSocket connection handling
- Protocol-based message passing (`type: 'scenario-p2p'`, etc.)
- Direct `fetch()` calls to `/health`, `/servers` endpoints
- Inline stats tracking
- UUID generation in browser
- **NO use of `global.ONCE` kernel**
- **WRONG terminology**: Uses "client/server" instead of "peer kernels"

**Web4 Principle 13**: P2P Kernel Everywhere - No Client/Server Terminology
- ONCE = Object Network Communication Engine (P2P micro kernel)
- Runs everywhere: Browser, Node.js, Worker, PWA, Edge
- NO "client/server" - Every ONCE is a peer kernel
- Terminology: "ONCE kernel", "peer", "node" (NOT "client", "server")

**This is a GAP**: Browser kernel integration was NOT explicitly planned in original tracking PDCA.

---

### **Objective**

Refactor browser peer to use the ONCE kernel (global.ONCE) instead of hardcoded functional JavaScript, implementing TRUE Radical OOP P2P architecture.

### **Problem Statement**

**Issue**: Browser peers are using outdated functional programming patterns from before Web4 architecture:
1. **No kernel integration**: Browser doesn't load or use `global.ONCE`
2. **Protocol-based messaging**: Uses message types instead of scenario state transfer
3. **Direct HTTP calls**: `fetch('/health')` instead of IOR-based method invocation
4. **Inline JavaScript**: 400+ lines embedded in HTML
5. **Functional patterns**: No objects, no Radical OOP
6. **Wrong terminology**: "client" instead of "peer kernel"

**Impact**:
- Browser code doesn't follow Web4 principles
- Can't leverage ONCE kernel features (IOR resolution, method invocation, etc.)
- Inconsistent with P2P architecture
- Hard to maintain and test
- Violates Radical OOP discipline
- Confusing "client/server" terminology violates P2P paradigm

### **Design**

#### **Architecture**

**BEFORE** (Functional - BAD):
```javascript
// Inline in HTML
let ws = new WebSocket('ws://localhost:8080');
ws.onmessage = (event) => {  // ❌ Arrow function callback
    const data = JSON.parse(event.data);
    if (data.type === 'scenario-p2p') {  // ❌ Protocol-based
        // Handle message
    }
};

fetch('/health').then(r => r.json()).then(data => {  // ❌ Direct HTTP, promises
    // Update UI
});
```

**AFTER** (Radical OOP - GOOD):
```javascript
// External module: once-browser-kernel.js
import { ONCE } from '/ONCE/0.3.21.6/bundle.js';

// Browser ONCE kernel peer (NOT "client"!)
class BrowserONCEKernel {
    constructor() {
        // ✅ Empty constructor (Radical OOP)
        this.model = null;
    }
    
    async init(scenario) {
        // ✅ Initialize with scenario
        this.model = {
            kernel: window.global.ONCE,
            peerHost: scenario?.peerHost || window.location.host,  // NOT "serverHost"
            stats: { messagesSent: 0, messagesReceived: 0 },
            connectionTime: null,
            peers: []  // NOT "servers"
        };
        
        // ✅ Connect to network (method, not callback)
        await this.connect();
        
        // ✅ Poll for updates (method-based, not event callbacks)
        this.startPolling();
        
        return this;
    }
    
    async connect() {
        // ✅ Use kernel's WebSocket, not manual connection
        console.log('🔌 Connecting to ONCE P2P network...');
        await this.getHealth();  // Initial health check
    }
    
    async getHealth() {
        // ✅ IOR-based method invocation (not direct fetch)
        const health = await this.model.kernel.invokeMethod('getHealth', {});
        this.updateHealthDisplay(health);
    }
    
    async getPeers() {
        // ✅ IOR-based method invocation (NOT "getServers")
        const peers = await this.model.kernel.invokeMethod('getPeers', {});
        this.model.peers = peers.peers;
        this.updatePeersDisplay(peers);
    }
    
    startPolling() {
        // ✅ Method-based polling (not event listeners)
        // Poll every 5 seconds for state updates
        setInterval(() => {
            this.getHealth();
            this.getPeers();
        }, 5000);
    }
    
    updateHealthDisplay(health) {
        // ✅ Method operating on model
        const statusEl = document.getElementById('connectionStatus');
        statusEl.textContent = health.status === 'healthy' ? '🟢 Connected' : '🔴 Disconnected';
    }
    
    updatePeersDisplay(peers) {
        // ✅ Method operating on model (NOT "servers")
        const peersEl = document.getElementById('connectedPeers');
        peersEl.textContent = `${peers.peers.length} peers`;
    }
    
    // ✅ NO CALLBACKS, NO ARROW FUNCTIONS - Just methods!
    // ✅ NO event listeners (once.on) - Use polling or Observer Pattern
    // ✅ P2P terminology: peers, nodes, kernels (NOT client/server)
}

// Usage (P2P kernel instantiation)
const kernel = new BrowserONCEKernel();
await kernel.init({ peerHost: window.location.host });
```

**Key Differences**:
1. ❌ **NO arrow functions** (`() => {}`)
2. ❌ **NO callbacks** (`.then()`, `.onmessage`)
3. ❌ **NO event listeners** (`once.on('event', callback)`)
4. ❌ **NO client/server terminology** (use "peer", "kernel", "node")
5. ✅ **YES methods** operating on `this.model`
6. ✅ **YES polling** or Observer Pattern (method-based)
7. ✅ **YES IOR-based** method invocation
8. ✅ **YES P2P terminology** (peers, not clients/servers)
