# **Iteration Impact Analysis: Learnings from 1.6 & 1.7**

**Date**: 2025-11-22 UTC 19:00

## **Key Learnings from Iteration 1.6 & 1.7**

### **1. Web4 Principles Added** ✅

**New Principles from Iteration 1.6**:
- **Principle 10**: Global ONCE Singleton Pattern (`global.ONCE`)
- **Principle 11**: Protocol-Less Communication (REST state transfer via scenarios)
- **Principle 12**: IOR-based Method Invocation (HTTP/CLI method routing)

**Impact**: These are FUNDAMENTAL architectural patterns that affect ALL future iterations.

---

### **2. Micro Kernel Pattern** ✅

**Learning**: DefaultONCE acts as a micro kernel:
- Empty constructor + `static start()` factory
- Routes delegate to methods (not hardcoded handlers)
- Methods operate on `this.model` (Radical OOP)
- Global singleton registration in all environments

**Impact**: All component implementations should follow this pattern.

---

### **3. Protocol-Less Communication** ✅

**Learning**: Real-time state transfer, NOT message passing:
- Servers send full scenarios (not protocol messages with `type` field)
- Primary acts on scenario state (register/deregister based on scenario)
- WebSocket is for state transfer (like REST, but real-time)
- NO `ONCEScenarioMessage`, NO `type: 'register'`, etc.

**Impact**: Any iteration dealing with WebSocket, messaging, or P2P needs major revision.

---

### **4. IOR-based Method Invocation** ✅

**Learning**: HTTP endpoints map directly to object methods:
- `GET /ONCE/method?param1=value1` → `DefaultONCE.method({ param1: 'value1' })`
- Query params map to model properties
- Like CLI routing, but over HTTP
- Error handling returns available methods list

**Impact**: HTTPRouter (Iteration 5) needs to support IOR-based invocation, not just file/handler routing.

---

### **5. Browser Client Refactoring** ⚠️

**Learning**: Browser clients need kernel integration:
- Load `global.ONCE` kernel in browser
- Use IOR-based method invocation (not direct `fetch`)
- Use scenario-based state transfer (not protocol messages)
- Extract inline JavaScript to external modules
- Implement Radical OOP in browser

**Impact**: This was NOT in the tracking PDCA! Added as Iteration 1.7.
