### **What Needs Improvement**

1. ⚠️ **Multi-Server Test Timeouts**
   - Tests 3, 4, 5 all involve client-primary communication
   - Need investigation:
     - WebSocket connection issues?
     - Cleanup between tests?
     - Port conflicts?

2. ⚠️ **Test Isolation for Multi-Server**
   - Setting `projectRoot` might not be enough
   - Client servers may be trying to connect to wrong primary
   - Need to verify both servers using same `testProjectRoot`

3. ⚠️ **Deprecated Protocol Warnings**
   - Client servers still trying to send registration messages
   - These are deprecated and don't work anymore
   - May be causing connection hangs

### **Lessons Learned**

1. **Housekeeping Timing**
   - Housekeeping runs DURING `startServer()`, not after
   - Tests must create scenarios BEFORE starting server

2. **Test Isolation is Critical**
   - Must set `projectRoot` for ALL servers (primary AND clients)
   - Otherwise servers look in wrong directories

3. **Protocol Deprecation Impact**
   - Removing protocol-based registration affects tests MORE than production
   - Tests relied heavily on instant WebSocket registration
   - Filesystem discovery is async and requires explicit triggering
