## **🔴 REMAINING FAILURES (5 Tests)**

### **Root Cause Analysis**

All 5 failures are in **good black-box tests** but have **legitimate bugs**:

#### **1. `cli-path-authority.test.ts` (1 failure)**

**Issue**: White-box anti-pattern disguised as comparison test
```typescript
const cli = new DefaultCLI();  // ❌ Direct instantiation
cli.init();
expect(cli.model.projectRoot).toBe(productionProjectRoot);  // ❌ Internal state
```

**Problem**: `DefaultCLI` doesn't have path discovery like `NodeJsOnce`

**Recommendation**: **DEPRECATE** this test - it's testing internal state

---

#### **2. `lifecycle-management.test.ts` (3 failures)**

**Test A: "should register new client server dynamically"**

**Issue**: Port allocation mismatch
```typescript
// Test expects port 8080
const clientScenarios = getScenarioFiles(scenarioDir).filter(f => {
    return port === 8080;  // ❌ Hardcoded port
});

// But server starts on dynamic port 8085
🔵 Started as CLIENT SERVER on port 8085
```

**Root Cause**: Test doesn't specify port in scenario, ONCE assigns next available port

**Fix Options**:
- A) Update test to look for ANY client port (not 8080)
- B) Add port specification to test scenario
- C) Update test to read actual port from server startup logs

**Recommendation**: **Option A** - Test should be port-agnostic

---

**Test B: "should handle client disconnect and cleanup"**

**Issue**: Same as Test A - port mismatch

---

**Test C: "should restart primary and rediscover running clients after crash"**

**Issue**: Timing + port mismatch
```
expected client1Scenarios.length to be 1
received: 0
```

**Root Causes**:
1. Port mismatch (looking for 8081, actual port dynamic)
2. Timing - scenarios may not be written yet
3. State transfer timing - scenarios may be in different state
