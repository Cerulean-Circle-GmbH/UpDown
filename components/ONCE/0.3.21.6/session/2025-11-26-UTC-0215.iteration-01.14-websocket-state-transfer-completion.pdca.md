### **Architecture Design**:

**Protocol-Less State Transfer** (Web4 Principle 11):
```
NodeJsOnce (Peer A)
    ↓ State changes (port starts, scenario updates)
    sendScenarioUpdate(scenario)
    ↓ WebSocket
    ↓
Primary Server (42777)
    ↓ Broadcasts to all connected peers
    ↓ WebSocket
    ↓
BrowserOnce (Peer B)
    receiveScenarioUpdate(scenario)
    ↓ Updates this.model.peers
    ↓ Notifies UI (via lifecycle events)
```

**State Synchronization**:
- Full scenario sent (not delta)
- No message type field (protocol-less)
- Scenario UUID identifies peer
- Timestamp for conflict resolution

## **A - ACT**

**Status**: ✅ **COMPLETE**

### **Lessons Learned**:

1. **Protocol-Less Communication is Simpler**:
   - Removed message envelope wrapping
   - Browser receives pure scenarios
   - State-based logic (no message type field)

2. **Lifecycle Broadcasting is Critical**:
   - Must broadcast on RUNNING, STOPPING, STOPPED
   - Peers need real-time state updates
   - Enables dynamic peer registry

3. **State Machine Handles Peer Management**:
   - Browser removes peers with STOPPED/SHUTDOWN
   - No explicit "deregister" message needed
   - Convention-over-configuration
