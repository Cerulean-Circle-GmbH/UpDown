**Objective:** Remove deprecated `ONCEModel`, consolidate `Web4TSComponentModel` with `CLIModel` to achieve DRY compliance.

### **Problem Analysis**

#### **Issue 1: ONCEModel Deprecated but Still Used**

**File:** `ONCEModel.interface.ts` (90 lines)

```typescript
// Line 15-16: Explicit deprecation marker
// ⚠️ DEPRECATED: Keeping type definition for backward compatibility until Iteration 1.6.3

export interface ONCEModel extends Model {
  uuid: string;
  name: string;
  origin: string;
  definition: string;
  // ... 70+ lines of deprecated properties
  messageTracker?: ONCEMessageTracker;  // ⚠️ DEPRECATED: Protocol-based messaging
}
```

**Current Usages (4 files):**

| File | Usage | Migration Path |
|------|-------|----------------|
| `NodeJsOnce.ts` | `extends DefaultOnceKernel<ONCEModel>` | Change to `ONCEPeerModel` |
| `NodeJsOnce.ts` | `modelDefault(): ONCEModel` | Change return type |
| `OnceOverView.ts` | Local interface `interface ONCEModel` | Delete, use ONCEPeerModel |
| `ONCEPeerModel.interface.ts` | Comment "Replaces ONCEModel" | Keep (documentation) |
| `ONCEModel.interface.ts` | The file itself | **DELETE** |

#### **Issue 2: Web4TSComponentModel Duplicates CLIModel**

**File:** `Web4TSComponentModel.interface.ts` (83 lines)

```typescript
// DUPLICATED from CLIModel:
export interface Web4TSComponentModel extends Model {
  projectRoot: string;           // ❌ Also in CLIModel
  componentsDirectory: string;   // ❌ CLIModel has componentsDir
  isTestIsolation: boolean;      // ❌ CLIModel has testDataDir concept
  testDataDirectory?: string;    // ❌ CLIModel has testDataDir
  // ...
}
```

**File:** `CLIModel.interface.ts` (44 lines)

```typescript
export interface CLIModel extends Model {
  projectRoot: string;          // ✅ Single point of truth
  componentsDir: string;        // ✅ Single point of truth
  scriptsDir: string;
  scriptsVersionDir: string;
  testDataDir: string;          // ✅ Single point of truth
  // ... completion context
}
```

**DRY Violation:** 4 properties duplicated:
1. `projectRoot` (identical)
2. `componentsDirectory` vs `componentsDir` (same concept, different names)
3. `testDataDirectory` vs `testDataDir` (same concept, different names)
4. Implicit test isolation logic duplicated
