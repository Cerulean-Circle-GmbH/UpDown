#### **Issue 2: Init Chain Chaos (DRY Violation) — TRON 2026-01-04**

**Entry Point:** `ONCECLI.ts` (Layer 5) — Terminal entry point when `./once` is called

**Problem:** Multiple init methods with mixed async/sync patterns:

```typescript
// ONCECLI.ts - THREE different init patterns!
constructor() {
  super();
  this.init();  // ❌ init called in constructor
}

async initComponent(): Promise<void> {  // ❌ Separate async init
  const component = new NodeJsOnce();
  await (component as any).getWeb4TSComponent();  // ❌ async in Layer 5
  this.component = await DelegationProxy.start(component) as any;  // ❌ async
}

init(scenario?: any): this {  // ❌ Third init method
  super.init(scenario);
  // ...
}

static async start(args: string[]): Promise<void> {
  const cli = new ONCECLI();
  await cli.initComponent();  // ❌ async init sequence
  await cli.execute(args);
}
```

```typescript
// UcpComponent.ts (161-163) - async wraps sync!
async init(scenario?: { model?: TModel }): Promise<this> {
  return this.initSync(scenario);  // ❌ Why async if just calling sync?
}

initSync(scenario?: { model?: TModel }): this {  // ❌ Redundant method
  // ... actual init logic
}
```

**Target Pattern (Web4 Radical OOP P6+P7):**
```typescript
// Simple, clear, SYNC init
let instance = new SomeWeb4Component().init(scenario);

// ALL async in Layer 4 orchestrators ONLY (P7)
```

**Reason:** External component initialization was the original reason for async init. Since all components are now insourced, we can consolidate to sync init.

export interface CLIModel extends Model {

1. `projectRoot` (identical)

3. `testDataDirectory` vs `testDataDir` (same concept, different names)
4. Implicit test isolation logic duplicated

### **RE-PLAN: NodeJsOnce Architecture with CLIModel**

**Current Problem:**
- NodeJsOnce uses `this.model.componentRoot`, `this.model.projectRoot` (28 usages)
- These path properties are in deprecated ONCEModel
- CLIModel already has these properties as single source of truth

**Correct Solution (Composition, NOT Duplication):**

```
NodeJsOnce
├── model: ONCEPeerModel          ← Peer network state (uuid, host, port, state, peers)
└── cli: CLIModel                 ← Path authority (projectRoot, componentsDirectory)
```

**Migration Pattern:**
```typescript
// BEFORE (DRY violation):
this.model.componentRoot
this.model.projectRoot

// AFTER (CLIModel as single source of truth):
this.cli.projectRoot
this.cli.componentsDirectory
```
