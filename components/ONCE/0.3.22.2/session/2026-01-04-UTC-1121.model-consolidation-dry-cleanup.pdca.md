#### **Issue 2: Init Chain Chaos (DRY Violation) вАФ TRON 2026-01-04**

**Entry Point:** `ONCECLI.ts` (Layer 5) вАФ Terminal entry point when `./once` is called

**Problem:** Multiple init methods with mixed async/sync patterns:

```typescript

  super();
  this.init();  // вЭМ init called in constructor
}

async initComponent(): Promise<void> {  // вЭМ Separate async init
  const component = new NodeJsOnce();
  await (component as any).getWeb4TSComponent();  // вЭМ async in Layer 5
  this.component = await DelegationProxy.start(component) as any;  // вЭМ async
}

init(scenario?: any): this {  // вЭМ Third init method
  super.init(scenario);
  // ...
}

static async start(args: string[]): Promise<void> {
  const cli = new ONCECLI();
  await cli.initComponent();  // вЭМ async init sequence
  await cli.execute(args);
}
```

```typescript
// UcpComponent.ts (161-163) - async wraps sync!
async init(scenario?: { model?: TModel }): Promise<this> {
  return this.initSync(scenario);  // вЭМ Why async if just calling sync?
}

initSync(scenario?: { model?: TModel }): this {  // вЭМ Redundant method
  // ... actual init logic
```

**Target Pattern (Web4 Radical OOP P6+P7):**
```typescript

// ALL async in Layer 4 orchestrators ONLY (P7)
```

**Reason:** External component initialization was the original reason for async init. Since all components are now insourced, we can consolidate to sync init.
