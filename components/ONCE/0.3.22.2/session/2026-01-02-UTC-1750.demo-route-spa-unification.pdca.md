## Problem Statement (TRON 2026-01-02)

> "Make sure https://mcdonges.fritz.box:42777/demo does not render via demo-lit.html or any other html view but as once.html with UcpRouter rendering `<once-over-view>`"

---

## P - PLAN

### Current State Analysis

**Server-side routing** (`ServerHierarchyManager.ts`):

| Route | Provider | HTML Template |
|-------|----------|---------------|
| `/` | `serveOnceApp()` | `once.html` ✅ |
| `/demo` | `serveDemoLit()` | `demo-lit.html` ❌ |
| `/demo/` | `serveDemoLit()` | `demo-lit.html` ❌ |
| `/demo-lit` | `serveDemoLit()` | `demo-lit.html` (legacy alias) |
| `/app` | `serveOnceApp()` | `once.html` ✅ |
| `/EAMD.ucp` | `serveOnceApp()` | `once.html` ✅ |

**Browser-side routing** (`BrowserOnceOrchestrator.ts`):

| Route | View Component | Status |
|-------|----------------|--------|
| `/` | `once-peer-default-view` | ✅ |
| `/demo` | `once-over-view` | ✅ Already registered |
| `/EAMD.ucp/*` | `folder-over-view` | ✅ |

**Problem**: The server routes `/demo` and `/demo/` serve `demo-lit.html` which:
1. Has its own asset preloading logic (duplicates orchestrator's)
2. Has its own ONCE kernel boot (duplicates `once.html`)
3. Violates DRY principle

**Target State**: ALL SPA routes should serve `once.html`, and `UcpRouter` handles view selection based on URL path.

## D - DO

### DR.1: Change `/demo` Route

**File**: `src/ts/layer2/ServerHierarchyManager.ts` (lines ~302-307)

**Before**:
```typescript
const demoRoute = new HTMLRoute().init();
demoRoute.model.uuid = this.idProvider.create();
demoRoute.setPattern('/demo', HttpMethod.GET);
demoRoute.setProvider(() => this.component!.serveDemoLit()); // ❌ Legacy HTML
demoRoute.model.priority = 50;
this.httpRouter.registerRoute(demoRoute);
```

**After**:
```typescript
const demoRoute = new HTMLRoute().init();
demoRoute.model.uuid = this.idProvider.create();
demoRoute.setPattern('/demo', HttpMethod.GET);
// ✅ DR.1: Use once.html SPA - UcpRouter renders <once-over-view>
// @pdca 2026-01-02-UTC-1750.demo-route-spa-unification.pdca.md
demoRoute.setProvider(() => this.component!.serveOnceApp());
demoRoute.model.priority = 50;
this.httpRouter.registerRoute(demoRoute);
```
