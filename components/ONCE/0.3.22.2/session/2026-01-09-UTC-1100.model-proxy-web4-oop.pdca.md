## P — PLAN

### Problem Statement

The current `UcpModel` (ModelProxy) uses **functional JavaScript Proxy pattern**:

```typescript
// CURRENT: Functional shit pattern
new Proxy(target, {
  get(target, prop) { ... },  // ← naked function
  set(target, prop, value) { ... }  // ← naked function
});
```

This forces `(this.model as any)` casts everywhere because:
1. TypeScript can't infer types through functional handlers
2. Dynamic property access loses type information
3. The Proxy constructor takes a plain object handler, not an OOP class

### Web4 Radical OOP Solution

Replace functional handlers with proper OOP classes:

```typescript
// PROPOSED: Web4 Radical OOP pattern
class Property<T> {
  private valueField: T;
  
  get value(): T { return this.valueField; }
  set value(v: T) { this.valueField = v; }
}

class ModelProxy<TModel extends Model> extends Proxy<TModel> {
  // Properties are typed Property instances, not functional handlers
}
```

## Analysis Required

### Questions for TRON

**Q1: Property Class Design**

Should `Property<T>` have:
- Just `get/set value`?
- Validation hooks?
- Change listeners (for reactive updates)?

```typescript
class Property<T> {
  get value(): T;
  set value(v: T);
  
  // Optional:
  validate?(newValue: T): boolean;
  onChange?(oldValue: T, newValue: T): void;
}
```

**Q2: How to handle dynamic properties?**

Current UcpModel allows any property to be set dynamically. In Web4 OOP:
- Should we require all properties to be declared?
- Or support a `dynamicProperties: Map<string, Property<unknown>>` for backwards compatibility?

**Q3: Integration with existing code**

How to migrate existing code that does:
```typescript
this.model.someProperty = value;
```

To the new pattern without breaking everything?
