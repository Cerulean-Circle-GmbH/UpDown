## Root Cause Analysis (CMM4)

### The Flow

1. User navigates to `/EAMD.ucp/components`
2. `FolderOverView.folderLoad('/EAMD.ucp/components')` is called
3. `FileOrchestrator.folderLoadFromServer('/EAMD.ucp/components')` is called
4. Creates: `new IOR().init('/EAMD.ucp/components?format=json')`
5. IOR calls `parseRelativeUrl()` which sets:
   ```
   protocol: 'https'
   host: 'localhost'
   port: 42777
   path: '/EAMD.ucp/components?format=json'
   iorString: 'https://localhost:42777/EAMD.ucp/components?format=json'
   ```
6. IOR.resolve() calls `performResolution()` with protocol `https`
7. HTTPSLoader.load() is called with URL `https://localhost:42777/EAMD.ucp/components?format=json`
8. **fetch() throws an error** (possibly SSL certificate issue in browser context)
9. HTTPSLoader reports "All IOR profiles exhausted"

### The Real Issue

The `HTTPSLoader.load()` is actually making the fetch request, but:
1. **The request succeeds on the server** — `StaticFileRoute` correctly maps `/EAMD.ucp/components` to `{projectRoot}/components`
2. **The browser is rejecting the response** — likely due to:
   - SSL certificate issues with localhost
   - CORS issues
   - Or the request timing out

Let me verify by checking the server logs...

**Hypothesis**: The `StaticFileRoute.handleRequest()` IS being hit (we can verify via server logs), but something in the HTTPSLoader is failing.

### FN.2: Fix Virtual Root Subpath Handling (30 min)

**Current Issue**: The `/EAMD.ucp` virtual root only handles exact paths:
- ✅ `/EAMD.ucp` → returns virtual `[components, scenarios]`
- ❌ `/EAMD.ucp/components` → tries to resolve as real path

**Fix**: The `pathResolve()` method IS correctly handling this:

```typescript
// Pattern 1: /EAMD.ucp/components/** or /EAMD.ucp/scenarios/**
if (normalized.startsWith('/EAMD.ucp/')) {
    const subPath = normalized.substring('/EAMD.ucp/'.length);
    // subPath is either "components/..." or "scenarios/..."
    // Map directly to projectRoot
    return path.join(this.projectRoot, subPath);
}
```

But the issue is that the **request might not be reaching StaticFileRoute** because:
1. The SPA catch-all route might be intercepting it first
2. OR the IOR in the browser is failing before the request is made

```typescript
// BEFORE (broken):
return subPath.startsWith('components/') || subPath.startsWith('scenarios/');
```

### FN.3: URL Updates During Navigation (COMPLETE)

**Issue**: Browser URL did not update when navigating through folders (clicking chevron)

**Root Cause**: `FolderItemView.handleNavigate()` dispatched `item-navigate` event with only the folder name (`uuid`), not the full path. `FolderOverView.handleItemNavigate()` then passed this incomplete path to `folderLoad()`, which called `history.pushState` with just the folder name.

**Fix**: Modified `FolderOverView.handleItemNavigate()` to construct the full path:

**File**: `src/ts/layer5/views/FolderOverView.ts`

```typescript
// FN.3: Construct full path by appending folder name to current path
const basePath = this.currentPath || this.cwd || this.rootPath || '/EAMD.ucp';
const fullPath = basePath.endsWith('/') 
  ? `${basePath}${folderName}` 
  : `${basePath}/${folderName}`;
```
