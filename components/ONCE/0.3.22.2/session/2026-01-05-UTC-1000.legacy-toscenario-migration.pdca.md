# Legacy toScenario() → UcpComponent Migration

**Status:** 🟡 PLAN  
**Created:** 2026-01-05  
**Parent:** [Iteration Tracking](../../0.3.22.1/session/2025-12-12-UTC-2100.iteration-tracking.pdca.md)

---

## 🎯 PROBLEM

15+ classes have their own `async toScenario()` implementations that are DRY violations.
Now that `UcpComponent.toScenario()` is the Single Source of Truth (sync), these need migration.

---

## 📊 ANALYSIS

### Classes with Legacy `async toScenario()`

| Class | Extends | Priority | Notes |
|-------|---------|----------|-------|
| **HTTPServer** | — | High | Core server |
| **HTTPSServer** | HTTPServer | High | TLS server |
| **HTTPRouter** | — | High | Route registry |
| **Route** | — | High | Base route |
| **UcpStorage** | — | Medium | Persistence |
| **BrowserScenarioStorage** | — | Medium | Browser storage |
| **DefaultUser** | — | Medium | User identity |
| **SemanticVersion** | — | Low | Version parsing |
| **ScenarioLoader** | — | Low | Loader |
| **RESTLoader** | — | Low | Loader |
| **TLSCertificateLoader** | — | Low | Loader |
| **ServerNameIndicationManager** | — | Low | SNI |
| **LetsEncryptCertificateProvider** | — | Low | Cert provider |
| **DomainCertificateStore** | — | Low | Cert store |
| **CertificateRenewalScheduler** | — | Low | Scheduler |

### Root Cause

These classes were created before `UcpComponent` existed or before it had `toScenario()`.
