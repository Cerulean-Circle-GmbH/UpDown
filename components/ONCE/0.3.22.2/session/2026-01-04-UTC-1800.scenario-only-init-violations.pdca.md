## **🚨 PROBLEM**

Multiple classes violate the **Web4 Core Principle**: Components MUST initialize with `Scenario<T>` ONLY.

**Scenario is the Web4 main configuration principle:**
```typescript
interface Scenario<T extends Model> {
  ior: { uuid, component, version, iorString? };  // REQUIRED — Identity
  owner: string;                                   // REQUIRED — Ownership
  model: T;                                        // REQUIRED — State
  unit?: ScenarioUnit;                            // Optional — Tracking
}
```

**Current violations pass `{ model }` without `ior` and `owner`:**

## **📋 PLAN**

### **Phase 1: UcpComponent Scenario Infrastructure (SOI.1)**

**PRINCIPLE: Single Source of Truth — NO DRY Violations — Ocam's Razor**

Every `UcpComponent` MUST support these accessors/methods:

| # | Accessor/Method | Returns | Purpose |
|---|-----------------|---------|---------|
| 1 | `get ior()` | `IOR` | Component's IOR identity |
| 2 | `get owner()` | `string` | Owner UUID from User service |
| 3 | `get scenarioUnit()` | `ScenarioUnit` | Unit tracking info |
| 4 | `toIOR()` | `IOR` | Serialize to IOR object |
| 5 | `toScenario()` | `Scenario<TModel>` | Serialize to full Scenario |
| 6 | `scenarioDefault()` | `Scenario<TModel>` | Create default Scenario |

**`toScenario()` is SIMPLE because all parts exist as accessors:**

```typescript
toScenario(): Scenario<TModel> {
  return { ior: this.ior, owner: this.owner, model: this.model, unit: this.scenarioUnit };
}
```

**Code:**
```typescript
// UcpComponent.ts — P16: TypeScript Accessors, P3: fieldNameField

// === PRIVATE FIELDS ===
private iorField: IOR | null = null;
private scenarioUnitField: ScenarioUnit | null = null;

// === ACCESSORS (P16) ===

get ior(): IOR {
  if (!this.iorField) {
    this.iorField = {
      uuid: this.model?.uuid ?? this.uuidCreate(),
      component: this.constructor.name,
      version: this.componentVersion ?? '0.0.0.0'
    };
  }
  return this.iorField;
}

get owner(): string {
  const userService = this.controller?.relatedObjectLookup(DefaultUser);
  return userService?.model?.uuid ?? 'system';
}

get scenarioUnit(): ScenarioUnit | undefined {
  return this.scenarioUnitField ?? undefined;
}

// === METHODS ===

toIOR(): IOR {
  return this.ior;
}

toScenario(): Scenario<TModel> {
  return { ior: this.ior, owner: this.owner, model: this.model, unit: this.scenarioUnit };
}

scenarioDefault(): Scenario<TModel> {
  return {
    ior: { uuid: this.uuidCreate(), component: this.constructor.name, version: this.componentVersion ?? '0.0.0.0' },
    owner: this.owner,
    model: this.modelDefault()
  };
}

init(scenario?: Scenario<TModel>): this {
  const s = scenario ?? this.scenarioDefault();
  this.iorField = s.ior;
  this.scenarioUnitField = s.unit ?? null;
  // ... initialize model from s.model
  return this;
}
```

**VALID init() calls:**
```typescript
// ✅ NO parameter — uses scenarioDefault()
let instance = new SomeWeb4Component().init();

// ✅ FULL Scenario<TModel>
let instance = new SomeWeb4Component().init(fullScenario);
```

**INVALID init() calls:**
```typescript
// ❌ WRONG — empty object is nonsense
let instance = new SomeWeb4Component().init({});

// ❌ WRONG — partial object violates Scenario<TModel>
let instance = new SomeWeb4Component().init({ model: someModel });
```

**Benefits:**
1. **Single Source of Truth** — each accessor returns ONE thing
2. **toScenario() is SIMPLE** — just assembles from accessors
3. **init() initializes JSON** — model compliant to `Scenario<TModel>`
4. **No DRY Violations** — ior, owner, model stored once, used everywhere
5. **Ocam's Razor** — minimal complexity, maximum clarity
