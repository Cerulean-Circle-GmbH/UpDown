## **🚨 PROBLEM**

Multiple classes violate the **Web4 Core Principle**: Components MUST initialize with `Scenario<T>` ONLY.

**Scenario is the Web4 main configuration principle:**
```typescript
interface Scenario<T extends Model> {
  ior: { uuid, component, version, iorString? };  // REQUIRED — Identity
  owner: string;                                   // REQUIRED — Ownership
  model: T;                                        // REQUIRED — State
  unit?: ScenarioUnit;                            // Optional — Tracking
}
```

**Current violations pass `{ model }` without `ior` and `owner`:**

**Key Insight:** Owner should ALWAYS come from the User service — not hardcoded as 'system'.

**Code:**
```typescript
// UcpComponent.ts — P16: TypeScript Accessor

/**
 * Get owner from User service
 * @returns Owner identifier from current user
 */
protected get owner(): string {
  // Lookup User service via RelatedObjects
  const userService = this.controller.relatedObjectLookup(DefaultUser);
  if (userService) {
    return userService.model.uuid;
  }
  // Fallback to 'system' only if no User service available
  return 'system';
}

/**
 * Default scenario for this component type
 * REPLACES modelDefault() — every component initializes with full Scenario
 * 
 * Override in subclass to provide component-specific defaults
 * @returns Full Scenario<TModel> with ior, owner, model
 */
protected scenarioDefault(): Scenario<TModel> {
  const model = this.modelDefault();
  return {
    ior: {
      uuid: model.uuid ?? this.uuidCreate(),
      component: this.constructor.name,
      version: this.componentVersion ?? '0.0.0.0'
    },
    owner: this.owner,  // P16: Uses getter
    model
  };
}

/**
 * Convert current state to Scenario (for persistence)
 * @returns Scenario<TModel> representing current state
 */
async toScenario(): Promise<Scenario<TModel>> {
  return {
    ior: this.ior,     // P16: Uses getter
    owner: this.owner, // P16: Uses getter
    model: this.model
  };
}

/**
 * Initialize component with scenario
 * Uses scenarioDefault() when no scenario provided
 */
init(scenario?: Scenario<TModel>): this {
  const effectiveScenario = scenario ?? this.scenarioDefault();
  // ... rest of init logic using effectiveScenario
}
```

**Benefits:**
1. **Owner always correct** — comes from User service, not hardcoded
2. **Full Scenario always** — no partial `{ model }` objects
3. **Consistent pattern** — `scenarioDefault()` replaces `modelDefault()` as the standard
4. **Type safe** — `Scenario<TModel>` enforced everywhere

```typescript
init(scenario?: Scenario<FileSystemModel>): this {
  super.init(scenario);
  
  if (!this.rootFolderInstance) {
    const folder = new DefaultFolder();
    const folderScenario = folder.scenarioDefault();
    folderScenario.model.name = 'Root';
    folderScenario.model.path = '';
    folder.init(folderScenario);
    this.rootFolderInstance = folder;
  }
  return this;
}

folderCreate(name: string, parentFolder?: DefaultFolder): DefaultFolder {
  const folder = new DefaultFolder();
  const folderScenario = folder.scenarioDefault();
  folderScenario.model.name = name;
  folderScenario.model.folderName = name;
  folderScenario.model.parent = parentFolder ? `ior:scenario:${parentFolder.model.uuid}` : null;
  folder.init(folderScenario);
  return folder;
}
```
