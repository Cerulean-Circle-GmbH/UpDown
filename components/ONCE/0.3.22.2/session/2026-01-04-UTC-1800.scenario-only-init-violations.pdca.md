## **🚨 PROBLEM**

Multiple classes violate the **Web4 Core Principle**: Components MUST initialize with `Scenario<T>` ONLY.

**Scenario is the Web4 main configuration principle:**
```typescript
interface Scenario<T extends Model> {
  ior: { uuid, component, version, iorString? };  // REQUIRED — Identity
  owner: string;                                   // REQUIRED — Ownership
  model: T;                                        // REQUIRED — State
  unit?: ScenarioUnit;                            // Optional — Tracking
}
```

**Current violations pass `{ model }` without `ior` and `owner`:**

---

## **📋 VIOLATION INVENTORY**

### **V1: DefaultCLI.ts:145**

**Current Code (WRONG):**
```typescript
init(scenario?: Scenario<CLIModel>): this {
  super.init(scenario as { model?: CLIModel });  // ❌ Cast strips ior/owner
  return this;
}
```

**Problem:** Cast to `{ model?: CLIModel }` loses type safety and violates Scenario contract.

**Fix:** Pass scenario directly without cast.

---

### **V2: DefaultFile.ts:255 — linkCreate()**

## **📋 PLAN**

### **Phase 1: Create scenarioCreate() Helper (SOI.1)**

Add a helper method to `UcpComponent` that creates a proper Scenario:

| Task | Description | File |
|------|-------------|------|
| SOI.1.1 | Add `scenarioCreate(model: TModel): Scenario<TModel>` to `UcpComponent` | `layer2/UcpComponent.ts` |

**Code:**
```typescript
/**
 * Create a full Scenario for this component
 * @param model The model state
 * @param owner Owner identifier (defaults to 'system')
 * @returns Full Scenario<TModel> with ior, owner, model
 */
protected scenarioCreate(model: TModel, owner: string = 'system'): Scenario<TModel> {
  return {
    ior: {
      uuid: model.uuid ?? this.uuidCreate(),
      component: this.constructor.name,
      version: this.componentVersion ?? '0.0.0.0'
    },
    owner,
    model
  };
}
```
