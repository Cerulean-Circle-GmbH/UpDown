## **📋 PLAN**

### **Current State (Problem)**

```typescript
// NodeJsOnce.start()
const once = new NodeJsOnce();
if (scenario) {
  once.init(scenario);  // ❌ WRONG — init only if scenario provided
}
```

**Problem:**
- `new UcpComponent()` creates a blackbox — not operational
- `init()` makes it fully operational
- But code conditionally calls `init()` only when scenario provided

### **Target State (Solution)**

```typescript
// UcpComponent base class
abstract class UcpComponent<TModel> {
  private isInitializedField: boolean = false;
  
  get isInitialized(): boolean {
    return this.isInitializedField;
  }
  
  init(scenario?: Scenario<TModel>): this {
    // ... existing init logic ...
    this.isInitializedField = true;
    return this;
  }
  
  /**
   * Guard — throws if not initialized
   * Call at start of methods that require initialization
   */
  protected requireInitialized(): void {
    if (!this.isInitializedField) {
      throw new Error(`${this.constructor.name} not initialized. Call init() first.`);
    }
  }
}

// All callers ALWAYS call init()
const once = new NodeJsOnce();
once.init(scenario);  // ✅ ALWAYS called (scenario optional)
```

### **Benefits**
1. **Explicit** — components know their initialization state
2. **Defensive** — methods can guard against uninitialized state
3. **ALWAYS init()** — fix all conditional init calls
