### **Current DRY Violations Found**

| File | Method | DRY Violation |
|------|--------|---------------|
| `IdealMinimalComponent/DefaultIdealMinimalComponent.ts` | `getWeb4TSComponent()` | Calculates `projectRoot` from `import.meta.url` |
| `NodeJsOnce.ts` | `discoverComponentRootAndVersion()` | Calculates `componentRoot`, derives but doesn't store `projectRoot` |
| `DefaultWeb4TSComponent.ts` | `updateModelPaths()` | Re-derives `projectRoot` from `componentRoot` |
| `DefaultWeb4TSComponent.ts` | `typesExtract()` | Uses `this.model.projectRoot || componentRoot` |
| `DefaultWeb4TSComponent.ts` | `componentDescriptorRead()` | Uses `context?.model?.projectRoot || this.model.projectRoot` |
| `DefaultCLI.ts` | `calculateProjectRootInternal()` | **SINGLE SOURCE** (correct!) |

### **Target Architecture (After Consolidation)**

```
CLI (calculateProjectRootInternal)
  └── cli.model.projectRoot = "/path/to/project"  ← SINGLE CALCULATION
        │
        ├── component.cli = cli  ← Reference passed
        │     └── component.projectRoot accessor → this.cli.model.projectRoot
        │
        └── web4ts.cli = component.cli  ← Reference passed through
              └── web4ts.projectRoot accessor → this.cli.model.projectRoot
              └── web4ts.model.context.projectRoot → delegate accessor
```

**Key Principles:**
1. **CLI calculates ALL project paths ONCE** in `calculateProjectRootInternal()`:
   - `projectRoot` — base path
   - `componentsDirectory` — `{projectRoot}/components`
   - `scriptsDirectory` — `{projectRoot}/scripts`
   - `scriptsVersionDirectory` — `{projectRoot}/scripts/versions`
   - `testDataDirectory` — `{projectRoot}/test/data`
2. **Components discover ONLY their own identity**:
   - `componentRoot` (from `import.meta.url`) — e.g., `/project/components/ONCE/0.3.22.2`
   - `version` (from `componentRoot` directory name) — e.g., `0.3.22.2`
3. **All project paths flow through CLI reference**: `this.cli.model.*`
4. **No model storage of derived paths**: Use accessors that delegate to CLI

### **Implementation Notes**

**PC.3 Key Change — Web4TSComponent accessor:**

```typescript
// DefaultWeb4TSComponent.ts

// Add cli property (like NodeJsOnce has)
protected cliField: CLI | null = null;
get cli(): CLI | null { return this.cliField; }
set cli(value: CLI | null) { this.cliField = value; }

// Add projectRoot accessor
get projectRoot(): string {
  // 1. CLI is path authority
  if (this.cli?.model?.projectRoot) return this.cli.model.projectRoot;
  
  // 2. Derive from context (delegating component)
  const context = (this.model as any)?.context;
  if (context?.projectRoot) return context.projectRoot;  // Use accessor
  
  // 3. Derive from componentRoot
  const componentRoot = this.model?.componentRoot;
  if (componentRoot) {
    return path.resolve(componentRoot, '..', '..', '..');
  }
  
  return '';
}
```

**PC.4 Key Change — IdealMinimalComponent simplification:**

```typescript
// BEFORE (DRY violation):
private async getWeb4TSComponent(): Promise<any> {
  const componentRoot = path.resolve(path.dirname(__filename), '../../..');
  const componentsDir = path.dirname(path.dirname(componentRoot));
  const projectRoot = path.dirname(componentsDir);  // ← DUPLICATE CALCULATION
  
  this.model.projectRoot = projectRoot;  // ← STORING DUPLICATE
  // ...
}

// AFTER (DRY compliant):
private async getWeb4TSComponent(): Promise<any> {
  // componentRoot is component-specific (from import.meta.url)
  const componentRoot = path.resolve(path.dirname(__filename), '../../..');
  this.model.componentRoot = componentRoot;
  
  // projectRoot comes from CLI (path authority)
  const projectRoot = this.projectRoot;  // ← USE ACCESSOR
  
  // Pass CLI reference to web4ts
  if (this.cli) {
    this.web4ts.cli = this.cli;
  }
  // ...
}
```
