### **Bug Fix: Server Shutdown (Discovered During MC.3)** ‚Äî ‚úÖ COMPLETE

**Problem:** After `peerStopAll` IOR call, the Node.js process didn't exit because:
1. `ServerHierarchyManager.stopServer()` only called `process.exit(0)` for client servers, not primary
2. `HTTPSServer.stop()` used `server.close()` which waits for active connections to close naturally
3. Active connections (e.g., from `curl` or browser) caused deadlock

**Solution:**
- [x] `HTTPSServer.ts`: Added socket tracking (`activeConnections: Set<Socket>`)
- [x] `HTTPSServer.ts`: Forcefully destroy all sockets before `server.close()`
- [x] `ServerHierarchyManager.ts`: Call `process.exit(0)` for both primary AND client servers
- [x] `Test02_DemoPagePlaywright.ts`: Added DEMO-11 criterion to verify server exits WITH active connections

**Key Code (`HTTPSServer.stop()`):**
```typescript
// Destroy all active connections to prevent deadlock
console.log(`üîå Closing ${this.activeConnections.size} active socket(s)...`);
for (const socket of this.activeConnections) {
    socket.destroy();
}
this.activeConnections.clear();
```

**Verification:**
- ‚úÖ Test02 DEMO-11: "Port 42777 is free - server exited despite active connections"
- ‚úÖ Manual test: `curl` to `peerStopAll` now exits cleanly
