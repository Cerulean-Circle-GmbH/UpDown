**Problem:** When `DefaultWeb4TSComponent` is directly instantiated:
```typescript
// Current: No CLI, paths in model
const component = new DefaultWeb4TSComponent().init({ model: { projectRoot: '...' }});
```

**TRON's Direction:** DefaultCLI should ALWAYS be the path authority:
```typescript
// Target: CLI is path authority even for direct instantiation

// Internally: component.cli = new DefaultCLI().init();
// Paths: component.cli.model.projectRoot
```

### **Key Principles**

1. **CLI is Path Authority** — All path calculations happen in `DefaultCLI.init()`
2. **Every Component has CLI** — Even direct instantiation creates/uses CLI
3. **Models are Path-Free** — Component models don't store duplicate paths
4. **Accessor Pattern** — `this.projectRoot` → `this.cli.model.projectRoot`

**Code Pattern:**

```typescript
// CPA.1.1-1.4: DefaultWeb4TSComponent.ts
export class DefaultWeb4TSComponent extends UcpComponent<Web4TSComponentModel> {
  // P5: Reference<T> for nullable
  private cliField: Reference<DefaultCLI> = null;

  // P16: TypeScript accessors
  get cli(): Reference<DefaultCLI> {
    return this.cliField;
  }
  
  set cli(value: DefaultCLI) {
    this.cliField = value;
  }

  // Path accessors — delegate to CLI (single source of truth)
  get projectRoot(): string {
    return this.cli?.model?.projectRoot || '';
  }
  
  get componentsDirectory(): string {
    return this.cli?.model?.componentsDirectory || '';
  }

  init(scenario?: { model?: Web4TSComponentModel; cli?: DefaultCLI }): this {
    // CPA.1.3: Create CLI if not provided
    if (!this.cliField && !scenario?.cli) {
      this.cliField = new DefaultCLI().init() as DefaultCLI;
    } else if (scenario?.cli) {
      this.cliField = scenario.cli;
    }
    
    super.init(scenario);
    return this;
  }
}
```
