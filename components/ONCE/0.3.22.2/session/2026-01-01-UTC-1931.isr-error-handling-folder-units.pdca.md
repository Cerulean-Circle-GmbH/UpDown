## PLAN

### Context

The P20 fix (static ESM import in UcpModel.ts) enables `DefaultUnit.from()` to work. However, the build crashes when the ISR (IOR Self-Replacement) pattern tries to resolve origin IOR strings in the background:

```
Error: All IOR profiles exhausted for: https://git:443/Cerulean-Circle-GmbH/UpDown/blob/dev/web4v0100/components/ONCE/0.3.22.1/src
```

### Root Cause Analysis

1. `DefaultUnit.folderUnitCreate()` sets `model.origin` to an IOR string like `ior:git:github.com/...`
2. When `toScenario()` serializes the model via `JSON.stringify()`, UcpModel's Proxy getter intercepts
3. The Proxy detects `ior:` prefix and creates an IOR object via ISR pattern
4. ISR calls `ior.resolveAndReplace()` in fire-and-forget mode (no await)
5. `resolveAndReplace()` calls `resolveInstance()` which tries HTTPSLoader
6. HTTPSLoader fails (no network/invalid URL), throws unhandled rejection
7. Node.js crashes on unhandled promise rejection

## DO

### 1. Added try/catch in IOR.resolveAndReplace()

```typescript
// src/ts/layer4/IOR.ts
async resolveAndReplace<C = T>(): Promise<C | null> {
    try {
        const instance = await this.resolveInstance<C>();
        // ... replace logic ...
        return instance;
    } catch (error) {
        // Fire-and-forget: log error but don't crash
        console.warn(`[IOR.resolveAndReplace] Fire-and-forget resolution failed:`, 
            (error as Error).message);
        return null;
    }
}
```
