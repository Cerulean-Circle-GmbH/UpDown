# PDCA: Eliminate Functional Helpers - Make Identity Model-Driven

**Component**: Web4TSComponent  
**Version**: 0.3.18.6  
**Date**: 2025-11-10 UTC 14:00  
**Status**: PLAN  
**Type**: Code Quality / Radical OOP Compliance

## Problem Statement

Current `printQuickHeader()` implementation uses functional helper methods with inline `if` statements and logic:
- `getComponentIdentity()` - calculates identity on every call with `if (this.model.context)`
- `getDelegationInfo()` - calculates delegation status with `if (!this.model.context)`
- `getTestIsolationContext()` - extracts context with regex matching on every call
- `buildTestIsolationLine()` - computes display string

**This is FUNCTIONAL SHIT, not Radical OOP!**

### Radical OOP Principle Violation

**WRONG (Current - Functional)**:
```typescript
private getComponentIdentity(): { name: string; version: string } {
  if (this.model.context) {  // ← Functional logic!
    return { name: this.model.context.model.component, ... };
  } else {
    return { name: this.model.component, ... };
  }
}
```

**RIGHT (Radical OOP - Model-Driven)**:
```typescript
// Model properties (calculated ONCE in updateModelPaths()):
model.displayName: string;           // "Web4TSComponent" or "IdealMinimalComponent"
model.displayVersion: string;        // "0.3.18.6"
model.isDelegation: boolean;         // true/false
model.delegationInfo?: string;       // "via Web4TSComponent v0.3.18.6"
model.testIsolationContext?: string; // "Web4TSComponent v0.3.18.6"

// Usage (NO logic, just read):
console.log(`${this.model.displayName} v${this.model.displayVersion}`);
if (this.model.delegationInfo) {
  console.log(`(${this.model.delegationInfo})`);
}
```

## Root Cause

The refactoring in `2025-11-10-UTC-1115.radical-oop-delegation-analysis.pdca.md` broke down the monolithic `printQuickHeader()` into smaller methods, but did NOT make them model-driven. It just moved the functional logic into smaller functions.

**Single Source of Truth violated**: Identity, delegation status, and test isolation context are CALCULATED on every access instead of being MODEL PROPERTIES.

**Breaking down a monolithic method into smaller methods is NOT Radical OOP if those methods still contain functional logic!**

- Methods read model, NO logic
- Zero `if` statements for business logic
