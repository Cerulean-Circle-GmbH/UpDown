## **📊 SUMMARY**

### **Overall Objective**

Enable Web4TSComponent to dynamically load concrete Layer2 implementations based on scenario configuration, rather than using hardcoded `Default{Component}` pattern.

**User Request (Original):**
> "the components scenario model must conatin a filed with the leayer2 implementation name which is only Default[Component] in the case that its a fundamental base implementation. in our case DefultOnce or DefultOnceKernel is abstract and the concrete name is NodeJsOnce. so the component needs to have a way to save the main implementations calss name and write it to its Once.scenario.json and recover it from there."

### **Problem Statement**

**Root Issue:**
`web4tscomponent upgrade` command failed for ONCE because:
1. It looks for `DefaultONCE.js` (hardcoded pattern)
2. Actual file is `NodeJsOnce.js` (concrete implementation)
3. No mechanism to discover the correct implementation class name

**Discovery:**
Testing revealed that `component.json` files were NEVER created by Web4TSComponent, even though the feature appeared to exist in Web4Articles.

### **Solution Architecture**

**Web4 Principle: "Everything is a Scenario"**
- Store `implementationClassName` in component's scenario model
- Create `component.json` as symlink to scenario in `scenarios/` directory
- CLI reads scenario to discover implementation class name
- Fallback: `Default${Component}` if field missing

### **Iteration 1.19: Add componentDescriptorUpdate Command** ✅ COMPLETE

**Status:** ✅ **COMPLETE** (Thu Nov 27 19:37 UTC 2025)

**Objective:** Provide CLI command to generate `component.json` for existing components

**Naming Convention (Radical OOP):**
- ✅ `componentDescriptorUpdate` groups with future methods like `componentDescriptorValidate`
- ✅ Pattern: `{noun}{verb}` - all operations on same noun grouped together
- ❌ `updateComponentDescriptor` spreads related methods across alphabetical listing
- ✅ Follows Web4 principle: "Objects before actions"

**Problem:**
- Iteration 1.18 only affects NEW components created after the fix
- ONCE 0.3.21.7 and other existing components still lack `component.json` files
- No way to generate scenario files without recreating the entire component

**Solution:**
Add `componentDescriptorUpdate()` CLI command that:
1. Requires component context (via `on()` command)
2. Extracts metadata from `this.model.context`
3. Calls existing `componentScenarioCreate()` method (renamed per Principle 16)
4. Creates scenario + symlink

**Usage:**
```bash
web4tscomponent on ONCE latest componentDescriptorUpdate
```

**Expected Output:**
- `scenarios/ONCE/0.3.21.7/{uuid}.scenario.json`
- `components/ONCE/0.3.21.7/ONCE.component.json` → symlink
