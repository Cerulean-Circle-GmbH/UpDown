# PDCA: DRY Shell Libraries - Eliminate find_project_root Duplication

**Date**: 2025-11-05 UTC 23:01  
**Component**: Web4TSComponent 0.3.18.1  
**Issue**: `find_project_root()` function duplicated in multiple shell scripts  
**Priority**: 🟡 IMPORTANT (DRY violation, maintenance nightmare)  
**Status**: 🔄 IN PROGRESS

## Problem

**BUG**: The `find_project_root()` function is duplicated in:
1. Bash CLI wrappers (generated from template)
2. `source.env` files
3. `clean-global.sh` script (FIXED)
4. Potentially other shell scripts

**ROOT CAUSE**: No DRY shell library - each script reimplements project root discovery

**IMPACT**:
- 🔴 Maintenance nightmare: Bug fixes must be applied to multiple files
- 🔴 Inconsistency: Different implementations may diverge
- 🔴 Template bloat: Templates contain repeated 30+ line functions
- 🟡 Testing overhead: Same logic tested in multiple places

**DETECTION**: User pointed out duplication after clean-global.sh security fix

## Plan

### Objective
Create DRY shell library system and eliminate all `find_project_root()` duplication

### Scope
**IN SCOPE**:
1. ✅ Create `src/sh/lib-project-root.sh` (DONE)
2. ✅ Update `clean-global.sh` to source library (DONE)
3. Update bash CLI wrapper template (`sh/version-wrapper.sh.template`)
4. Update `source.env` template or generation logic
5. Verify all shell scripts source the library

**OUT OF SCOPE**:
- Other shell libraries (do later if needed)
- TypeScript code (different language)

### Success Criteria
- ✅ All shell scripts source `lib-project-root.sh`
- ✅ No duplication of `find_project_root()` function
- ✅ All scripts work correctly (manual test)
- ✅ Templates updated so new components use library
