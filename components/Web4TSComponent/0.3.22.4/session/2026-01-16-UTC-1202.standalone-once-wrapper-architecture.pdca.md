### **Bug Fix: Getter Access in analyzeComponentMethods()**

**Problem Discovered:**
DefaultCLI's `analyzeComponentMethods()` iterates prototype methods using:
```javascript
const method = prototype[name];
```
This triggers getters like `componentRoot` on the prototype object where `ucpModel` is undefined (not null), causing:
```
TypeError: Cannot read properties of undefined (reading 'model')
```

**Fix Applied:**
Override `analyzeComponentMethods()` in Web4TSComponentCLI to use `getOwnPropertyDescriptor`:
```typescript
const descriptor = Object.getOwnPropertyDescriptor(prototype, name);
if (descriptor.get || descriptor.set) continue; // Skip getters
if (typeof descriptor.value !== 'function') continue;
```

## **ACT**

### **Summary**

Successfully created Web4TSComponent 0.3.22.4 as a thin wrapper using @web4x/once as npm dependency:

1. **Architecture:** Single local CLI file + npm package dependency
2. **Build:** Single-stage build that ensures ONCE is built first
3. **Bug Fix:** Discovered and fixed getter access issue in analyzeComponentMethods()
4. **Pattern:** Follows ONCECLI pattern with static async start() and chained .init()

### **Key Learnings**

1. **UcpComponent Initialization:** The new ONCE architecture uses UcpModel for reactive state. The `ucpModel` property is `null` on instances but `undefined` on prototypes, causing issues when iterating methods with `prototype[name]`.

2. **ONCECLI Pattern:** Empty constructor + static async start() with chained `.init()` is the proper way to initialize UcpComponent-based CLIs.

3. **Getter Safety:** When iterating prototype properties, use `getOwnPropertyDescriptor` to check if something is a function without triggering getters.

- Clean npm-based dependency management
