## Plan

### Problem Statement

**Discovered**: While investigating `once test shell` failure, discovered that the version reporting was incorrect:
```
Web4 ONCE CLI Tool v0.3.21.2 (via Web4TSComponent v0.3.21.2)
```

**Root Cause Analysis**:
1. Web4TSComponent loads via symlink: `components/Web4TSComponent/latest/` → `0.3.20.4/`
2. Version discovery in `DefaultWeb4TSComponent.init()` used `path.resolve()` which preserves symlink names
3. Path was `.../Web4TSComponent/latest/dist/ts/layer2/...`
4. `path.basename('latest')` = "latest" (not a valid version)
5. ONCE then OVERRODE the version by passing `this.model.version` (0.3.21.2) to Web4TSComponent
6. Web4TSComponent accepted the override and reported the LIE: "via Web4TSComponent v0.3.21.2"

**Impact**:
- **High**: Users cannot trust version information in CLI headers
- **High**: Delegation reporting is incorrect and misleading
- **Medium**: Violates Single Source of Truth principle for version discovery
- **Medium**: Component identity is compromised when loaded via symlinks

**Affected Code**:
- `components/Web4TSComponent/0.3.20.4/src/ts/layer2/DefaultWeb4TSComponent.ts` (lines 162-172)
- `components/ONCE/0.3.21.2/src/ts/layer2/DefaultONCE.ts` (line 213, line 239)

### Web4 Principles Violated

1. **Single Source of Truth**: Version should be discovered from filesystem, not accepted as parameter
2. **Path Authority**: Symlinks must be resolved to determine real component location
3. **Component Identity**: Each component must know its true version from its physical location

### Goals

1. **Primary**: Web4TSComponent correctly discovers its REAL version even when loaded via symlink
2. **Secondary**: Version reporting in delegation headers is accurate
3. **Tertiary**: Establish pattern for symlink-aware path discovery

### Success Criteria

- [ ] Web4TSComponent loaded via `latest` reports its real version (e.g., "0.3.20.4")
- [ ] ONCE delegation header correctly shows: "via Web4TSComponent v0.3.20.4"
- [ ] `once info` displays accurate version information
- [ ] Test isolation continues to work with symlink resolution

1. **Symlinks Hide Reality**: Node.js path operations preserve symlink names by default - must explicitly resolve
2. **Single Source of Truth**: Version should NEVER be accepted as a parameter - always discover from filesystem
3. **Trust but Verify**: When investigating one bug, found two (version discovery + test shell auto-create)

   - The symlink structure (`latest` → `0.3.20.4`)
   - The version discovery bug
   - The delegation override pattern
5. **Path Authority Pattern**: `realpathSync()` is essential for discovering true component location
