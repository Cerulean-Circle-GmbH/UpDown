## **P - PLAN**

### **1. Problem Statement**

During version upgrades (`upgrade nextBuild`), the `scenarios` directory from the source component was being copied to the new component version. This violates the Web4 principle that **scenarios ALWAYS belong at PROJECT ROOT**, never in component directories.

**Evidence**:
```bash
$ ls -la /Users/Shared/Workspaces/2cuGitHub/UpDown/components/ONCE/0.3.21.4/
drwxr-xr-x@  6 donges  wheel    192 Nov 21 19:59 scenarios  # ❌ WRONG!
```

### **2. Current Situation (AS-IS)**

- `Web4TSComponent.upgrade()` calls `createVersionFromExisting()`
- `createVersionFromExisting()` calls `copyDirectory()` to copy entire component
- `copyDirectory()` skips: `session/` and `test/data/` contents
- `copyDirectory()` **does NOT skip**: `scenarios/` ❌

**Result**: If source component has `scenarios/`, it gets copied to new version.

### **3. Target Situation (TO-BE)**

- `copyDirectory()` must skip `scenarios/` directory (like it skips `session/`)
- No scenarios directory should ever exist in component root
- Scenarios belong ONLY at:
  - **Production**: `<projectRoot>/scenarios/`
  - **Test Isolation**: `<projectRoot>/test/data/scenarios/`

### **4. Root Cause Analysis**

**Why was scenarios being copied?**
- `copyDirectory()` had no skip rule for `scenarios/`
- The method only skipped `session/` and `test/data/`

**5W1H**:
- **What**: `scenarios/` directory incorrectly copied during upgrade
- **Why**: Missing skip rule in `copyDirectory()`
- **When**: During every `upgrade nextBuild` execution
- **Where**: `Web4TSComponent/src/ts/layer2/DefaultWeb4TSComponent.ts:4416`
- **Who**: Any component using Web4TSComponent for upgrades
- **How**: `copyDirectory()` recursively copies all subdirectories without filtering

### **5. Success Criteria**

- [x] `copyDirectory()` skips `scenarios/` directory
- [x] New component versions have NO scenarios directory
- [x] Existing incorrect scenarios directories cleaned up
- [x] Warning logged when scenarios directory is skipped

## **A - ACT**

### **Lessons Learned**

**Key Principle Reinforced**:
> **Scenarios belong at PROJECT ROOT, NEVER in component directories.**

**Why This Matters**:
1. **Path Authority**: Components should not pollute their own directory with runtime data
2. **Separation of Concerns**: Code (component) vs. Data (scenarios) should be separated
3. **Upgrade Hygiene**: New versions should be clean slate for code, not carry over runtime state
4. **Test Isolation**: Test scenarios go in `test/data/scenarios/`, production in project root

**Anti-Pattern Recognition**:
```typescript
// ❌ BAD: Component creates scenarios in its own directory
const scenarioPath = path.join(__dirname, 'scenarios', filename);

// ✅ GOOD: Component uses projectRoot for scenarios
const scenarioPath = path.join(this.model.projectRoot, 'scenarios', filename);
```

### **Future Prevention**

1. **copyDirectory() Skip List**: Always review when adding new directories to component structure
2. **Upgrade Smoke Test**: After any upgrade, check for unexpected directories
3. **Component Root Audit**: Periodically scan component directories for data pollution
