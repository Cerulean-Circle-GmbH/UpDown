## **🔧 DO**

**Implementation: Remove Auto-Fix Side-Effect from check3c()**

**1. Remove Auto-Fix from check3c() - Make It Read-Only**

The fix is simple: Remove lines 1710-1722 that auto-fix files during compliance checks:

```typescript
// BEFORE (lines 1709-1723):
private async check3c(content: string, pdcaFilePath?: string): Promise<boolean> {
  // Auto-fix dual links first if we have the file path ❌ SIDE-EFFECT!
  if (pdcaFilePath) {
    const path = await import('path');
    const fs = await import('fs/promises');
    const projectRoot = await this.getProjectRoot();
    const fullPath = path.join(projectRoot, pdcaFilePath);
    
    // Try to auto-fix links ❌ MODIFIES FILE!
    const fixed = await this.fixMarkdownFile(fullPath, projectRoot, fs, path);
    if (fixed) {
      // Re-read the fixed content
      content = await fs.readFile(fullPath, 'utf-8');
    }
  }
  
  // ... continue with validation ...
}

// AFTER (read-only validation):
private async check3c(content: string, pdcaFilePath?: string): Promise<boolean> {
  // ✅ Pure validation - no side effects
  // Just check the content as-is
  
  // ... continue with validation ...
}
```

**2. Why This Works**

**Design Principle:** Compliance checks should be **read-only queries**, not **mutations**.

- ✅ **Before:** `check3c()` was trying to be helpful by auto-fixing links before checking
- ❌ **Problem:** This created a side-effect that broke dry-run isolation
- ✅ **After:** `check3c()` only validates, `fixDualLinks()` handles fixes (when not in dry-run)

**3. No Breaking Changes**

The auto-fix was an optimization, not a requirement:
- `fixAllPDCAs` already calls `fixDualLinks()` explicitly in Phase 2 (line 6044)
- That call already respects `dryRun` (line 6040)
- Removing the auto-fix from `check3c()` just prevents duplicate work
