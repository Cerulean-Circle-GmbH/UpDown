```typescript
TC-FIX-04: Strips description from filename ❌
  Error: Timestamp regex mismatch after strip
  - File after strip: 2025-11-07-UTC-080000.pdca.md (HHMMSS)
  - Regex expects: (\d{4}-\d{2}-\d{2}-UTC-\d{6})
  - But then tries to extract with: (\d{4}-\d{2}-\d{2}-UTC-\d{4,6})
  - Inconsistency causes `rename creationDate` to fail

// SKIPPED TESTS (8/12):
TC-FIX-05 through TC-FIX-12 not executed (test stopped at first failure)
```

**2. Root Cause Identified - TC-FIX-04**

The issue is in `fixAllPDCAs` line 5792:

```typescript
const timestampMatch = currentName.match(/(\d{4}-\d{2}-\d{2}-UTC-\d{6})/);
```

This regex **only matches HHMMSS** format (6 digits), but:
- Test files use HHMM format (4 digits): `2025-11-07-UTC-0800`
- After `rename strip`, file becomes `2025-11-07-UTC-080000.pdca.md` (HHMMSS)
- But the code extracts timestamp with HHMMSS-only regex
- Later `rename creationDate` can't find the file because regex doesn't match HHMM

**Fix:** Change regex to accept both formats: `/(\d{4}-\d{2}-\d{2}-UTC-\d{4,6})/`
