# Onboarding for Frontend

**Date:** 2025-07-23

## Purpose
This onboarding document provides the Frontend role with essential process context, best practices, and quality gates for UpDown. It ensures Frontend always has access to the latest workflow, process, and development steps, and can reference this file as needed.

## Key Process Points
- Frontend is responsible for identifying and implementing main client-side components/modules, including networking, state management, authentication/session, and persistence.
- Frontend must ensure all class state syncs with backend/peers using the scenario-based API.
- Frontend should always check the latest `scrum-master/process.md` for process improvements and quality gates.
- When onboarding new Frontend developers or updating the process, this file should be updated and referenced in daily work.

## Reference Documents
- `scrum-master/process.md` (authoritative process and quality gates)
- `docs/containerized-developer-workflow.md` (developer workflow)
- `sprints/iteration-n/daily-history.md` (sprint history)

## Best Practices
- Keep this onboarding file up to date with any process or workflow changes.
- Reference this file and the process documentation when planning or reviewing frontend tasks.
- Ensure all status updates and changes are documented per the definition of done.
