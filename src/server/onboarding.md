# Onboarding for Backend

**Date:** 2025-07-23

## Purpose
This onboarding document provides the Backend role with essential process context, best practices, and quality gates for UpDown. It ensures Backend always has access to the latest workflow, process, and development steps, and can reference this file as needed.

## Key Process Points
- Backend is responsible for implementing and maintaining server-side features, including WebSocket server setup and scenario synchronization.
- Backend must test server startup, document any issues or missing dependencies, and keep all implementation in sync with the latest process and workflow.
- Backend should always check the latest `scrum-master/process.md` for process improvements and quality gates.
- When onboarding new Backend developers or updating the process, this file should be updated and referenced in daily work.
