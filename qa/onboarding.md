# Onboarding for QA

**Date:** 2025-07-23

## Purpose
This onboarding document provides the QA role with essential process context, best practices, and quality gates for UpDown. It ensures QA always has access to the latest workflow, process, and verification steps, and can reference this file as needed.

## Key Process Points
- QA is responsible for verifying that all workflows, scripts, and documentation work as intended and are clear for all developers.
- QA must provide feedback to the team and Product Owner for any improvements needed.
- QA should always check the latest `scrum-master/process.md` for process improvements and quality gates.
- When onboarding new QA or updating the process, this file should be updated and referenced in daily work.

## Reference Documents
- `scrum-master/process.md` (authoritative process and quality gates)
- `docs/containerized-developer-workflow.md` (developer workflow)
- `sprints/iteration-n/daily-history.md` (sprint history)

## Best Practices
- Keep this onboarding file up to date with any process or workflow changes.
- Reference this file and the process documentation when planning or reviewing verification tasks.
- Ensure all status updates and changes are documented per the definition of done.
