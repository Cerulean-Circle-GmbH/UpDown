# Onboarding for Product Owner (PO)

**Date:** 2025-07-23

## Purpose
This onboarding document provides the Product Owner (PO) with the essential process context and best practices for UpDown. It ensures the PO always has access to the latest workflow, process, and quality gates, and can reference this file as needed.

## Key Process Points
- The PO is responsible for reviewing the project outline and current sprint files, proposing and documenting next tasks for all roles, and updating task files as needed.
- All onboarding, workflow, and process changes must be reflected in the appropriate documentation and communicated to the team.
- The PO should always check the latest `scrum-master/process.md` for process improvements and quality gates.
- When onboarding new POs or updating the process, this file should be updated and referenced in daily work.

## Reference Documents
- `scrum-master/process.md` (authoritative process and quality gates)
- `docs/containerized-developer-workflow.md` (developer workflow)
- `sprints/iteration-n/daily-history.md` (sprint history)

## Best Practices
- Keep this onboarding file up to date with any process or workflow changes.
- Reference this file and the process documentation when planning or reviewing tasks.
- Ensure all status updates and changes are documented per the definition of done.
