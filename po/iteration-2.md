# Role-Based Prompts for WBS Tasks - Iteration 2

**Date:** 2025-07-21

## 1. Define the high-level product specification for the UpDown game
**Role:** Game Designer / Product Owner
- Prompt: Describe the core gameplay loop, user roles, win/loss conditions, and unique features of the UpDown game.

## 2. Outline the main architectural layers: client, server, and communication
**Role:** Solution Architect
- Prompt: Create a high-level diagram and description of the main architectural layers, including client, server, and communication protocols (P2P, WebSocket, etc.).

## 3. Identify the main components/modules for the server
**Role:** Backend Developer / Architect
- Prompt: List and briefly describe the main server-side modules (game logic, lobby management, player state, etc.).

## 4. Identify the main components/modules for the client
**Role:** Frontend Developer / Architect
- Prompt: List and briefly describe the main client-side modules (UI, state management, networking, etc.).

## 5. Specify the initial data models and interfaces for communication between client and server
**Role:** Full Stack Developer / API Designer
- Prompt: Define the initial data models and API interfaces for client-server communication, including message formats and protocols.

## 6. Document the technology stack and libraries to be used for each layer/component
**Role:** Tech Lead / Architect
- Prompt: Specify the recommended technology stack and libraries for each architectural layer and component, justifying choices where relevant.
