### **Lessons Learned**

**Why This Happened:**
- PDF extraction failed to recognize wide multi-column table
- Column-first extraction mixed up content from different categories
- PDF had visual column separators that extraction couldn't interpret

**What Worked:**
- TRON's PDF screenshot showed exact 8-column layout
- Using `<br>` tags for multi-item cells maintained readability
- Cross-referencing VIP table avoided duplication

**Process Improvement:**
- Wide tables (>5 columns) need special attention during conversion
- Visual PDF inspection critical for understanding table structure
- Using `<br>` in markdown tables handles complex cell content well
