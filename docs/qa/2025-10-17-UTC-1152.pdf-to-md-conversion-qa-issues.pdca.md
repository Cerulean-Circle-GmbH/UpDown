## **🔧 DO**

### **1. Issue Analysis**

#### **Issue 1: Table Structure Loss (HIGH SEVERITY)**

**Evidence - Example 1 (Ranking Rewards):**
```markdown
Lines 565-589:
1st place
Daily
Weekly
Monthly
Yearly

### 100 D
1'000 D
10'000 D
100'000 D

### 200 D
2'000 D
20'000 D
200'000 D

### 300 D
3'000 D
30'000 D
300'000 D
```

**What It Should Be:**
```markdown
| Ranking | Daily | Weekly | Monthly | Yearly |
|---------|-------|--------|---------|--------|
| 1st place | 100 D | 1'000 D | 10'000 D | 100'000 D |
| 2nd place | 200 D | 2'000 D | 20'000 D | 200'000 D |
| 3rd place | 300 D | 3'000 D | 30'000 D | 300'000 D |
```

**Root Cause:**
- PDF table cells converted to individual headings
- Table structure not recognized by conversion tool
- Column headers separated from data

**Impact:**
- Information unreadable/incomprehensible
- Wrong semantic structure (headings vs. table cells)
- TOC polluted with currency amounts
- Data relationships lost

**Evidence - Example 2 (Achievement Rewards):**
```markdown
Lines 758-762:
### 50 D

### 100 D

### 1'000 D

Title rewards
```

**Root Cause:**
- Likely a reward tier table
- Context (achievement names) missing or separated
- Table converted to standalone headings

#### **Issue 2: TOC Pollution (MEDIUM SEVERITY)**

**Evidence:**
```markdown
Lines 22-45 (Table of Contents):
- [100 D](#100-d)
- [200 D](#200-d)
- [300 D](#300-d)
- [1'000 D](#1000-d)
- [10'000 D](#10000-d)
... (17 currency entries total)
```

**Root Cause:**
- Automatic TOC generation picked up currency headings
- No semantic understanding that these aren't real sections

**Impact:**
- TOC unusable for navigation
- Professional appearance damaged
- Actual sections hard to find
- User experience severely degraded

#### **Issue 3: Missing Context (HIGH SEVERITY)**

**Evidence:**
```markdown
Lines 571-589: Currency amounts with no description of what they represent
Lines 758-762: Reward amounts with no achievement names
```

**Root Cause:**
- PDF layout information lost
- Text positioning/proximity not preserved
- Column/row associations broken

**Impact:**
- Data meaningless without context
- Cannot understand reward structure
- Business logic unclear
- Specifications incomplete

#### **Issue 4: Semantic Structure Corruption (HIGH SEVERITY)**

**Evidence:**
- Table cells (data) marked as headings (###)
- List items may be missing
- Hierarchy incorrect (### for currency vs. ## for actual sections)

**Root Cause:**
- PDF conversion tool lacks semantic understanding
- Visual structure (bold, size) mapped to markdown without context
- No validation of resulting structure

**Impact:**
- Document structure broken
- Navigation broken
- Assistive technology (screen readers) will misinterpret
- Maintainability compromised

#### **Issue 5: Number Format Inconsistency (LOW SEVERITY)**

**Evidence:**
```markdown
1'000 D    (apostrophe as thousand separator)
10'000 D
20'000 D
vs.
100 D      (no separator needed)
200 D
```

**Root Cause:**
- European number format preserved from PDF
- Inconsistent application (sometimes present, sometimes not)
- May be locale-specific formatting

**Impact:**
- Minor: Unusual for markdown but technically valid
- May confuse parsers expecting US format (1,000)
- Aesthetic inconsistency
