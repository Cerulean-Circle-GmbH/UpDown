### **Root Cause**
I relied on PDF text extraction which presented table data in column-first format, making it difficult to map achievements to rewards correctly. When TRON provided direct PDF screenshot, the error became immediately visible.

### **Errors Identified**
Three achievement rewards were incorrect due to pattern misalignment:

| Achievement | MY ERROR | PDF REALITY |
|-------------|----------|-------------|
| Streak 40 | 2 cards + 2'000 D | 1 card + 1'000 D |
| Streak 45 | 5 cards + 5'000 D | 2 card + 2'000 D |
| Streak 50 | 1'000 D only | 5 cards + 5'000 D |

### **Pattern Recognition Failure**
The correct progression pattern is:
- Streak 35: **1'000 D** (currency only)
- Streak 40: **1 card + 1'000 D** (starts card rewards)
- Streak 45: **2 card + 2'000 D** (doubles)
- Streak 50: **5 cards + 5'000 D** (escalates significantly)

I had shifted the rewards "up" by one level, mapping streak 45's reward to 40, streak 50's reward to 45, etc.

### **Root Cause Analysis**

**Why did this happen?**
1. **PDF Extraction Limitation:** `pdf2txt.py` output presented table in column-first format
2. **Insufficient Visual Verification:** I relied on text extraction instead of visual PDF inspection
3. **Pattern Assumption:** I assumed the pattern without verifying against source
4. **Partial Verification:** Fixed streak 35 but didn't re-verify the full progression

**Why didn't I catch it in CHECK phase?**
- I verified currency values matched
- I verified rewards existed
- I did NOT verify the reward-to-achievement mapping was correct
- I flagged streak 50 as "suspicious" but didn't investigate why

### **Process Improvements**

**For PDF-to-MD Table Conversions:**
1. ✅ **Visual First:** Always inspect PDF visually before relying on text extraction
2. ✅ **Pattern Recognition:** Identify logical progressions and verify they make sense
3. ✅ **Full Re-verification:** When fixing errors, re-verify surrounding context
4. ✅ **Trust TRON Evidence:** When TRON provides screenshots, prioritize over extraction

**For QA Reporting:**
1. ⚠️ **Link Verification:** Test all PDCA links before reporting (my link was broken for TRON)
2. ⚠️ **"Understand First" Protocol:** When TRON says "still not correct", stop and compare systematically
